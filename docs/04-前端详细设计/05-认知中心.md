# 认知中心 - 详细设计

> 路由前缀：`/cognitive`
> 对应：悟(Grasp) — 知识库、认知评估、注入管理
> 子页面：知识库、评估、注入管理

---

## 1. 知识库（/cognitive/knowledge）

### 1.1 页面布局

```
┌────────────────────────────────────────────────────────────────┐
│ 🧠 认知知识库                                    [🔍 搜索...]   │
│ Grasp 知识库中的所有认知文档                    [类型筛选▼]     │
├────────────────────────────────────────────────────────────────┤
│ [全部] [meta] [经验] [教训] [流程]              [重置筛选]     │
├────────────────────────────────────────────────────────────────┤
│ 认知标题       │ 类型  │ 标签         │ 来源  │ 创建时间  │ 操作 │
│ ────────────────────────────────────────────────────────────── │
│ 预案匹配经验   │ 经验  │ workflow     │ 刚子  │ 04-15    │ 查看  │
│ 地震救援教训   │ 教训  │ rescue       │ 谷子  │ 04-11    │ 查看  │
│ 资源调度流程   │ 流程  │ schedule     │ 麻子  │ 04-10    │ 查看  │
├────────────────────────────────────────────────────────────────┤
│ 共 25 条                          [上一页] 1 2 3 [下一页]      │
└────────────────────────────────────────────────────────────────┘
```

### 1.2 字段定义

| 字段 | 数据来源 | 显示格式 | 交互 |
|------|----------|----------|------|
| 认知标题 | `cognition.title` | 文本，最多 25 字 | 点击 → 打开认知详情 Modal |
| 类型 | `cognition.type` | 标签：meta(蓝)/经验(绿)/教训(橙)/流程(紫) | |
| 标签 | `cognition.tags` | 多个小标签，最多 3 个 | 超出显示 `+N` |
| 来源 | `cognition.source` 或 `cognition.agent_id` | 智能体名称 | |
| 创建时间 | `cognition.created_at` | `YYYY-MM-DD` | |
| 操作 | — | [查看] 按钮 | 打开认知详情 Modal |

### 1.3 认知详情 Modal

**触发**：点击认知行 / [查看]

**内容**：

| 区块 | 字段 | 显示格式 |
|------|------|----------|
| 头部 | 标题 | 大标题 |
| 头部 | 类型 | 标签 |
| 头部 | 标签 | 多个标签 |
| 头部 | 来源 | 智能体名称 |
| 头部 | 创建时间 | `YYYY-MM-DD HH:mm` |
| 内容 | 正文 | `cognition.content`，支持 Markdown 渲染 |
| 底部 | 关闭按钮 | [关闭] |

**关闭方式**：[关闭] 按钮 / ESC 键 / 点击遮罩

### 1.4 筛选

| 筛选项 | 选项 | 说明 |
|--------|------|------|
| 类型 | 全部/meta/经验/教训/流程 | 从认知数据中提取所有类型 |
| 搜索 | 输入关键词 | 300ms 防抖 |

### 1.5 API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 加载认知列表 | GET | `/api/v1/grasp/knowledge` | |
| 搜索 | 前端本地过滤 | — | 如果后端不支持 `q` 参数 |

## 3. 注入管理（/cognitive/inject）

### 3.1 页面布局

```

├────────────────────────────────────────────────────────────────┤
│ ── 注入历史 ─────────────────────────────────────────────────── │
│ 时间    │ 来源     │ 类型           │ 认知数 │ 状态            │
│ ────────────────────────────────────────────────────────────── │
│ 09:30   │ 任务完成 │ task_result    │   3    │ ✅ 成功         │
│ 09:25   │ 工作流   │ workflow_result│   5    │ ✅ 成功         │
```

### 3.2 字段定义

| 区块 | 字段 | 数据来源 | 显示格式 |
|------|------|----------|----------|
| 注入历史 | 时间 | 注入操作时间 | `HH:mm` |
| 注入历史 | 来源 | 触发来源 | 文本：任务完成/工作流完成/争议解决 |
| 注入历史 | 类型 | 注入类型 | 标签：task_result/workflow_result/dispute_result |
| 注入历史 | 认知数 | 注入的认知数量 | 数字 |
| 注入历史 | 状态 | 注入结果 | ✅成功(绿) / ❌失败(红) |

| 注入规则 | 开关 | 规则状态 | Toggle 开关 |

### 3.3 交互

| 元素 | 触发行为 | 说明 |
|------|----------|------|
| Toggle 开关 | PATCH `/api/v1/grasp/inject/rules/{rule_id}` | 切换开/关 |
| 注入历史行 | 点击展开详情 | 显示注入的完整内容 |

### 3.4 API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 加载注入状态 | GET | `/api/v1/grasp/inject/status` | 返回结构见 §4 |
| 切换规则 | PATCH | `/api/v1/grasp/inject/rules/{id}` | 请求/返回结构见 §4 |

## 4. API 返回结构定义

### 4.1 注入状态 API

**端点**：`GET /api/v1/grasp/inject/status`

**返回结构**：
```typescript
interface InjectStatusResponse {
  service_status: 'running' | 'stopped' | 'degraded';
  recent_injections: InjectLogEntry[];  // 最近 50 条
}

interface InjectLogEntry {
  id: string;
  source: 'task' | 'workflow' | 'dispute';
  type: 'task_result' | 'workflow_result' | 'dispute_result';
  cognition_count: number;
  status: 'success' | 'failed';
  error_message?: string;  // 失败时有值
  created_at: string;       // ISO 8601
}
```

### 4.2 注入规则 API

**端点**：`GET /api/v1/grasp/inject/rules`

**返回结构**：
```typescript
interface InjectRule[] {
  id: string;
  name: string;
  trigger_condition: string;   // 如 "task.status=done"
  target_kb: string;           // 如 "default" 或 "experience"
  enabled: boolean;
  created_at: string;
  updated_at: string;
}
```

**切换规则**：`PATCH /api/v1/grasp/inject/rules/{id}`

**请求体**：
```typescript
interface UpdateRuleRequest {
  enabled: boolean;
}
```

**返回结构**：
```typescript
interface UpdateRuleResponse {
  success: boolean;
  rule: InjectRule;
}
```

**错误处理**：
| HTTP 状态码 | 返回 | 前端处理 |
|-------------|------|----------|
| 404 | `{ "error": "Rule not found" }` | Toast `规则不存在` |
| 400 | `{ "error": "Invalid request" }` | 红色提示 |
| 500 | `{ "error": "Internal error" }` | 红色横幅 + [重试] |

## 4. 验收标准

| # | 验收项 | 测试方法 | 通过标准 |
|---|--------|----------|----------|
| 1 | 知识库加载 | 访问 `/cognitive/knowledge` | 显示认知文档列表 |
| 2 | 知识库搜索 | 输入关键词 | 300ms 后过滤 |
| 3 | 类型筛选 | 选择不同类型 | 列表正确过滤 |
| 4 | 认知详情 | 点击认知行 | 打开 Modal，显示 Markdown 内容 |
| 5 | 评估数据加载 | 访问 `/cognitive/assessment` | 显示智能体评分表格 |
| 6 | 评估 Tab 切换 | 切换维度 Tab | 进度条更新 |
| 7 | 评分颜色 | 不同分数段 | 颜色正确（绿/蓝/橙/红）|
| 8 | 注入管理 | 访问 `/cognitive/inject` | 显示注入历史 + 规则（或开发中提示）|
| 9 | API 不可用 | 关闭端点 | 友好提示 |
| 10 | 空状态 | 无认知数据 | `暂无认知数据` |
