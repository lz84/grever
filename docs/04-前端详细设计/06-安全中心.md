| 区块 | 字段 | 数据来源 | 显示格式 |
|------|------|----------|----------|
| 服务状态 | 服务名称 | 配置 | 文本 |
| 服务状态 | 状态 | 健康检查 API | 🟢正常/🟡降级/🔴异常 |
| 服务状态 | 延迟 | 健康检查响应时间 | `x ms` |
| 服务状态 | 运行时间 | 服务启动时间 | `x 天 x 小时` |
| 资源使用 | CPU | 系统指标 | 进度条 + 百分比 |
| 资源使用 | 内存 | 系统指标 | 进度条 + 百分比 |
| 资源使用 | 磁盘 | 系统指标 | 进度条 + 百分比 |
| 智能体健康 | 智能体名称 | `agent.name` | 文本 |
| 智能体健康 | 状态 | `agent.status` | 在线(绿)/繁忙(橙)/离线(红) |
| 智能体健康 | 最后心跳 | `agent.last_heartbeat` | 相对时间：`10 秒前`/`15 分钟前` |
| 智能体健康 | 连续失败 | 失败计数 | 数字 |
| 智能体健康 | 当前任务 | `agent.current_tasks` | 数字 |

| SSE 推送 | `eventStream.connect()` | 智能体状态变更实时更新 |

## 4. API 返回结构定义

### 4.1 审计日志 API

**端点**：`GET /api/v1/audit/logs`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `actor` | string | ❌ | 操作者过滤 |
| `action` | string | ❌ | 操作类型过滤 |
| `since` | string | ❌ | 起始时间 ISO 8601 |
| `until` | string | ❌ | 结束时间 ISO 8601 |
| `q` | string | ❌ | 搜索关键词 |
| `limit` | number | ❌ | 默认 20 |
| `offset` | number | ❌ | 默认 0 |

**返回结构**：
```typescript
interface AuditLogResponse {
  total: number;
  items: AuditLogEntry[];
}

interface AuditLogEntry {
  id: string;
  created_at: string;        // ISO 8601
  actor: string;             // 智能体名称/用户名
  action: 'create' | 'update' | 'delete' | 'config';
  target: string;            // 操作目标
  detail: string;            // 操作详情
}
```

**错误处理**：
| HTTP 状态码 | 返回 | 前端处理 |
|-------------|------|----------|
| 400 | `{ "error": "Invalid filter" }` | Toast `筛选条件无效` |
| 401 | `{ "error": "Unauthorized" }` | 跳转登录 |
| 500 | `{ "error": "Internal error" }` | 红色横幅 + [重试] |

**导出**：`GET /api/v1/audit/logs?export=csv`
- 返回 `text/csv` 文件
- 文件名：`audit_log_{YYYY-MM-DD}.csv`

### 4.2 告警 API

**端点**：`GET /api/v1/alerts`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `status` | string | ❌ | `open`/`in_progress`/`resolved` |
| `severity` | string | ❌ | `high`/`medium`/`low` |
| `limit` | number | ❌ | 默认 20 |
| `offset` | number | ❌ | 默认 0 |

**返回结构**：
```typescript
interface AlertResponse {
  total: number;
  items: Alert[];
}

interface Alert {
  id: string;
  title: string;
  severity: 'high' | 'medium' | 'low';
  source: string;             // 告警来源
  status: 'open' | 'in_progress' | 'resolved';
  created_at: string;         // ISO 8601
  updated_at?: string;        // ISO 8601
  resolved_at?: string;       // ISO 8601
  resolved_by?: string;       // 解决者
  resolution_note?: string;   // 解决备注
}
```

**更新告警状态**：`PATCH /api/v1/alerts/{id}`

**请求体**：
```typescript
interface UpdateAlertRequest {
  status: 'open' | 'in_progress' | 'resolved';
  resolution_note?: string;
}
```

**返回结构**：
```typescript
interface UpdateAlertResponse {
  success: boolean;
  alert: Alert;
}
```

**全部标记已读**：`PATCH /api/v1/alerts/mark-read`
- 无需请求体
- 返回 `{ "success": boolean, "count": number }`

## 4. 验收标准

| # | 验收项 | 测试方法 | 通过标准 |
|---|--------|----------|----------|
| 1 | 监控页面加载 | 访问 `/security/monitor` | 显示服务状态 + 资源使用 + 智能体健康 |
| 2 | 自动刷新 | 等待 10 秒 | 数据自动更新 |
| 3 | 服务状态颜色 | 不同状态 | 🟢正常/🟡降级/🔴异常 |
| 4 | 审计页面加载 | 访问 `/security/audit` | 显示审计日志（或开发中提示）|
| 5 | 审计筛选 | 选择类型/操作者/时间 | 列表正确过滤 |
| 6 | 告警页面加载 | 访问 `/security/alerts` | 显示告警列表 |
| 7 | 告警筛选 | 选择状态 | 列表正确过滤 |
| 8 | 告警处理 | 点击 [处理] | 打开处理 Modal，状态更新 |
| 9 | API 不可用 | 关闭端点 | 友好提示 |
