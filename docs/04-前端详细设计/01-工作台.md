## 2. 字段定义

### 2.1 统计卡片

| 卡片 | 数据来源 | 显示格式 | 点击跳转 |
|------|----------|----------|----------|
| 目标数 | `GET /api/v1/goals` 的 `total` 字段 | 数字 | `/coordination/goals` |
| 项目数 | `GET /api/v1/projects` 的 `total` 字段 | 数字 | `/coordination/projects` |
| 任务数 | `GET /api/v1/tasks` 的 `total` 字段 | 数字 | `/coordination/tasks` |
| 智能体数 | `GET /api/v1/agents` 数组长度 | 数字 | `/system/agents` |
| 活跃冲突数 | `GET /api/v1/disputes?status=active` 数组长度 | 数字（红色高亮）| `/security/alerts` |

### 2.2 快捷入口

| 按钮 | 跳转 | 说明 |
|------|------|------|
| [新建目标] | `/coordination/goals/new` | |
| [查看执行] | `/coordination/executions` | |
| [场景库] | `/scenarios` | |
| [智能体状态] | `/system/agents` | |

### 2.3 最近目标（最近 5 条）

| 字段 | 数据来源 | 显示格式 | 说明 |
|------|----------|----------|------|
| 目标名称 | `goal.title` | 文本，最多 20 字 | 点击 → `/coordination/goals/:id` |
| 状态 | `goal.status` | 标签：进行中(蓝)/已完成(绿)/已计划(灰) | |
| 进度 | 通过 `GET /api/v1/tasks?goal_id={id}` 计算 `已完成/总数` | 进度条 | 百分比显示 |
| 更新时间 | `goal.updated_at` | 相对时间：`2 分钟前`/`1 小时前`/`昨天` | |

### 2.4 最近执行（最近 5 条）

| 字段 | 数据来源 | 显示格式 | 说明 |
|------|----------|----------|------|
| 执行名称 | `trace.task_title` 或 `workflow.name` | 文本，最多 20 字 | 点击 → `/coordination/executions/:id` |
| 状态 | `trace.final_state` 或 `workflow.status` | 标签：运行中(蓝)/已完成(绿)/失败(红) | |
| 步骤 | `workflow.steps` | `已完成/总数` | |
| 耗时 | `trace.total_duration_ms` | `x 分钟` | 运行中显示"运行中" |

### 2.5 智能体状态

| 字段 | 数据来源 | 显示格式 | 说明 |
|------|----------|----------|------|
| 智能体名称 | `agent.name` | 文本 | 点击 → `/system/agents` |
| 状态 | `agent.status` | 圆点 + 文字：在线(绿)/繁忙(橙)/离线(红) | |
| 负载 | `agent.load` | 百分比 | >90% 红色 |
| 当前任务 | `agent.current_tasks` | 数字 | |

## 3. API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 统计-目标数 | GET | `/api/v1/goals` | 取 `total` 或数组长度 |
| 统计-项目数 | GET | `/api/v1/projects` | 取 `total` 或数组长度 |
| 统计-任务数 | GET | `/api/v1/tasks` | 取 `total` 或数组长度 |
| 统计-智能体数 | GET | `/api/v1/agents` | 数组长度 |
| 统计-冲突数 | GET | `/api/v1/disputes?status=active` | 数组长度 |
| 最近目标 | GET | `/api/v1/goals?sort=updated_at&limit=5` | |
| 最近执行 | GET | `/api/v1/traces?limit=5` | |
| 智能体状态 | GET | `/api/v1/agents` | |

> **优化**：以上 8 个请求应**并行发起**（`Promise.all`），而非串行。

## 4. 错误处理

| 场景 | 显示 | 操作 |
|------|------|------|
| 部分 API 失败 | 失败的卡片显示 `--`，其余正常显示 | 不阻断页面 |
| 全部 API 失败 | 红色横幅 + 居中提示：`无法加载工作台数据` | [重试] |
| 无目标数据 | 最近目标区块显示：`暂无目标，点击「新建目标」开始` | [新建目标] |
| 无执行数据 | 最近执行区块显示：`暂无执行记录` | [查看执行] |
| 加载中 | 各区块显示骨架屏（3 行）| 自动消失 |

## 5. 自动刷新

| 配置 | 值 | 说明 |
|------|------|------|
| 刷新间隔 | 30 秒 | 自动刷新统计数据 |
| SSE 推送 | `eventStream.connect()` | 智能体状态变更实时更新 |

## 6. 验收标准

| # | 验收项 | 测试方法 | 通过标准 |
|---|--------|----------|----------|
| 1 | 5 个统计卡片加载 | 访问 `/` | 数字正确显示 |
| 2 | 统计卡片点击跳转 | 点击任意卡片 | 跳转到正确页面 |
| 3 | 4 个快捷入口 | 点击任意按钮 | 跳转到正确页面 |
| 4 | 最近目标显示 | 查看列表 | 最多 5 条，按更新时间倒序 |
| 5 | 最近执行显示 | 查看列表 | 最多 5 条，状态颜色正确 |
| 6 | 智能体状态显示 | 查看列表 | 状态圆点颜色正确 |
| 7 | 自动刷新 | 等待 30 秒 | 数据自动更新 |
| 8 | 部分 API 失败 | 关闭部分端点 | 失败卡片显示 `--`，其余正常 |
| 9 | 全部 API 失败 | 关闭后端 | 红色横幅 + [重试] |
| 10 | 空状态 | 删除所有数据 | 友好提示文案 |
