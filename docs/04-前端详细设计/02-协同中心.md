# 协同中心 - 详细设计

> 路由前缀：`/coordination`
> 对应：御(Reins) — 目标/项目/任务/执行的协同调度
> 子页面：目标列表、目标详情、新建目标、项目列表、项目详情、任务列表、执行列表、执行详情

---

## 1. 目标列表（/coordination/goals）

### 1.1 页面布局

```
┌────────────────────────────────────────────────────────────────┐
│ 🎯 目标管理                                      [🔍 搜索...]   │
│ 管理所有业务目标                                [新建目标]      │
├────────────────────────────────────────────────────────────────┤
│ [全部▼] [状态▼] [优先级▼]                        [重置筛选]     │
├────────────────────────────────────────────────────────────────┤
│ 目标名称     │ 优先级 │ 状态  │ 项目数 │ 任务进度 │ 更新时间  │
│ ────────────────────────────────────────────────────────────── │
│ 城市应急...  │ P1-高  │ 进行中│   3    │ 12/20   │ 2 分钟前   │
│ 智能投资...  │ P2-中  │ 已计划│   2    │  0/15   │ 1 小时前   │
│ 抢险救灾...  │ P0-紧急│ 已完成│   1    │  5/5    │ 昨天       │
├────────────────────────────────────────────────────────────────┤
│ 共 12 条                          [上一页] 1 2 [下一页]        │
└────────────────────────────────────────────────────────────────┘
```

### 1.2 字段定义

| 字段 | 数据来源 | 显示格式 | 交互 |
|------|----------|----------|------|
| 目标名称 | `goal.title` | 文本，最多 20 字，超出 `...` | 点击 → `/coordination/goals/:id` |
| 优先级 | `goal.priority` | 标签：P0-紧急(红) / P1-高(橙) / P2-中(蓝) / P3-低(灰) | |
| 状态 | `goal.status` | 标签：进行中(蓝) / 已完成(绿) / 已计划(灰) / 已取消(灰) | |
| 项目数 | `GET /api/v1/projects` 前端按 `goal_id` 分组计数 | 数字 | 0 显示 `-` |
| 任务进度 | `GET /api/v1/tasks` 前端按 `goal_id` 分组计算 `已完成/总数` | `x/y` + 进度条 | |
| 更新时间 | `goal.updated_at` | 相对时间：`2 分钟前`/`1 小时前`/`昨天` | |

### 1.3 筛选

| 筛选项 | 选项 | 说明 |
|--------|------|------|
| 状态 | 全部/进行中/已完成/已计划/已取消 | 前端本地过滤 |
| 优先级 | 全部/P0/P1/P2/P3 | 前端本地过滤 |
| 搜索 | 输入关键词 | 300ms 防抖，过滤 `title` 和 `description` |

### 1.4 API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 加载目标列表 | GET | `/api/v1/goals` | |
| 获取项目数 | GET | `/api/v1/projects` | 一次性获取全部，前端按 `goal_id` 分组计数 |
| 获取任务进度 | GET | `/api/v1/tasks` | 一次性获取全部，前端按 `goal_id` 分组计数 |

> **关键优化**：项目数和任务进度**不要**每个目标单独调用 API。一次性获取所有项目和任务，前端本地按 `goal_id` 分组计数。

### 1.5 新建目标按钮

**点击** → 跳转 `/coordination/goals/new`

### 2.2 表单字段

| 字段 | 类型 | 必填 | 默认值 | 验证规则 |
|------|------|------|--------|----------|
| 目标名称 | 文本输入 | ✅ | 空 | 1-100 字符，不能为空 |
| 描述 | 文本域 | ❌ | 空 | 最多 500 字符 |
| 优先级 | 单选 | ❌ | `P2` | P0/P1/P2/P3 |
| 截止日期 | 日期选择器 | ❌ | 空 | 格式 `YYYY-MM-DD` |

### 2.3 场景实例化参数传递

当用户从场景库点击 [实例化此场景] 时，跳转到新建目标页面并预填：

**URL 参数**：`/coordination/goals/new?scenario_id={id}`

**预填字段**：
| 场景字段 | 目标表单字段 | 说明 |
|----------|-------------|------|
| `scenario.name` | 目标名称 | 直接预填 |
| `scenario.description` | 描述 | 直接预填 |
| `scenario.steps` | 不直接预填 | 场景步骤在目标详情页展示，可一键创建为子项目 |

### 2.4 提交行为

| 结果 | 行为 |
|------|------|
| 成功 | POST `/api/v1/goals` → 跳转 `/coordination/goals/{id}` → Toast `目标创建成功` |
| 验证失败 | 表单底部红色错误提示，必填字段标红 |
| API 失败 | 表单底部红色提示 `创建失败：{error.message}` |

### 3.2 字段定义

| 区块 | 字段 | 数据来源 | 显示格式 | 交互 |
|------|------|----------|----------|------|
| 头部 | 标题 | `goal.title` | 大标题 | |

| 执行 | 步骤 | `workflow.steps.length` | `已完成/总数` | |

### 3.3 按钮与交互

| 元素 | 触发行为 | 说明 |
|------|----------|------|
| ← 返回列表 | 跳转 `/coordination/goals` | |
| [新建项目] | 跳转 `/coordination/projects/new?goal_id={id}` | 预填所属目标 |
| [激活执行] | POST `/api/v1/workflows/from-goal?goal_id={id}` | 成功后跳转 `/coordination/executions` |

**激活执行的边界情况**：
- 已有运行中的 workflow：按钮禁用 + Tooltip `已有执行中的工作流`
- 激活中：按钮显示加载中
- 激活失败：红色 Toast `激活失败：{error.message}`

### 3.4 API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 加载目标详情 | GET | `/api/v1/goals/{id}` | |
| 加载关联项目 | GET | `/api/v1/projects?goal_id={id}` | |
| 加载执行记录 | GET | `/api/v1/workflows?goal_id={id}` | |
| 激活执行 | POST | `/api/v1/workflows/from-goal?goal_id={id}` | |

| 耗时 | `workflow.completed_at - workflow.started_at` | `x 分钟` | 运行中显示"运行中" |

### 7.3 筛选

| 筛选项 | 说明 |
|--------|------|
| 状态 | 全部/运行中/已完成/失败/已取消 |
| 时间范围 | 今天/最近 7 天/最近 30 天/自定义 |

### 7.4 SSE 实时推送

**连接**：页面加载时 `eventStream.connect()`

**更新行为**：
- 新执行出现：列表顶部插入
- 状态变更：对应行实时更新
- 断线重连：自动重连，最多重试 3 次

### 7.5 API 调用

| 操作 | 方法 | 端点 | 说明 |
|------|------|------|------|
| 加载执行列表 | GET | `/api/v1/workflows` | |
| SSE 连接 | WebSocket | `eventStream.connect()` | 实时推送 |
