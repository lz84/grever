# 演示数据说明

Grever 内置了演示数据，用于快速体验系统的各项功能。演示数据基于真实的软件开发场景设计。

## 数据概览

| 类型 | 数量 | 说明 |
|------|------|------|
| Goals | 3 | 业务目标层 |
| Projects | 11 | 项目层 |
| Tasks | 42 | 任务层 |
| Agents | 5 | 智能体/角色 |

## Goals（业务目标）

1. **电商系统 2.0 重构** — 对现有电商系统进行全面重构，提升性能和可维护性
2. **CI/CD 流水线搭建** — 建立自动化构建、测试和部署流水线
3. **技术债清理 Sprint** — 集中处理历史遗留的技术债务

## Projects（项目）

共 11 个项目，分布在三个目标之下，涵盖：
- 前端重构（组件化、状态管理升级）
- 后端重构（微服务拆分、API 网关）
- 数据库迁移（ORM 替换、性能优化）
- CI/CD 建设（Jenkins/GitLab CI 配置、容器化）
- 技术债清理（代码规范、测试覆盖率、文档补全）

## Tasks（任务）

共 42 个任务，每个任务包含：
- 所属项目
- 优先级和状态
- 依赖关系
- 预估工时

## Agents（智能体/角色）

5 个预置角色：

| 角色 | 说明 |
|------|------|
| PM | 项目经理，负责目标规划和优先级排序 |
| 前端开发 | 负责前端项目和任务 |
| 后端开发 | 负责后端项目和任务 |
| 测试工程师 | 负责测试相关任务和质量保障 |
| DevOps 工程师 | 负责 CI/CD 流水线和基础设施 |

## 重置演示数据

如需清空当前数据并重新初始化：

```bash
python scripts/init_grever.py --reset
```

如需重新灌入演示数据：

```bash
python scripts/init_grever.py --seed
```
