# P13 争议可视化界面 Mockup 需求文档

---

## 一、页面概述

**页面名称**：智能体 间争议可视化仲裁界面

**使用场景**：
- 多个智能体 对某个决策无法达成共识
- 超时后自动推送给人类管理员仲裁
- 人类管理员在界面中查看争议详情，一键做出仲裁决定

**用户**：人类管理员（仲裁者角色）

**设计目标**：
- 人类只需做选择题，不是问答题
- 5 分钟内理解争议并做出决策
- 信息结构化呈现，减少认知负担

## 三、交互需求

### 3.1 核心交互流程

```
1. 管理员收到通知
   └─> 点击通知进入争议页面
   
2. 管理员浏览页面
   ├─> 查看争议焦点（5 秒）
   ├─> 查看推荐方案（10 秒）
   ├─> 快速扫视对比表格（15 秒）
   ├─> 详细阅读各方论据（30 秒，可选）
   └─> 查看妥协历史（20 秒，可选）
   
3. 管理员做出决定
   ├─> 点击方案按钮（批准/选择其他）
   ├─> 填写备注（可选）
   └─> 提交仲裁决定
   
4. 系统处理
   ├─> 记录决策依据
   ├─> 通知相关智能体
   ├─> 更新任务状态
   └─> 跳转到任务执行页面
```

### 3.2 按钮交互

| 按钮 | 行为 | 后续操作 |
|------|------|---------|
| **✅ 批准推荐方案** | 高亮选中，显示确认对话框 | 确认后提交 |
| **❌ 选择 Playwright** | 高亮选中，显示风险提示 | 确认后提交 |
| **❌ 选择 Scrapy** | 高亮选中，显示风险提示 | 确认后提交 |
| **📝 自定义方案** | 展开文本输入框 | 填写后提交 |
| **⏸️ 暂缓决策** | 展开原因输入框（必填） | 填写后返回任务列表 |
| **提交仲裁决定** | 最终提交按钮 | 记录决策 + 通知智能体 |

### 3.3 状态反馈

| 状态 | 视觉反馈 |
|------|---------|
| 加载中 | 页面顶部显示进度条 |
| 选中方案 | 方案卡片高亮（蓝色边框） |
| 提交成功 | 绿色成功提示 + 自动跳转 |
| 提交失败 | 红色错误提示 + 保留已填内容 |
| 超时警告 | 顶部计时器变红（>30 分钟） |

## 四、视觉设计需求

### 4.1 配色方案

| 元素 | 颜色 | 说明 |
|------|------|------|
| 主背景 | `#F5F7FA` | 浅灰，减少视觉疲劳 |
| 卡片背景 | `#FFFFFF` | 纯白，突出内容 |
| 推荐方案卡片 | `#E8F5E9` | 浅绿，暗示推荐 |
| 主按钮（批准） | `#4CAF50` | 绿色，正面操作 |
| 次按钮（选择其他） | `#2196F3` | 蓝色，中性操作 |
| 警告/风险 | `#FF9800` | 橙色，提醒注意 |
| 错误/反对 | `#F44336` | 红色，负面操作 |
| 文字主色 | `#212121` | 深灰，高对比度 |
| 文字次色 | `#757575` | 中灰，次要信息 |

### 4.2 字体规范

| 元素 | 字体大小 | 字重 | 说明 |
|------|---------|------|------|
| 页面标题 | 24px | Bold | 争议编号 + 状态 |
| 章节标题 | 18px | SemiBold | 争议焦点/各方论据等 |
| 卡片标题 | 16px | Medium | 推荐方案/对比表格等 |
| 正文内容 | 14px | Regular | 论据/约束/时间线等 |
| 按钮文字 | 14px | Medium | 所有操作按钮 |
| 辅助文字 | 12px | Regular | 提示/说明/备注等 |

### 4.3 卡片样式

```css
争议卡片 {
  背景：#FFFFFF;
  边框：1px solid #E0E0E0;
  圆角：8px;
  阴影：0 2px 4px rgba(0,0,0,0.08);
  内边距：20px;
  间距：16px;
}

推荐方案卡片 {
  背景：#E8F5E9;
  边框：2px solid #4CAF50;
  其他同上;
}
```

### 4.4 响应式设计

| 屏幕宽度 | 布局 |
|---------|------|
| ≥1440px | 三栏布局（各方论据并排） |
| 1024px-1439px | 两栏布局（各方论据并排） |
| 768px-1023px | 单栏布局（各方论据上下排列） |
| <768px | 移动端优化（折叠次要信息） |

## 五、内容模块详细说明

### 5.1 争议焦点模块

**目的**：让管理员在 5 秒内理解争议核心

**内容**：
- 争议主题（一句话）
- 影响范围（哪个系统/模块）
- 紧急程度（低/中/高 + 颜色标识）
- 等待时长（从争议产生开始计时）

**视觉**：
- 红色边框（高紧急）/橙色边框（中紧急）/蓝色边框（低紧急）
- 置顶显示，固定高度

---

### 5.2 推荐方案模块

**目的**：突出显示智能体 集体推荐的方案

**内容**：
- 方案名称
- 方案描述（2-3 句话）
- 关键指标（成本/收益/风险）
- 批准按钮（主操作）

**视觉**：
- 浅绿色背景
- 绿色边框
- 放大显示（比其他卡片高 20%）
- 批准按钮使用实心绿色

---

### 5.3 对比表格模块

**目的**：快速对比各方案优劣

**内容**：
- 对比维度（5-6 个关键指标）
- 各方案评分（星级或分数）
- 推荐方案列高亮

**视觉**：
- 表格形式
- 推荐方案列浅绿色背景
- 星级评分用颜色区分（绿/黄/红）

---

### 5.4 各方论据模块

**目的**：展示各方观点，帮助管理员理解分歧原因

**内容**：
- 智能体 名称 + 头像/图标
- 核心论据（3-5 条）
- 前提约束（2-3 条）
- 适用场景（2-3 条）

**视觉**：
- 两栏并排（或单栏上下）
- 智能体 名称用不同颜色区分
- 论据用项目符号列表

---

### 5.5 妥协历史模块

**目的**：展示协商过程，证明已尽力尝试达成共识

**内容**：
- 时间线形式
- 每条记录包含：
  - 时间戳
  - 提案方
  - 提案内容
  - 反对者及原因

**视觉**：
- 垂直时间线
- 左侧时间，右侧内容
- 反对意见用缩进 + 不同颜色

---

### 5.6 相关资源模块

**目的**：提供上下文信息，辅助决策

**内容**：
- 任务上下文链接
- 智能体 能力报告链接
- 历史决策记录
- 影响评估（任务依赖图）

**视觉**：
- 链接列表形式
- 图标 + 文字
- 可点击跳转

---

### 5.7 仲裁决定模块

**目的**：收集管理员决策

**内容**：
- 方案选择按钮（4-5 个选项）
- 备注输入框（可选）
- 提交按钮

**视觉**：
- 固定在页面底部
- 背景色区分（浅灰色）
- 提交按钮使用主色调
