## 一、日志引擎（Log Engine）

### 调研结论

**Tavily 搜索验证：Loguru 是 2025 年 Python 日志的首选方案。**

| 组件 | 类型 | 验证结果 | 推荐度 |
|------|------|---------|--------|
| **Loguru** | Python 库 | 零配置，5行代码接入，异步写入，自动轮转 | ⭐⭐⭐⭐⭐ |
| **structlog** | Python 库 | 结构化日志，与标准logging兼容，OTel集成 | ⭐⭐⭐⭐ |
| **OpenTelemetry** | 标准+SDK | 日志/追踪/指标三合一，W3C trace context | ⭐⭐⭐⭐ |
| **Fluent Bit** | 独立服务 | 10MB内存占用，轻量日志采集转发 | ⭐⭐⭐⭐ |
| **Loki** | 独立服务 | Grafana原生，比ES轻量10倍 | ⭐⭐⭐⭐ |
| **ELK Stack** | 独立服务 | 企业级，太重 | ⭐⭐ |

**Tavily 来源**：
- Real Python 教程：《How to Use Loguru for Simpler Python Logging》
- Better Stack 指南：《A Complete Guide to Logging in Python with Loguru》
- OpenTelemetry 2025：《How to Structure Logs Properly in Python with OpenTelemetry》

**推荐方案**：

| 阶段 | 组件 | 说明 |
|------|------|------|
| **Phase 1** | Loguru | 零配置，5行代码接入，异步写入 |
| **Phase 3+** | Fluent Bit → Loki | 生产升级，Grafana集成 |

**Loguru 示例**：

```python
from loguru import logger

logger.add(
    "logs/nexus_{time:YYYY-MM-DD}.log",
    rotation="50 MB",
    retention="30 days",
    format="{time} | {level} | {name} | {message} | {extra[trace_id]}"
)

logger.bind(trace_id="abc-123").info("Task assigned", task_id="task-001")
```

**结论**：❌ 不自研。Loguru 是最活跃的Python日志库，2025年仍在维护，生态完善。

## 二、DAG 流程定义模块

### 调研结论

**Tavily 验证：NetworkX 是 Python 图论算法的事实标准，文档完善。**

| 组件 | 类型 | 验证结果 | 推荐度 |
|------|------|---------|--------|
| **NetworkX** | Python库 | 17k+ stars，拓扑排序/环路检测/并行组 | ⭐⭐⭐⭐⭐ |
| **Prefect** | Python库+平台 | 30k+ stars，现代化DAG编排，API友好 | ⭐⭐⭐ |
| **Dagster** | Python库+平台 | 数据工程导向，学习曲线陡 | ⭐⭐⭐ |
| **Airflow** | 独立平台 | 太重，面向ETL | ⭐⭐ |
| **Temporal** | 独立工作流引擎 | 可靠执行，但重 | ⭐⭐⭐ |

**Tavily 来源**：
- NetworkX 官方文档：《Directed Acyclic Graphs & Topological Sort》
- NetworkX 3.6 文档：《Cycle Detection》
- Stack Overflow：《How to detect a cycle in a directed graph with Python?》

**不推荐 Airflow/Prefect/Temporal 的原因**：
- 需要独立服务或额外运行时
- 面向数据工程/ETL，不是面向 Agent 任务
- 学习成本高，团队需要额外维护

**推荐方案**：

| 组件 | 用途 | 说明 |
|------|------|------|
| **NetworkX** | 图论算法 | `nx.topological_sort()`、`nx.find_cycle()`、`nx.topological_generations()` |
| **Pydantic** | 数据模型 | WorkflowDefinition、NodeDefinition、EdgeDefinition |
| **自研前端** | DAG 可视化编辑 | 已有 `workflow_edit*.py`，继续优化 |

**NetworkX 示例**：

```python
import networkx as nx

G = nx.DiGraph()
G.add_edge("task-001", "task-002")
G.add_edge("task-001", "task-003")

# 检测环路
try:
    cycle = nx.find_cycle(G)
except nx.NetworkXNoCycle:
    pass  # DAG 合法

# 拓扑排序 + 并行组
order = list(nx.topological_sort(G))
parallel_groups = list(nx.topological_generations(G))
```

**结论**：⚠️ 半自研。NetworkX 图论算法成熟稳定，不需要重复实现。WorkflowDefinition 数据模型和前端编辑器自研。

## 三、流程执行引擎

### 调研结论

**Tavily 验证：transitions 是 Python 状态机的首选库，文档完善。**

| 组件 | 类型 | 验证结果 | 推荐度 |
|------|------|---------|--------|
| **transitions** | Python库 | 4.8k stars，轻量，异步支持，回调机制 | ⭐⭐⭐⭐⭐ |
| **Celery** | Python分布式任务队列 | 60k+ stars，成熟，需要Redis/RabbitMQ | ⭐⭐⭐ |
| **Dramatiq** | Python任务队列 | 比Celery轻量，Redis后端 | ⭐⭐⭐⭐ |
| **Temporal** | 独立工作流引擎 | 可靠执行+重试+状态持久化，但重 | ⭐⭐⭐ |
| **Python state_machine** | 手写 | 已有11KB，需要重构 | ⭐ |

**Tavily 来源**：
- GitHub: `pytransitions/transitions`（4.8k stars）
- Python State Machine 2.3k stars

**推荐方案**：

| 阶段 | 组件 | 说明 |
|------|------|------|
| **Phase 1** | transitions | 替代手写 `state_machine.py` (11KB) |
| **Phase 3+** | Dramatiq | 需要分布式执行时引入，Redis后端 |

**transitions 示例**：

```python
from transitions import Machine

class Task:
    states = ['todo', 'in_progress', 'verifying', 'review_needed',
              'disputed', 'done', 'failed', 'timeout', 'paused']
    transitions = [
        {'trigger': 'assign', 'source': 'todo', 'dest': 'in_progress'},
        {'trigger': 'complete', 'source': 'in_progress', 'dest': 'verifying'},
        {'trigger': 'pass', 'source': 'verifying', 'dest': 'done'},
        {'trigger': 'fail', 'source': 'verifying', 'dest': 'review_needed'},
        {'trigger': 'dispute', 'source': 'review_needed', 'dest': 'disputed'},
        {'trigger': 'rule_done', 'source': 'disputed', 'dest': 'done'},
    ]

    def __init__(self):
        self.machine = Machine(model=self, states=Task.states,
                              transitions=Task.transitions, initial='todo')
```

**结论**：⚠️ 半自研。状态机用 transitions 库（已有 11KB 手写代码需要替换），调度核心和验证引擎继续自研。

**Tavily 来源**：

**推荐方案**：

| 组件 | 用途 | 说明 |

**AdapterRegistry 设计**：

```python
class AgentAdapter(ABC):
    @abstractmethod
    async def send_task(self, agent_id: str, task: Task) -> str: ...
    @abstractmethod
    async def get_result(self, agent_id: str, execution_id: str) -> TaskResult: ...
    @abstractmethod
    async def heartbeat(self, agent_id: str) -> AgentStatus: ...

class OpenClawAdapter(AgentAdapter): ...   # 当前
class LangChainAdapter(AgentAdapter): ...  # 未来
class MCPAdapter(AgentAdapter): ...        # 未来

class AdapterRegistry:
    def register(self, platform: str, adapter: AgentAdapter): ...
    def get(self, agent_id: str) -> AgentAdapter: ...
```

**结论**：✅ 完全自研。无现成方案。AdapterRegistry 自研，LangChain/CrewAI/AutoGen 作为适配器实现参考。

| 组件 | 类型 | 验证结果 | 推荐度 |
|------|------|---------|--------|
| **scikit-learn** | Python ML库 | 60k+ stars，余弦相似度、TF-IDF | ⭐⭐⭐⭐⭐ |
| **FAISS** | 向量检索库 | Meta出品，30k+ stars，极速大规模检索 | ⭐⭐⭐⭐⭐ |
| **Whoosh** | Python全文检索 | 轻量，适合小规模 | ⭐⭐⭐ |
| **Elasticsearch** | 独立服务 | 重量级，大规模 | ⭐⭐ |

**Tavily 来源**：
- Pinecone 教程：《Introduction to Facebook AI Similarity Search (Faiss)》
- SparkCO AI：《Mastering Faiss: Deep Dive into Vector Similarity Search》

**scikit-learn 匹配增强示例**：

```python
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

# Agent 能力标签 → 向量
agent_vectors = np.array([
    tag_to_vector(agent_tags[agent_id]) for agent_id in online_agents
])

# 任务需求 → 向量
task_vector = np.array([tag_to_vector(task_capability_tags)])

# 余弦相似度

# 综合打分
scores = similarities * 0.5 + load_scores * 0.3 + reliability_scores * 0.2
```

**FAISS 适用场景**：

当 Agent 数量 > 100 时，线性扫描变慢：

```python
import faiss

index = faiss.IndexFlatIP(dimension)  # 内积索引
index.add(agent_vectors)  # Agent 向量入库

```

**结论**：✅ 核心自研 + scikit-learn/FAISS 辅助。匹配策略（四维标签+权重衰减）是 Nexus 差异化，不能替换。

## 六、消息总线

### 调研结论

**Tavily 验证：Redis Pub/Sub 是 Python 异步消息总线的事实标准，文档完善。**

| 组件 | 类型 | 验证结果 | 推荐度 |
|------|------|---------|--------|
| **Redis Pub/Sub** | 内存消息队列 | 轻量，5分钟部署，Python asyncio支持 | ⭐⭐⭐⭐⭐ |
| **RabbitMQ** | 独立消息服务 | 企业级，可靠投递，但重 | ⭐⭐⭐⭐ |
| **Kafka** | 分布式消息平台 | 超大规模，太重 | ⭐⭐ |
| **ZeroMQ** | 网络库 | 轻量，需要自己封装 | ⭐⭐⭐ |
| **自研 Event Bus** | Python | 已有，需要升级为双总线 | ⭐⭐⭐ |

**Tavily 来源**：
- OneUptime：《How to Use Redis Pub/Sub for Real-Time Messaging》
- OneUptime：《How to Use Async Redis in Python with redis.asyncio》
- Sakti Dwicahyono：《Async and Sync Python Pub/Sub with Redis》

**推荐方案**：

| 组件 | 用途 | 说明 |
|------|------|------|
| **自研 Command Bus** | 命令分发 | 纯代码封装，基于函数注册表 |
| **Redis Pub/Sub** | Event 广播 | `redis.asyncio`，异步发布订阅 |

**Command Bus 示例（不需要额外服务）**：

```python
class CommandBus:
    def __init__(self):
        self._handlers: Dict[str, Callable] = {}

    def register(self, command_type: str, handler: Callable):
        self._handlers[command_type] = handler

    async def dispatch(self, command: Command) -> Result:
        handler = self._handlers.get(command.type)
        if not handler:
            raise ValueError(f"No handler for {command.type}")
        return await handler(command)
```

**Event Bus 示例（基于 Redis Pub/Sub）**：

```python
import redis.asyncio as redis

class EventBus:
    def __init__(self, redis_url: str = "redis://localhost:6379"):
        self.redis = redis.from_url(redis_url)
        self._subscribers: Dict[str, list] = {}

    async def publish(self, event: Event):
        await self.redis.publish(event.type, json.dumps(event.to_dict()))

    async def subscribe(self, event_type: str, handler: Callable):
        self._subscribers.setdefault(event_type, []).append(handler)
        pubsub = self.redis.pubsub()
        await pubsub.subscribe(event_type)
        asyncio.create_task(self._listen(pubsub, handler))
```

**结论**：⚠️ 半自研。Command Bus 纯代码封装（不需要额外服务），Event Bus 基于 Redis Pub/Sub（Redis 是通用组件，不是专门开发的）。
