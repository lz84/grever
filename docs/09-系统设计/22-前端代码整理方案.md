# 前端代码整理方案

## 目标

对齐后端五兄弟架构，将前端代码按域分类组织，页面保留在 `pages/`，业务组件/逻辑按域归位。

## 目录结构

```
packages/ui/src/
├── pages/              路由页面入口（按域 import 业务组件）
├── layout/             布局（MainLayout, Sidebar, Header）
├── reins/              驾驭域业务代码
│   ├── components/     GoalCard, TaskTable, ProjectTree, AgentList...
│   ├── hooks/          useGoal, useTask, useProject...
│   └── services/       域专用 API 封装
├── grasp/              认知域业务代码
│   ├── components/     CognitiveCard, KnowledgeGraph, InjectionPanel...
│   └── hooks/          useCognition, useKnowledge...
├── evo/                进化域业务代码
│   ├── components/     VerificationCard, EvolutionPanel...
│   └── hooks/          useVerification...
├── reach/              拓展域业务代码
│   ├── components/     ScenarioCard, IndustryTagPanel, McpConfig...
│   └── hooks/          useScenario, useIndustryTag...
├── vigil/              安全域业务代码
│   ├── components/     SecurityAlert, TrustScore...
│   └── hooks/          useSecurity...
└── shared/             共享资源
    ├── components/     Button, Modal, Table, Pagination（通用 UI）
    ├── hooks/          useAuth, useFetch, useDebounce
    ├── utils/          format, validation, constants
    ├── api/            paths.ts + api.ts（API 层）
    └── types/          全局类型（Goal, Task, Agent...）
```

## 整理原则

1. **页面统一在 `pages/`**：路由直接映射 `pages/GoalList.tsx` → `/goals`
2. **业务组件按域归位**：被页面引用但属于特定域的业务组件，放到对应域目录
3. **通用组件进 `shared/`**：被 2 个以上域引用的组件
4. **API 路径唯一入口**：`shared/api/paths.ts`，禁止硬编码
5. **不再用 `${API_V1}`**：`request()` 已自动加 `/api/v1` 前缀
6. **禁止跨域相对路径**：不用 `../../../components/`，用 `@/reins/components/`
