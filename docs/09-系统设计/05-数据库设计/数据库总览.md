## 2. ER 关系描述

### 2.1 Goal / Project / Task 三层结构

```
goals
  │
  ├── 1:N ──── projects
  │              │
  │              └── 1:N ──── tasks
  │                            │
  │                            └── N:N ──── task_dependencies
  │                                         （同一层级，task ↔ task）
  │
  └── 1:N ──── scenarios（Goal 可关联场景）
                   │
                   └── 1:N ──── scenario_task_templates
                                    │
                                    └── N:1 ──── tasks（通过模板实例化）

projects
  │
  ├── N:1 ──── goals
  ├── N:1 ──── workflows（通过 workflow_id）
  └── 1:N ──── tasks

tasks
  │
  ├── N:1 ──── projects
  ├── N:1 ──── goals
  ├── N:1 ──── workflow_steps（通过 workflow_step_id）
  ├── N:1 ──── agents（通过 assigned_agent）
  ├── N:1 ──── agents（通过 verifier_agent_id）
  └── N:N ──── tasks（通过 task_dependencies 自关联）
```

**关系说明**：

- **Goal → Project**：一个 Goal 包含多个 Project，Project 通过 `goal_id` 外键指向 Goal
- **Project → Task**：一个 Project 包含多个 Task，Task 通过 `project_id` 外键指向 Project
- **Task → Task（自关联）**：通过 `task_dependencies` 多对多关系表实现 DAG 依赖，`depends_on` 和 `next_step` 为冗余的 JSON 列
- **Goal → Scenario**：通过 `scenario_id` 字段关联（可选），Goal 可从一个 Scenario 实例化而来
- **Scenario → Task**：Scenario 通过 `scenario_task_templates` 定义任务模板，实例化时生成具体 Task

### 2.2 Agent 相关关系

```
agents
  │
  ├── 1:N ──── tasks（作为 assigned_agent）
  │
  ├── 1:N ──── tasks（作为 verifier_agent_id）
  │
  ├── 1:N ──── execution_logs（Agent 执行记录）
  │
  └── N:N ──── industry_capability_tags
                （通过 agents.capability_tags JSON 列存储）
                或 agents 的 tags 列（如有）
```

**Agent 能力标签**：
- Agent 的能力通过 `capability_tags` JSON 列存储（四维标签）
- 不直接依赖 `industry_capability_tags` 表
- 通过标签匹配实现 Task-Agent 分配

### 2.3 场景库（Scenario）关系

```
scenarios
  │
  ├── 1:N ──── scenario_task_templates
  │              │
  │              └── N:1 ──── industry_capability_tags（required_capabilities）
  │
  ├── N:N ──── skill_tags（通过 scenario_skills）
  │              │
  │              └── N:1 ──── skills
  │
  └── 1:N ──── solutions（探索模式方案）
                   │
                   └── N:1 ──── goals

industry_packs
  │
  ├── 1:N ──── industry_capability_tags
  │
  ├── 1:N ──── pack_customizations（定制变更记录）
  │
  └── 1:N ──── pack_customization_releases（发布记录）
```

### 2.4 验证与争议关系

```
tasks
  │
  ├── N:1 ──── agents（verifier_agent_id）
  │
  └── 1:N ──── human_input_requests（等待人工输入）

human_input_requests
  │
  └── N:1 ──── tasks

disputes
  │
  └── N:1 ──── tasks（related_task_id）
```

## 3. 关键表结构

### 3.1 tasks（任务表）

任务表是 Nexus 系统的核心，存储所有任务的详细信息：

```sql
CREATE TABLE tasks (
    id                      TEXT PRIMARY KEY,       -- 任务 ID（UUID）
    title                   TEXT NOT NULL,          -- 任务标题
    description             TEXT,                   -- 任务描述
    project_id             TEXT,                   -- 所属项目 ID
    goal_id                TEXT,                   -- 所属目标 ID（可推导）
    assigned_agent         TEXT,                   -- 分配的 Agent ID

    status                 TEXT NOT NULL DEFAULT 'todo',  -- 任务状态
    priority               TEXT DEFAULT 'medium',  -- 优先级 high/medium/low
    category               TEXT,                   -- 任务类型（废弃，用 capability_tags）

    -- 能力标签（四维 JSON，替代 category）
    capability_tags        TEXT DEFAULT '{}',       -- JSON: {business:[], professional:[], technical:[], management:[]}

    -- DAG 依赖关系
    depends_on             TEXT,                   -- JSON array: ["task-xxx", ...] 我依赖谁
    next_step              TEXT DEFAULT '[]',      -- JSON array: ["task-yyy", ...] 谁依赖我

    -- 生命周期时间戳
    created_at             DATETIME NOT NULL,       -- 创建时间（ISO 格式字符串）
    updated_at             DATETIME NOT NULL,        -- 更新时间
    started_at             DATETIME,                 -- 开始执行时间
    completed_at           DATETIME,                 -- 完成时间

    -- 重试机制
    retry_count            INTEGER DEFAULT 0,       -- 已重试次数
    max_retries            INTEGER DEFAULT 3,       -- 最大重试次数

    -- 执行结果
    result                 TEXT,                   -- 执行结果摘要
    result_summary         TEXT,                   -- 结果详细摘要
    error_type             TEXT,                   -- 错误类型
    error_message          TEXT,                   -- 错误信息
    blocked_reason         TEXT,                   -- 阻塞原因

    -- 暂停信息（Sprint 82）
    paused_reason          TEXT,                   -- 暂停原因：human/orphan_on_restart/orphan_on_offline/orphan_on_timeout

    -- 验收机制（Sprint 50-66）
    acceptance_criteria    TEXT,                   -- 验收标准（JSON）
    needs_verification     INTEGER DEFAULT 1,       -- 是否需要验证（布尔）
    verification_cycle     INTEGER DEFAULT 0,      -- 验证循环次数
    verifier_agent_id       TEXT,                   -- 验证 Agent ID

    -- 上下文文档（Sprint 86）
    context_md             TEXT,                   -- 三级上下文文档（执行摘要）
    workspace_path         TEXT,                   -- 工作区路径（从 Project 继承）

    -- 工作流关联
    workflow_step_id       TEXT,                   -- 关联的工作流步骤 ID

    -- 验证裁决（Sprint 54）
    ruling_comment_id      TEXT,                   -- 裁决评论 ID
    instruction_comment_id TEXT,                   -- 指令评论 ID
    ruling_instruction     TEXT,                   -- 裁决指令

    -- 交付标准（Sprint 86）
    delivery_criteria      TEXT,                   -- 交付标准
    doc_refs               TEXT,                   -- 文档引用（JSON array）

    -- 父子任务
    parent_id              TEXT,                   -- 父任务 ID（自关联）
    executor_type          TEXT DEFAULT 'ai',       -- 执行者类型：ai/human/hybrid
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (parent_id) REFERENCES tasks(id)
);

CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_project ON tasks(project_id);
CREATE INDEX idx_tasks_goal ON tasks(goal_id);
CREATE INDEX idx_tasks_agent ON tasks(assigned_agent);
CREATE INDEX idx_tasks_priority ON tasks(priority);
```

### 3.2 task_dependencies（任务依赖关系表）

实现 Task DAG 依赖关系的多对多关联表：

```sql
CREATE TABLE task_dependencies (
    task_id         TEXT NOT NULL,  -- 当前任务 ID
    dependency_id   TEXT NOT NULL,  -- 依赖的任务 ID
    PRIMARY KEY (task_id, dependency_id),
    FOREIGN KEY (task_id) REFERENCES tasks(id),
    FOREIGN KEY (dependency_id) REFERENCES tasks(id)
);

CREATE INDEX idx_td_dependency ON task_dependencies(dependency_id);
```

**使用说明**：
- `task_id` 依赖 `dependency_id`
- 查询"哪些任务依赖任务 X"：`SELECT task_id FROM task_dependencies WHERE dependency_id = 'task-X'`
- 查询"任务 X 依赖哪些"：`SELECT dependency_id FROM task_dependencies WHERE task_id = 'task-X'`

### 3.3 projects（项目表）

Project 是 Goal 的子单元，组织一组相关任务：

```sql
CREATE TABLE projects (
    id                  TEXT PRIMARY KEY,       -- 项目 ID
    name                TEXT NOT NULL,          -- 项目名称
    description         TEXT,                   -- 项目描述
    goal_id            TEXT,                   -- 所属目标 ID
    status             TEXT NOT NULL DEFAULT 'active',  -- 状态：created/active/paused/done/failed
    priority            INTEGER DEFAULT 1,       -- 优先级
    members            TEXT,                    -- 成员列表（JSON）
    task_ids           TEXT,                    -- 任务 ID 列表（JSON）

    -- 工作流关联
    parent_workflow_id TEXT,                    -- 父工作流 ID
    workflow_id        TEXT,                    -- 关联的工作流 ID
    phase_order        INTEGER NOT NULL DEFAULT 0,  -- 阶段顺序

    -- 能力标签（Sprint 95：从 Project 继承到 Task）
    capability_tags    TEXT,                    -- JSON: 四维标签

    -- 生命周期时间戳
    created_at         INTEGER NOT NULL,
    updated_at         INTEGER NOT NULL,
    completed_at        INTEGER,

    -- 验证
    verifier_agent_id   TEXT,                    -- 验证 Agent ID

    -- 约束条件（Sprint 73）
    constraints         TEXT,                   -- JSON: 约束条件

    FOREIGN KEY (goal_id) REFERENCES goals(id)
);

CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_goal ON projects(goal_id);
CREATE INDEX idx_projects_parent_workflow ON projects(parent_workflow_id);
CREATE INDEX idx_projects_workflow ON projects(workflow_id);
```

### 3.4 goals（目标表）

Goal 是顶层执行单元，代表用户的一个高层目标：

```sql
CREATE TABLE goals (
    id                  TEXT PRIMARY KEY,       -- 目标 ID
    title               TEXT NOT NULL,          -- 目标标题
    description        TEXT,                   -- 目标描述
    parent_id          TEXT,                   -- 父目标 ID（可选，支持嵌套目标）
    status             TEXT NOT NULL DEFAULT 'draft',  -- 状态：draft/in_progress/paused/completed/failed

    -- 进度
    progress            REAL NOT NULL DEFAULT 0.0,  -- 进度 0.0-1.0

    -- 子目标/子任务
    task_ids           TEXT,                    -- 子任务 ID 列表（JSON）

    -- 生命周期时间戳
    created_at         INTEGER NOT NULL,
    updated_at         INTEGER NOT NULL,
    completed_at        INTEGER,

    -- 验证
    verifier_agent_id   TEXT,                    -- 验证 Agent ID

    -- 探索模式（Sprint 81）
    mode                TEXT,                    -- normal/exploration/iteration
    scenario_id         TEXT,                    -- 关联的场景模板 ID
    exploration_config  TEXT,                    -- 探索配置（JSON）

    -- 约束条件（Sprint 73）
    constraints         TEXT,                    -- JSON: 约束条件

    FOREIGN KEY (parent_id) REFERENCES goals(id)
);

CREATE INDEX idx_goals_status ON goals(status);
CREATE INDEX idx_goals_parent ON goals(parent_id);
```

### 3.5 agents（智能体注册表）

Agent 是实际执行任务的智能体：

```sql
CREATE TABLE agents (
    id                      TEXT PRIMARY KEY,       -- Agent ID
    name                    TEXT NOT NULL,          -- Agent 名称
    capabilities            TEXT,                    -- 能力列表（JSON，已废弃，用 capability_tags）
    status                  TEXT NOT NULL DEFAULT 'offline',  -- 状态：online/stale/offline

    -- 连接信息
    address                 TEXT,                    -- Agent 服务地址（如 http://localhost:18791）
    trigger_mode            TEXT DEFAULT 'sse',     -- 触发模式：sse/polling
    poll_interval_seconds   INTEGER DEFAULT 10,     -- 轮询间隔（秒，仅 polling 模式）

    -- 能力标签（四维，Sprint 95）
    capability_tags         TEXT,                    -- JSON: {business:[], professional:[], technical:[], management:[]}

    -- 模型配置
    model_name              TEXT,                    -- 使用的模型名称

    -- 负载与任务
    load                    INTEGER NOT NULL DEFAULT 0,   -- 负载百分比 0-100
    current_tasks           INTEGER NOT NULL DEFAULT 0,   -- 当前任务数

    -- 健康状态
    last_heartbeat          INTEGER NOT NULL,        -- 最后心跳时间（Unix 时间戳）
    health_status           TEXT DEFAULT 'online',   -- 健康状态：online/stale/offline
    consecutive_offline_count INTEGER DEFAULT 0,      -- 连续离线次数

    -- 元数据
    metadata                TEXT,                    -- 元数据（JSON）
    registered_at           INTEGER NOT NULL,        -- 注册时间
    updated_at              DATETIME NOT NULL,        -- 更新时间
    created_at              INTEGER NOT NULL,        -- 创建时间

    FOREIGN KEY (id) REFERENCES agents(id)

CREATE INDEX idx_agents_status ON agents(status);

```

## 4. 索引与性能

### 4.1 关键索引

| 表 | 索引 | 用途 |
|----|------|------|
| tasks | `idx_tasks_status` | 调度器查询待分配/执行中任务 |
| tasks | `idx_tasks_agent` | 查询某 Agent 的所有任务 |
| tasks | `idx_tasks_project` | 查询某 Project 的所有任务 |
| tasks | `idx_tasks_goal` | 查询某 Goal 的所有任务 |
| task_dependencies | `idx_td_dependency` | 依赖解锁查询 |
| projects | `idx_projects_goal` | 查询某 Goal 的所有 Project |
| goals | `idx_goals_status` | 调度器查询活动 Goal |

### 4.2 调度器常用查询

```sql
-- 查询待分配任务（最常用）
SELECT * FROM tasks
WHERE status IN ('todo', 'pending')
  AND assigned_agent IS NULL
  AND paused_reason IS NULL
ORDER BY priority DESC, created_at ASC
LIMIT 20;

-- 查询 in_progress 任务（超时检测）
SELECT * FROM tasks
WHERE status = 'in_progress'
  AND started_at < :threshold
ORDER BY started_at ASC;

-- 查询 waiting 任务（依赖解锁）
SELECT t.* FROM tasks t
JOIN task_dependencies td ON t.id = td.task_id
WHERE t.status = 'waiting'
GROUP BY t.id
HAVING COUNT(*) = SUM(CASE WHEN
    (SELECT status FROM tasks WHERE id = td.dependency_id) IN ('done', 'completed')
    THEN 1 ELSE 0 END);

-- 查询 Agent 的 in_progress 任务
SELECT * FROM tasks
WHERE assigned_agent = :agent_id
  AND status = 'in_progress';
```
