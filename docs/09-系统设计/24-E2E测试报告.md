## 测试局限性

✅ **已验证（真实 Chromium 渲染）**：

- 页面能加载（HTTP 200）
- 内容长度 > 1000 字符
- 不含 404 / Internal Server Error
- 基本 UI 元素存在（按钮、输入框）
- 22 个关键页面截图均渲染成功（34KB~179KB）

❌ **未验证**：

- 表单实际提交和数据入库（需要 API 联调）
- 删除/修改等写操作
- 完整的 CRUD 闭环流程
- API 响应数据的正确性

### 建议补充的 E2E 测试

1. **端到端 CRUD**：创建 Goal → 在列表出现 → 查看详情 → 分解 → 创建 Project → 创建 Task → 执行 → 完成
2. **HITL 流程**：提交 human input → 审批 → 验证结果
3. **场景实例化**：创建场景 → 填充参数 → 实例化 → 验证 Task 生成
4. **Agent 注册**：注册新 Agent → 验证出现在列表 → 验证心跳

## 运行方法

```bash
cd packages/ui

# 启动前端（端口 5173）
npm run dev

# 运行所有 E2E 测试
npx playwright test tests/e2e/ --reporter=list --project=chromium

# 只跑浅层测试
npx playwright test tests/e2e/page-flows.spec.ts

# 只跑深层测试
npx playwright test tests/e2e/page-flows-deep.spec.ts

# 只跑某个流程
npx playwright test tests/e2e/ --grep "F2"

# 带截图报告
npx playwright test tests/e2e/ --reporter=html
```
