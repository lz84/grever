## 执行步骤

### Step 1: 创建目录 + 迁移公共服务
```bash
# 创建 shared/
mkdir src/shared
mv src/database src/shared/database
mv src/eventbus src/shared/eventbus
mv src/auth src/shared/auth
mv src/common src/shared/common
mv src/exception src/shared/exception

# 全局替换 import
# from database → from shared.database
# from eventbus → from shared.eventbus
# from auth → from shared.auth
# from common → from shared.common
# from exception → from shared.exception
```

### Step 2: 创建 5 大领域 + 迁移业务代码
```bash
# cognitive
mkdir -p src/cognitive/{common,parser,registry,analysis,injection,api}
mv src/grasp/parsers src/cognitive/parser/
mv src/grasp/service.py src/grasp/base.py src/grasp/models.py src/cognitive/common/
mv src/grasp/registry.py src/cognitive/registry/
mv src/grasp/analysis.py src/grasp/cognitive_*.py src/cognitive/analysis/
# API files: grasp_*.py, solutions_*.py, inject.py, knowledge_injector*.py → cognitive/api/

# steering
mkdir -p src/steering/{common,core,scheduler,tracking,messaging,nexus_log,api}
mv reins/scheduler src/steering/scheduler/
mv reins/tracking src/steering/tracking/
mv reins/tracker.py reins/tracker_sync.py src/steering/tracking/
mv reins/messaging src/steering/messaging/
mv reins/nexus_log src/steering/nexus_log/
# common
mv reins/config.py src/steering/common/
mv reins/logging src/steering/common/logging/
mv reins/grasp src/steering/common/grasp_client/
mv reins/database.py src/steering/common/database.py
mv reins/event_bus.py reins/events.py reins/eventbus_integration.py src/steering/common/
# core
mv reins/engine.py reins/workflow_engine.py reins/assignment.py reins/background_tasks.py reins/recovery.py reins/context_injector.py reins/state_machine.py src/steering/core/
# API files → steering/api/

# evolution
mkdir -p src/evolution/{common,mutation,distillation,a2a,weight,api}
mv reins/correction_engine.py src/evolution/common/
mv reins/evo/mutation.py reins/evo/analyzer.py src/evolution/mutation/
mv reins/evo/distiller.py reins/evo/solidify.py src/evolution/distillation/
mv reins/evo/a2a.py src/evolution/a2a/
mv reins/evo/weight_updater.py src/evolution/weight/
# API: dispute_*.py → evolution/api/

# extension
mkdir -p src/extension/{common,scenarios,industry,mcp,artifacts,skills,capabilities,attachments,search}
mv reins/scenarios src/extension/scenarios/
mv reins/industry src/extension/industry/
mv reins/agent_sdk.py src/extension/common/
# API files → 各自子目录

# security
mkdir -p src/security/{common,trust,access,alerts,api}
mv reins/vigil/trust.py src/security/trust/
mv reins/vigil/access.py src/security/access/
mv reins/vigil/alerts.py src/security/alerts/
mv reins/audit.py reins/sanitize.py src/security/common/
# API: security*.py → security/api/
```

### Step 3: 迁移跨领域基础设施

```bash
mv reins/models src/models
mv reins/persistence src/persistence
mv reins/services src/services
mv reins/migration src/migration
# 合并 data
mv src/data/* src/reins/data/
mv reins/data src/data
```

### Step 6: 全局 import 替换
- `from reins.api.xxx` → `from {domain}.api.xxx`
- `from reins.xxx` → `from {domain}.xxx`
- `from reins.scheduler` → `from steering.scheduler`
- `from reins.models` → `from models`
- `from reins.persistence` → `from persistence`
- `from reins.services` → `from services`

### Step 7: 清理
- 删除空 reins/ 目录
- 删除空目录（src/memory, src/data, reins/data, reins/state）
- 移动散落测试文件到 src/tests/
- 删除一次性脚本
