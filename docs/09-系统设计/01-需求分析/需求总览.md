## 系统愿景

> **Nexus 是多 AI Agent 协同管理平台，让 AI 像团队一样工作，人在关键节点把关。**

Nexus 解决的核心问题是：从"有个目标"到"活干上了"之间的时间成本。通过将业务流程模板化（场景）、AI 能力标准化（能力标签）、执行过程可控化（HITL），实现 AI 协同工作的全链路管理。

## 系统目标

| 编号 | 目标 | 衡量指标 | 优先级 |
|------|------|---------|--------|
| M-01 | 实现多 Agent 自动协同调度 | 场景实例化 → 任务派发 → 执行完成，全流程无需人工干预 | P0 |
| M-02 | 内置 HITL（人在回路）机制 | 每个 Task 可配置 `condition_type`，关键节点自动暂停等人审批/输入 | P0 |
| M-03 | 支持行业包（场景+技能+知识+Agent方案）交付 | 标准行业包可安装、可运行、可定制 | P1 |
| M-04 | 提供能力标签匹配体系 | Agent 与 Task 通过四维标签（business/professional/technical/management）自动匹配 | P1 |
| M-05 | 支持行业包定制工具 | 将行业包定制从手艺活变成生产线，降低定制成本 | P2 |

## 范围定义

### 做什么

| 功能域 | 说明 |
|--------|------|
| **任务调度** | Goal → Project → Task 三级分解，DAG 拓扑排序，DependencyResolver 依赖解析与解锁 |
| **场景库** | 场景模板管理、场景实例化、场景浏览与安装、场景执行监控 |
| **智能体管理** | Agent Profile 注册、能力标签声明、执行状态跟踪 |
| **能力标签** | 四维标签体系（business/professional/technical/management）+ 行业能力标签库，支持标签生命周期管理 |
| **人机协同（HITL）** | `human_decision`（审批）、`human_input`（数据输入）、`auto_eval`（自动评估）、`none`（无需干预）四种模式 |
| **验证仲裁** | Task 执行结果验收、异常处理、仲裁裁决流程 |

### 不做什么

| 不做什么 | 原因 |
|---------|------|
| 不做 AI 模型训练/微调 | Nexus 是 AI 管理者，不生产 AI |
| 不做通用的聊天式对话界面 | 不是聊天机器人，是任务协同平台 |
| 不做企业级 SSO/多租户/审计（开源版） | 属于企业版功能，Phase 2 实现 |

## 核心功能域

### 任务调度

Nexus 的任务调度基于 Goal（战略目标）→ Project（战略）→ Task（战术）三级体系。

- **目标分解**：用户输入 Goal，系统自动或手动分解为 Projects 和 Tasks
- **DAG 调度**：Tasks 形成有向无环图拓扑序，依赖任务先行
- **自动派发**：通过能力标签匹配，将 Task 派发给最合适的 Agent
- **依赖解锁**：DependencyResolver 监听 Task 完成事件，自动解锁下游依赖

**关键实体**：`goals`、`projects`、`tasks`、`task_dependencies`

### 场景库

场景库是 Nexus 的核心交付形态，一个场景 = 业务流程模板。

- **场景模板**（Scenario Template）：预定义的业务流程骨架，包含 Task 列表、依赖关系、HITL 配置
- **场景实例**（Scenario Instance）：场景模板实例化后的执行实体，对应一个 Project
- **场景管理**：浏览、安装、卸载、版本管理
- **行业包集成**：场景库支持从行业包批量导入场景

**关键实体**：`scenario_templates`、`scenario_projects`、`scenario_tasks`

### 智能体管理

Agent 是实际执行 Task 的 AI 实体。

- **Agent Profile**：注册 Agent 的能力描述、角色定义、归属场景
- **能力声明**：每个 Agent 声明自己拥有哪些能力标签（`capability_tags`）
- **权重配置**：同类能力不同 Agent 有不同权重，影响匹配优先级
- **状态跟踪**：Agent 领 Task、执行中、完成、失败等状态流转

**关键实体**：`agents`、`agent_capabilities`、`agent_tag_weights`

### 能力标签

Nexus 采用双层能力标签体系：通用四维 + 行业标签库。

| 维度 | 含义 | 示例 |
|------|------|------|
| `business` | 业务领域知识 | `biz:chemical-park-operations` |
| `professional` | 专业技能/行业专长 | `chem:hazmat-identification` |
| `technical` | 技术工具/框架能力 | `chem:msds-parsing` |
| `management` | 管理/协调能力 | `chem:incident-command` |

- **行业标签格式**：`{行业前缀}:{能力标识符}`，如 `chem:hazmat-identification`
- **匹配引擎**：Task 声明 `requires`，Agent 声明 `has`，引擎计算交集得分进行匹配
- **生命周期**：标签有状态（active/deprecated），版本有语义（MAJOR/MINOR/PATCH）

**关键实体**：`capability_tags`、`industry_capability_tags`、`industry_packs`

### 人机协同（HITL）

HITL 是 Nexus 的核心差异化能力，确保 AI 不会在关键节点走偏。

| condition_type | 含义 | 执行时机 | 人类动作 |
|---------------|------|---------|---------|
| `none` | 无需干预 | 自动执行 | 无 |
| `auto_eval` | 自动评估 | 执行完成后自动评估结果 | 无 |
| `human_decision` | 审批决策 | 执行前暂停，等人拍板 | 审批（同意/拒绝/修改方案） |
| `human_input` | 数据输入 | 执行中暂停，等人提供数据 | 输入数据/信息 |

- **human_input_requests**：存储所有等待人类的请求，含分支选项、超时配置
- **飞书通知**：Task 进入 `waiting_human` 状态时自动推送通知
- **超时处理**：可配置超时策略，超时后自动执行默认分支或升级

**关键实体**：`human_input_requests`

### 验证仲裁

验证仲裁确保每个 Task 的执行结果符合预期。

- **验证（Verification）**：Task 执行完成后，验收脚本自动校验输出是否正确
- **仲裁（Ruling）**：验证不通过时，由仲裁机制决定是重试、跳过还是升级
- **裁决（Adjudication）**：人工对异常 Task 做最终决定（适用于 `human_decision` 模式）
- **探索模式 vs 验证模式**：探索模式下 Agent 可自由尝试；验证模式下按预设流程严格执行

**关键实体**：`task_verification_rules`、`task_adjudication_records`

## 非功能需求

### 性能（SLA）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 场景实例化时间 | < 5s | 从场景模板实例化到 Project/Tasks 创建完成 |
| Task 派发延迟 | < 1s | Task 创建到匹配到 Agent 并派发的延迟 |
| 匹配引擎 QPS | ≥ 50 次/秒 | 能力标签匹配引擎的吞吐量 |
| API 响应时间（P95） | < 500ms | 主要 CRUD 接口响应时间 |
| 并发 Agent 数 | ≥ 20 个 | 单实例支持的最大并发执行 Agent |

### 可用性

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | ≥ 99%（计划内维护除外） |
| 平均无故障时间（MTBF） | ≥ 720 小时（30 天） |
| 数据持久性 | ≥ 99.99%（写入后不丢失） |
| 故障恢复时间（RTO） | < 1 小时 |

### 安全

| 需求 | 说明 |
|------|------|
| 认证 | API 需要 API Key 认证，支持飞书 OAuth |
| 授权 | 租户隔离（Phase 2 企业版） |
| 数据加密 | 敏感数据（API Key、配置）加密存储 |
| 审计日志 | 所有写操作记录操作日志 |
| HITL 防注入 | 人类输入数据做输入校验，防止注入攻击 |

### 可扩展性

| 维度 | 扩展方式 |
|------|---------|
| Agent 类型 | 可接入任意支持 REST API 的 AI Agent |
| 场景规模 | 单场景支持 ≥ 100 个 Task |
| 标签体系 | 通过行业包可扩展新行业标签 |
| 通知渠道 | 插件化通知适配器（飞书、钉钉、企业微信） |
