## 1. 模块总览

域6提供系统扩展相关的核心能力，包含以下四个模块：

| 模块 | 说明 | 端点数量 |
|------|------|---------|
| **MCP** | Model Context Protocol，模型上下文协议。用于管理 MCP Server 实例，支持工具发现与调用。 | 7 |
| **Skills** | 技能管理。提供 Agent 技能包的元信息、文件结构和安装引导。 | 5 |
| **Attachments** | 统一附件管理。所有实体（会话、消息、任务等）的文件附件统一接入，支持上传、下载、关联与删除。 | 7 |
| **Artifacts** | AI 产出物管理。存储和检索 AI 生成的完整文件（如代码、报告、图片），支持版本更新与下载。 | 6 |

## 3. 各模块详细接口

### 3.1 MCP 模块

MCP（Model Context Protocol）模块负责管理 MCP Server 生命周期及工具发现。

#### 3.1.1 获取 MCP 连接状态

```
GET /api/v1/mcp
```

返回当前 MCP 连接的整体状态和统计信息。

**响应示例**

```json
{
  "status": "connected",
  "server_count": 3,
  "total_tools": 24,
  "active_servers": 2
}
```

---

#### 3.1.2 获取 MCP Server 列表

```
GET /api/v1/mcp-servers
```

返回所有已注册的 MCP Server 实例列表。

**响应示例**

```json
{
  "servers": [
    {
      "server_id": "srv_001",
      "name": "filesystem-mcp",
      "url": "http://localhost:3100",
      "status": "active",
      "tools_count": 8,
      "created_at": "2026-04-10T09:00:00Z",
      "updated_at": "2026-05-20T14:30:00Z"
    }
  ],
  "total": 1
}
```

---

#### 3.1.3 注册 MCP Server

```
POST /api/v1/mcp-servers
```

注册一个新的 MCP Server 实例。

**请求体**

```json
{
  "name": "filesystem-mcp",
  "url": "http://localhost:3100",
  "description": "文件系统访问服务"
}
```

**响应示例**

```json
{
  "server_id": "srv_002",
  "name": "filesystem-mcp",
  "url": "http://localhost:3100",
  "status": "inactive",
  "tools_count": 0,
  "created_at": "2026-05-27T17:00:00Z",
  "updated_at": "2026-05-27T17:00:00Z"
}
```

---

#### 3.1.4 获取指定 Server 详情

```
GET /api/v1/mcp-servers/{server_id}
```

根据 Server ID 获取详细信息。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| server_id | string | MCP Server 唯一标识 |

**响应示例**

```json
{
  "server_id": "srv_001",
  "name": "filesystem-mcp",
  "url": "http://localhost:3100",
  "description": "文件系统访问服务",
  "status": "active",
  "tools_count": 8,
  "created_at": "2026-04-10T09:00:00Z",
  "updated_at": "2026-05-20T14:30:00Z"
}
```

---

#### 3.1.5 更新 MCP Server

```
PUT /api/v1/mcp-servers/{server_id}
```

更新已注册的 MCP Server 配置。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| server_id | string | MCP Server 唯一标识 |

**请求体**

```json
{
  "name": "filesystem-mcp",
  "url": "http://localhost:3101",
  "description": "文件系统访问服务（已更新）"
}
```

---

#### 3.1.6 删除 MCP Server

```
DELETE /api/v1/mcp-servers/{server_id}
```

删除指定的 MCP Server 实例。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| server_id | string | MCP Server 唯一标识 |

**响应**

```
204 No Content
```

---

#### 3.1.7 获取 Server 工具列表

```
GET /api/v1/mcp-servers/{server_id}/tools
```

返回指定 MCP Server 提供的所有可用工具。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| server_id | string | MCP Server 唯一标识 |

**响应示例**

```json
{
  "server_id": "srv_001",
  "tools": [
    {
      "name": "read_file",
      "description": "读取指定路径的文件内容",
      "input_schema": {
        "type": "object",
        "properties": {
          "path": { "type": "string" }
        },
        "required": ["path"]
      }
    },
    {
      "name": "write_file",
      "description": "写入内容到指定路径文件",
      "input_schema": {
        "type": "object",
        "properties": {
          "path": { "type": "string" },
          "content": { "type": "string" }
        },
        "required": ["path", "content"]
      }
    }
  ],
  "total": 2
}
```

| skill_id | string | 技能唯一标识 |

**响应**

返回原始文件内容，`Content-Type` 根据文件类型确定。

### 3.3 Attachments 模块

Attachments 模块提供统一的文件附件管理，所有实体（会话、消息、任务等）的附件均通过此模块处理。

#### 3.3.1 获取附件列表

```
GET /api/v1/attachments
```

返回当前用户或全局的附件列表，支持分页和过滤。

**查询参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| entity_type | string | 按关联实体类型过滤（如 `message`, `task`） |
| entity_id | string | 按关联实体 ID 过滤 |
| page | int | 页码（默认 1） |
| page_size | int | 每页数量（默认 20） |

**响应示例**

```json
{
  "attachments": [
    {
      "id": "att_001",
      "filename": "report.pdf",
      "size": 1048576,
      "content_type": "application/pdf",
      "entity_type": "task",
      "entity_id": "task_123",
      "created_at": "2026-05-20T10:00:00Z"
    }
  ],
  "total": 1,
  "page": 1,
  "page_size": 20
}
```

---

#### 3.3.2 上传附件

```
POST /api/v1/attachments/upload
```

上传文件作为附件，支持 `multipart/form-data`。

**请求**

- `Content-Type`: `multipart/form-data`
- `file`: 二进制文件内容
- `entity_type`（可选）: 关联实体类型
- `entity_id`（可选）: 关联实体 ID

**响应示例**

```json
{
  "id": "att_002",
  "filename": "image.png",
  "size": 524288,
  "content_type": "image/png",
  "url": "/api/v1/attachments/att_002/download",
  "created_at": "2026-05-27T17:10:00Z"
}
```

---

#### 3.3.3 获取附件元信息

```
GET /api/v1/attachments/{id}
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 附件唯一标识 |

**响应示例**

```json
{
  "id": "att_001",
  "filename": "report.pdf",
  "size": 1048576,
  "content_type": "application/pdf",
  "entity_type": "task",

  "updated_at": "2026-05-20T10:00:00Z"
}
```

---

#### 3.3.4 下载附件

```
GET /api/v1/attachments/{id}/download
```

下载附件原始文件内容。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 附件唯一标识 |

**响应**

返回原始二进制文件流，`Content-Type` 与上传时的 `content_type` 一致。

---

#### 3.3.5 删除附件

```
DELETE /api/v1/attachments/{id}
```

删除指定附件（物理文件同步删除）。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 附件唯一标识 |

**响应**

```
204 No Content
```

---

#### 3.3.6 关联附件到实体

```
POST /api/v1/attachments/{id}/link
```

将已存在的附件关联到指定实体（会话、消息、任务等）。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 附件唯一标识 |

**请求体**

```json
{
  "entity_type": "task",
  "entity_id": "task_456"
}
```

---

#### 3.3.7 解除附件关联

```
DELETE /api/v1/attachments/{id}/link/{entity_type}/{entity_id}
```

解除附件与指定实体的关联关系（不删除附件本身）。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 附件唯一标识 |
| entity_type | string | 实体类型 |
| entity_id | string | 实体 ID |

**响应**

```
204 No Content
```

## 四、错误码定义

### 4.1 HTTP 状态码

| 状态码 | 说明 | 使用场景 |
|--------|------|---------|
| 200 | OK | 成功响应 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 删除成功（无返回体） |
| 400 | Bad Request | 请求参数错误 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 状态冲突（如重复创建） |
| 500 | Internal Server Error | 服务器内部错误 |

### 4.2 业务错误码

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|------------|------|---------|
| `MCP_SERVER_NOT_FOUND` | 404 | MCP Server 不存在 | 查询/更新/删除不存在的 Server |
| `MCP_CONNECTION_FAILED` | 500 | MCP 连接失败 | Server 不可达 |
| `MCP_DUPLICATE_NAME` | 409 | Server 名称已存在 | 注册同名 Server |
| `SKILL_NOT_FOUND` | 404 | 技能不存在 | 查询不存在的技能 |
| `SKILL_FILE_NOT_FOUND` | 404 | 技能文件不存在 | 请求不存在的技能文件 |
| `ATTACHMENT_NOT_FOUND` | 404 | 附件不存在 | 下载/删除不存在的附件 |
| `ATTACHMENT_UPLOAD_FAILED` | 500 | 上传失败 | 文件大小超限或存储异常 |
| `ATTACHMENT_FILE_TYPE_REJECTED` | 400 | 文件类型被拒绝 | 上传禁止的文件类型 |
| `ARTIFACT_NOT_FOUND` | 404 | 产出物不存在 | 查询/更新/删除不存在的产出物 |

### 4.3 错误响应格式

```json
{
  "error": "MCP_SERVER_NOT_FOUND",
  "message": "MCP Server 不存在",
  "server_id": "srv_999"
}
```
