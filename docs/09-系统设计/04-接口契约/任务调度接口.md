### 2.1 Task CRUD

**功能**：获取单个任务详情

```json
{

  "priority": "high",
  "capability_tags": {
    "business": ["emergency_response"],
    "professional": ["risk_assessment"],
    "technical": ["python"],
    "management": ["coordination"]
  },
  "depends_on": ["task-001"],
  "next_step": ["task-003", "task-004"],
  "assigned_agent": "guzi",
  "project_id": "proj-001",
  "goal_id": "goal-001",
  "retry_count": 0,
  "max_retries": 3,
  "acceptance_criteria": "{\"criteria\":[{\"type\":\"compile\",\"name\":\"TypeScript编译\",\"desc\":\"npx tsc --noEmit 0 errors\"}]}",
  "needs_verification": true,

```

| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 404 | TASK_NOT_FOUND | 任务不存在 |

#### POST /api/v1/tasks/

**功能**：创建任务

**请求体**：
```json
{
  "title": "场景勘测",
  "description": "对目标区域进行现场勘测",
  "project_id": "proj-001",
  "goal_id": "goal-001",
  "priority": "high",
  "capability_tags": {
    "business": ["emergency_response"],
    "professional": ["risk_assessment"],
    "technical": ["python"],
    "management": ["coordination"]
  },
  "depends_on": ["task-001"],
  "acceptance_criteria": "{\"criteria\":[{\"type\":\"compile\",\"name\":\"TypeScript编译\",\"desc\":\"npx tsc --noEmit 0 errors\"}]}",
  "needs_verification": true,
  "context_md": null,
  "strict_mode": true,
  "max_retries": 3
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 任务标题 |
| project_id | string | 是 | 所属项目 ID |
| goal_id | string | 否 | 所属目标 ID（未指定时从 Project 推导） |
| priority | string | 否 | high/medium/low，默认 medium |
| capability_tags | object | 是 | 四维能力标签 |
| depends_on | array | 是 | 依赖任务 ID 列表（DAG 顺序） |
| acceptance_criteria | string | 条件必填 | needs_verification=true 时必填，JSON 格式 |
| needs_verification | boolean | 否 | 默认 true |
| context_md | string | 条件必填 | needs_verification=true 且完成时必填 |
| strict_mode | boolean | 否 | true=缺失前置标签阻断，false=自动补全 |
| max_retries | integer | 否 | 最大重试次数，默认 3 |

**响应 201**：
```json
{
  "id": "task-7436736c8dd9",
  "title": "场景勘测",
  "status": "todo",
  "priority": "high",
  "capability_tags": { ... },
  "depends_on": ["task-001"],
  "project_id": "proj-001",
  "goal_id": "goal-001",
  "tag_warnings": [],
  "auto_added_tags": []
}
```

**错误响应**：
| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 400 | MISSING_PREREQUISITES | 缺失前置标签（strict_mode=true） |
| 400 | ACCEPTANCE_CRITERIA_REQUIRED | needs_verification=true 但未设置 acceptance_criteria |
| 400 | CIRCULAR_DEPENDENCY | 依赖关系存在环形 |
| 400 | VALIDATION_FAILED | 任务约束校验失败 |

---

#### PUT /api/v1/tasks/{task_id}

**功能**：完整更新任务

**请求体**（同 POST，但所有字段可选）：
```json
{
  "title": "场景勘测（修订版）",
  "status": "in_progress",
  "capability_tags": { ... },
  "depends_on": ["task-001", "task-002"],
  "assigned_agent": "guzi",
  "context_md": "## 执行摘要\n..."
}
```

**特殊逻辑**：
- `depends_on` 变更时自动同步三处（JSON + 关系表 + next_step）
- `status` 变为 done/completed/review_needed 时触发依赖解锁和 Project 状态检查
- `status` 变为 in_progress 时写入 `execution_logs`

**响应 200**：返回更新后的 Task 对象（格式同 GET）

---

#### PATCH /api/v1/tasks/{task_id}

**功能**：部分更新任务（兼容旧 API）

**请求体**：
```json
{
  "status": "done",
  "result": "执行完成，返回结果摘要",
  "context_md": "## 执行摘要\n..."
}
```

**响应 200**：返回更新后的 Task 对象

---

#### DELETE /api/v1/tasks/{task_id}

**功能**：删除任务并清理所有依赖关系

**清理操作**：
1. 从父任务的 `next_step` 中移除
2. 从子任务的 `depends_on` 中移除
3. 删除 `task_dependencies` 表中的记录
4. 清理 `task_comments`、`task_attachments`、`task_labels`、`task_activity_log` 等关联数据

**响应 204**：无内容

**错误响应**：
| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 404 | TASK_NOT_FOUND | 任务不存在 |

### 2.4 Agent 心跳与任务派发

#### POST /api/v1/agents/{agent_id}/heartbeat

**功能**：Agent 心跳上报（包含任务分配）

**请求体**：
```json
{
  "status": "online",
  "load": 45,
  "current_tasks": 2
}
```

**响应 200**：
```json
{
  "ok": true,
  "pending_tasks": [
    {
      "id": "task-7436736c8dd9",
      "title": "场景勘测",
      "priority": "high",
      "project_id": "proj-001"
    }
  ],
  "stop_signals": [],
  "pending_stops": []
}
```

**服务端处理**：
1. 更新 `agents.last_heartbeat` 为当前时间
2. 如果 Agent 状态为 stale/offline → 恢复为 online
3. 重置 `consecutive_offline_count`
4. 如果有离线期间被回收的任务 → 触发重新分配
5. 返回待领取任务列表

## 10. 错误码定义

### 10.1 HTTP 状态码

| 状态码 | 说明 | 使用场景 |
|--------|------|---------|
| 200 | OK | 成功响应 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 删除成功（无返回体） |
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未认证 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 状态冲突（如重复创建） |
| 422 | Unprocessable Entity | 语义错误（如铁律校验失败） |
| 500 | Internal Server Error | 服务器内部错误 |
| 503 | Service Unavailable | 服务不可用（如调度器未启动） |

### 10.2 业务错误码

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|------------|------|---------|
| `TASK_NOT_FOUND` | 404 | 任务不存在 | GET/PUT/PATCH/DELETE 任务时 ID 不存在 |
| `MISSING_PREREQUISITES` | 400 | 缺失前置标签 | strict_mode=true 且 capability_tags 有未满足的 prerequisites |
| `CIRCULAR_DEPENDENCY` | 400 | 环形依赖 | depends_on 设置后检测到环形 |
| `ACCEPTANCE_CRITERIA_REQUIRED` | 400 | 验收标准缺失 | needs_verification=true 但未设置 acceptance_criteria |
| `CONTEXT_MD_REQUIRED` | 400 | 执行摘要缺失 | needs_verification=true 且完成时未填写 context_md |
| `VALIDATION_FAILED` | 400 | 任务约束校验失败 | 任务约束条件不满足 |
| `DEPENDS_ON_NOT_FOUND` | 400 | 依赖任务不存在 | depends_on 中引用了不存在的任务 ID |
| `MAX_RETRIES_EXCEEDED` | 400 | 超过最大重试次数 | retry_count >= max_retries 时重试 |
| `INVALID_STATUS_TRANSITION` | 409 | 无效状态转换 | 如对 done 状态任务调用 pause |
| `ALREADY_ASSIGNED` | 409 | 任务已被分配 | 对已有 assigned_agent 的任务重复分配 |
| `ORPHAN_TASK` | 422 | 孤儿任务 | Server 重启后 in_progress 任务被标记为孤儿 |
| `AGENT_NOT_ONLINE` | 400 | Agent 不在线 | 向离线的 Agent 派发任务 |

### 10.3 错误响应格式

所有业务错误返回统一格式：

```json
{
  "error": "MISSING_PREREQUISITES",
  "message": "Missing prerequisites for capability tags: {'chem:hazmat-identification': ['chem:msds-parsing']}",
  "type": "missing_prerequisites",
  "detail": {
    "missing": {
      "chem:hazmat-identification": ["chem:msds-parsing"]
    }
  }
}
```
