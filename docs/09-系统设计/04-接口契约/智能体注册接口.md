## 一、概述

### 1.1 接口范围

本文档定义 Agent（智能体）注册、心跳、状态管理、能力更新和列表查询的所有 API 接口契约。

### 1.2 Agent 状态说明

| 状态 | 说明 |
|------|------|
| `online` | Agent 正常运行，心跳正常 |
| `stale` | 心跳超时（超过 60 秒无心跳），但未超过 5 分钟 |
| `offline` | Agent 下线或连续离线超过 5 分钟 |

## 四、心跳超时检测机制

### 4.1 心跳超时规则

| 条件 | 状态变更 | 说明 |
|------|---------|------|
| 心跳正常 | 保持 `online` | 最后心跳时间更新 |
| 超过 60 秒无心跳 | `online` → `stale` | 标记为失联但未确认下线 |
| 超过 5 分钟无心跳 | `stale` → `offline` | 确认 Agent 下线 |

### 4.2 Heartbeat 检测 stale/offline 任务

Heartbeat 端点同时负责检测和重新派发 `review_needed` 任务：

```python
# 伪代码：heartbeat 端点中的检测逻辑
def handle_heartbeat(agent_id: str):
    agent = db.get_agent(agent_id)
    now = time.time()

    # 检测心跳超时
    if agent.last_heartbeat < now - 60:
        agent.health_status = "stale"
    if agent.last_heartbeat < now - 300:
        agent.health_status = "offline"
        agent.consecutive_offline_count += 1

    # 检测该 Agent 的 review_needed 任务并重新派发
    review_tasks = db.query("""
        SELECT * FROM tasks
        WHERE status = 'review_needed'
          AND assigned_agent = :agent_id
          AND verification_cycle < 3
    """, {"agent_id": agent_id})

    for task in review_tasks:
        # 优先检查人工裁决
        ruling = db.get_latest_ruling(task.id)
        if ruling:
            task.status = "in_progress"
            task.ruling_instruction = ruling.content
        else:
            # 其次检查验证意见
            comment = db.get_latest_verification(task.id)
            if comment:
                task.status = "in_progress"
                task.instruction_comment_id = comment.id
        db.commit()
```

## 五、错误码定义

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | `invalid_capability_tags` | capability_tags 格式错误 |
| 400 | `invalid_trigger_mode` | trigger_mode 必须是 sse 或 polling |
| 404 | `agent_not_found` | Agent 不存在 |
| 409 | `agent_name_exists` | Agent 名称已被使用 |
| 503 | `agent_unreachable` | Agent 无法连接（仅 SSE 模式） |
