## 一、概述

### 1.1 接口范围

本章节定义行业包（Industry Pack）的管理与查询接口。行业包是场景库的重要扩展，用于封装特定行业的完整解决方案模板，支持行业包的创建、内容管理、实例化等操作。

### 2.1 接口总表

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建行业包 | POST | `/api/v1/industry-packs` | 创建标准包或定制包 |
| 查询行业包列表 | GET | `/api/v1/industry-packs` | 列表查询，支持类型/行业筛选 |
| 查询行业包详情 | GET | `/api/v1/industry-packs/{pack_id}` | 获取行业包详情 |
| 更新行业包 | PUT | `/api/v1/industry-packs/{pack_id}` | 更新行业包信息 |
| 删除行业包 | DELETE | `/api/v1/industry-packs/{pack_id}` | 删除行业包 |
| 添加定制内容 | POST | `/api/v1/industry-packs/{pack_id}/customizations` | 添加/修改/移除定制内容 |
| 查询定制内容 | GET | `/api/v1/industry-packs/{pack_id}/customizations` | 查询定制包的变更记录 |
| 查询完整视图 | GET | `/api/v1/industry-packs/{pack_id}/full-view` | 查询合并后的完整内容 |
| 查询差异 | GET | `/api/v1/industry-packs/{pack_id}/diff` | 查询定制包相对标准包的差异 |
| 发布定制包 | POST | `/api/v1/industry-packs/{pack_id}/release` | 发布定制包 |
| 实例化行业包 | POST | `/api/v1/industry-packs/{pack_id}/instantiate` | 实例化为 Goal |
| 查询发布历史 | GET | `/api/v1/industry-packs/{pack_id}/releases` | 查询发布记录 |
| 添加包内容 | POST | `/api/v1/industry-packs/{pack_id}/contents` | 添加场景/技能等内容 |
| 删除包内容 | DELETE | `/api/v1/industry-packs/{pack_id}/contents/{content_type}/{content_id}` | 删除指定内容 |

### 2.2 创建行业包

```
POST /api/v1/industry-packs
Content-Type: application/json

Request Body:
{
  "name": "化工应急行业包",
  "industry": "chemical-emergency",
  "version": "1.0.0",
  "pack_type": "standard",
  "description": "化工行业应急响应标准能力包"
}

Request Body (定制包):
{
  "name": "XX化工园区定制包",
  "industry": "chemical-emergency",
  "version": "1.0.0",
  "pack_type": "custom",
  "base_pack_id": "pack-chemical-emergency-v1",
  "description": "基于化工应急标准包，为XX园区定制"
}

Response 201:
{
  "id": "pack-chemical-custom-xxpark",
  "name": "XX化工园区定制包",
  "industry": "chemical-emergency",
  "version": "1.0.0",
  "pack_type": "custom",
  "base_pack_id": "pack-chemical-emergency-v1",
  "status": "draft",
  "created_at": 1716800000
}

Error Responses:
- 400: 定制包未指定 base_pack_id
  {"error": "custom_pack_requires_base", "detail": "定制包必须指定 base_pack_id"}
- 400: base_pack_id 不存在或不是标准包
  {"error": "invalid_base_pack", "base_pack_id": "pack-xxx"}
```

### 2.5 添加定制内容

```
POST /api/v1/industry-packs/{pack_id}/customizations
Content-Type: application/json

Request Body (新增标签):
{
  "content_type": "tag",
  "action": "add",
  "content_id": "chem:xxpark:specific-hazmat",
  "custom_data": {
    "tag_name": "XX园区特殊危化品",
    "dimension": "professional",
    "level": "advanced",
    "description": "XX园区特有的危化品种类识别"
  }
}

Request Body (修改标签):
{
  "content_type": "tag",
  "action": "modify",
  "content_id": "chem:hazmat-identification",
  "custom_data": {
    "tag_name": "危化品识别（XX园区定制版）",
    "level": "expert"
  }
}

Request Body (移除标签):
{
  "content_type": "tag",
  "action": "remove",
  "content_id": "chem:old-emergency-procedure"
}

Response 201:
{
  "id": "cust-xxx",
  "pack_id": "pack-chemical-custom-xxpark",
  "content_type": "tag",
  "action": "add",
  "content_id": "chem:xxpark:specific-hazmat",
  "created_at": 1716800000
}

Error Responses:
- 400: 定制标签ID不符合命名空间规则
  {"error": "invalid_custom_tag_id", "detail": "定制标签ID必须符合格式 '行业前缀:客户缩写:能力标识符'"}
- 400: 定制标签与标准包冲突
  {"error": "tag_conflict", "tag_id": "chem:hazmat-identification", "detail": "该标签ID已被标准包使用"}
```

## 三、错误码定义

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | `invalid_tag_id` | 标签ID不符合命名空间规则 |
| 400 | `invalid_custom_tag_id` | 定制标签ID不符合格式要求 |
| 400 | `custom_pack_requires_base` | 定制包未指定 base_pack_id |
| 400 | `prerequisites_not_satisfied` | 标签 prerequisites 不满足 |
| 400 | `circular_dependency_detected` | 检测到环形依赖 |
| 400 | `tag_conflict` | 定制标签与标准包冲突 |
| 400 | `replacement_tag_not_found` | 替换目标标签不存在 |
| 404 | `tag_not_found` | 标签不存在 |
| 404 | `pack_not_found` | 行业包不存在 |
| 404 | `invalid_base_pack` | base_pack_id 无效或非标准包 |
| 409 | `tag_already_exists` | 标签ID已存在 |
| 409 | `pack_already_exists` | 行业包名称已存在 |
