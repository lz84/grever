## 一、概述

### 1.1 接口范围

本文档定义 Nexus 系统中验证与仲裁相关的所有 API 接口，包括验证触发、验证执行、仲裁裁决、以及验证历史查询。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **验证** | Task 完成后自动或手动触发，检查产出是否符合验收标准 |
| **仲裁** | 验证循环超过 3 次仍不通过时，转入人工裁决流程 |
| **三级 Verifier** | Task → Project → Goal → 系统默认值 (kouzi) |

## 二、接口清单总表

### 2.1 验证触发

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 手动触发验证 | POST | `/api/v1/tasks/{task_id}/verify` | 手动触发验证流程 |
| 自动触发验证 | - | (Task 完成时自动触发) | 见任务调度接口 |

### 2.2 验证查询

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 查询验证历史 | GET | `/api/v1/tasks/{task_id}/verifications` | 获取验证记录列表 |

### 2.3 人工裁决

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 裁决任务争议 | POST | `/api/v1/tasks/{task_id}/ruling` | 对 disputed 任务做出裁决 |
| 获取裁决统计 | GET | `/api/v1/human-review/stats` | 获取裁决中心统计 |
| 获取待裁决列表 | GET | `/api/v1/human-review/pending` | 获取待裁决任务列表 |
| 批量裁决 | POST | `/api/v1/human-review/batch-ruling` | 批量裁决多个任务 |

## 三、接口详情

### 3.1 手动触发验证

```
POST /api/v1/tasks/{task_id}/verify
Content-Type: application/json

Request Body (可选):
{
  "acceptance_criteria_override": {
    "criteria": [
      {"type": "compile", "desc": "TypeScript 编译通过"}
    ]
  }
}

Response 200:
{
  "task_id": "task-xxx",
  "status": "verifying",
  "verifier_agent": "kouzi",
  "verification_cycle": 1,
  "message": "验证已触发"
}

Error Responses:
- 404: 任务不存在
  {"error": "task_not_found", "task_id": "task-xxx"}
- 400: 任务状态不允许验证
  {"error": "invalid_task_status", "current_status": "todo", "detail": "只有 in_progress 或 done 状态的任务可以触发验证"}
```

### 3.2 查询验证历史

```
GET /api/v1/tasks/{task_id}/verifications

Response 200:
[
  {
    "cycle": 1,
    "type": "verification",
    "verifier": "kouzi",
    "passed": false,
    "checks": [
      {"name": "compile", "passed": false, "detail": "3 errors"},
      {"name": "api", "passed": true, "detail": "/api/v1/tasks 返回 200"},
      {"name": "page", "passed": true, "detail": "页面正常渲染"}
    ],
    "body": "❌ 验证失败 (cycle 1/3)\n\n❌ 编译检查: 3 errors\n✅ API 检查: 通过\n✅ 页面检查: 通过",
    "created_at": "2026-05-04T10:30:00Z"
  },
  {
    "type": "human_ruling",
    "author": "用户",
    "ruling_action": "done",
    "body": "编译错误可以忽略，这是已知问题。标记为 done。",
    "created_at": "2026-05-04T11:00:00Z"
  }
]
```

### 3.3 裁决任务争议

```
POST /api/v1/tasks/{task_id}/ruling
Content-Type: application/json

Request Body:
{
  "ruling": "裁决意见",
  "action": "done | in_progress | verifying"
}

Response 200 (通过):
{
  "task_id": "task-xxx",
  "status": "done",
  "ruling_comment_id": "cmt-001",
  "verification_cycle": 0,
  "completed_at": 1716800200
}

Response 200 (打回):
{
  "task_id": "task-xxx",
  "status": "in_progress",
  "ruling_comment_id": "cmt-002",
  "message": "任务已打回，重新派发给 Executor"
}

Response 200 (调整验收标准):
{
  "task_id": "task-xxx",
  "status": "verifying",
  "ruling_comment_id": "cmt-003",
  "message": "验收标准已调整，重新触发验证"
}

Error Responses:
- 400: 任务不是 disputed 状态
  {"error": "not_disputed", "current_status": "review_needed", "detail": "只有 disputed 状态的任务可以裁决"}
- 400: action 无效
  {"error": "invalid_action", "action": "xxx", "detail": "action 必须是 done/in_progress/verifying 之一"}
- 404: 任务不存在
```

## 四、验证类型矩阵

| 类型 | type 值 | 说明 | 适用场景 |
|------|---------|------|---------|
| 编译检查 | `compile` | 执行编译命令，检查返回码 | 代码类任务 |
| API 检查 | `api` | 发送 HTTP 请求，验证状态码和响应结构 | 接口类任务 |
| 页面检查 | `page` | 访问页面 URL，验证渲染结果 | 前端类任务 |
| LLM 辅助审查 | `llm_review` | 用 LLM 做主观质量评估 | 设计/文档类任务 |
| 自定义脚本 | `custom` | 运行自定义脚本验证 | 特殊场景 |

## 五、状态流转

```
todo ──→ in_progress ──→ verifying ──→ review_needed ────────────────┐
  │              │           │           │                            │
  │              │           │           │ (cycle < 3)                │
  │              │           │           ↓                            │
  │              │           │     review_needed → in_progress（重新派发）│
  │              │           │           │                            │
  │              │           │           │ (cycle >= 3)                │
  │              │           │           ↓                            │
  │              │           │     disputed ──→ [human_ruling] ──→ done/in_progress
  │              │           │                                      │
  │              │           │ (通过)                                │
  │              │           ↓                                      │
  │              │         done                                      │
  │              │                                                  │
  │              └── on_hold                                         │
  └── abandoned
```

## 六、错误码定义

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | `invalid_task_status` | 任务状态不允许验证 |
| 400 | `not_disputed` | 任务不是 disputed 状态，不能裁决 |
| 400 | `invalid_action` | action 必须是 done/in_progress/verifying |
| 404 | `task_not_found` | 任务不存在 |
| 500 | `verification_failed` | 验证执行失败 |
