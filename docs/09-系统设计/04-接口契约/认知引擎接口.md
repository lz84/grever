## 1. 模块总览

认知引擎（Cognitive Engine）是 Agents Nexus 平台的核心推理与认知管理模块，负责：

- **知识注入**：接收外部任务结果、流程结果、争议结果，注入统一知识图谱
- **认知管理**：创建、读取、更新、删除认知节点（cognition），支持认知评估
- **图谱查询**：提供全局知识图谱查询能力，支持语义检索
- **推荐能力**：基于当前认知状态提供智能推荐
- **上下文管理**：管理对话/任务上下文，为 Agent 提供记忆支撑

> 认知引擎是平台的"大脑"，所有 Agent 的学习、记忆、推理均依赖本模块。

#### DELETE /api/v1/grasp/cognition/{cognition_id}

**功能描述**：删除指定的认知节点。执行后节点及其关联关系从知识图谱中移除。

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| cognition_id | string (UUID) | ✅ | 认知节点唯一标识 |

**请求头**：

| 头信息 | 值 | 说明 |
|--------|-----|------|
| Authorization | Bearer {token} | 用户访问令牌 |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "cognition_id": "cog-uuid-1234",
    "deleted_at": "2026-05-27T10:30:00Z"
  }
}
```

**错误码**：

| code | 说明 |
|------|------|
| 404 | 认知节点不存在 |
| 403 | 无权限删除 |

#### GET /api/v1/grasp/graph

**功能描述**：查询全局知识图谱，支持多维度过滤和分页返回全量或局部图谱数据。

| offset | integer | ❌ | 0 | 偏移量 |
| min_weight | float | ❌ | - | 最小关系权重过滤（0.0-1.0） |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "nodes": [

          "confidence": 0.92
        }
      }

    "edges": [
      {
        "source": "cog-uuid-1234",
        "target": "cog-uuid-5678",
        "relation_type": "related_to",
        "weight": 0.85
```

#### POST /api/v1/grasp/cognition

**功能描述**：创建一个新的认知节点，将其插入知识图谱。

**请求头**：

| 头信息 | 值 | 说明 |
|--------|-----|------|
| Content-Type | application/json | 请求体格式 |

**请求体**：

```json
{
  "type": "concept",
  "name": "市场趋势分析",
  "attributes": {
    "source": "manual",
    "confidence": 0.85,
    "tags": ["market", "trend", "analysis"]
  },
  "relations": [

      "target_id": "cog-uuid-existing",
      "relation_type": "related_to",
      "weight": 0.7
    }
  ]
}
```

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| type | string | ✅ | 节点类型：concept / event / entity / task |

| attributes.tags | string[] | ❌ | 标签列表 |
| relations | array | ❌ | 关联关系 |
| relations[].target_id | string | ✅ (relations非空时) | 目标节点ID |
| relations[].relation_type | string | ✅ (relations非空时) | 关系类型 |
| relations[].weight | float | ❌ | 关系权重（0.0-1.0） |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "cognition_id": "cog-uuid-new-9999",
    "type": "concept",
    "name": "市场趋势分析",
    "created_at": "2026-05-27T11:30:00Z"
  }
}
```

#### POST /api/v1/grasp/recommend

**功能描述**：基于当前认知状态，向 Agent 或用户返回智能推荐列表。推荐内容可为下一步行动、知识补充方向或决策建议。

**请求头**：

| 头信息 | 值 | 说明 |
|--------|-----|------|
| Content-Type | application/json | 请求体格式 |

**请求体**：

```json
{
  "agent_id": "agent-uuid-dddd",
  "context": {
    "current_task": "risk_assessment",
    "related_concepts": ["cog-uuid-1111", "cog-uuid-2222"]
  },
  "recommend_type": "action",
  "limit": 5
}
```

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| agent_id | string | ✅ | 请求推荐的主体 |
| context | object | ❌ | 当前上下文 |
| context.current_task | string | ❌ | 当前任务类型 |
| context.related_concepts | string[] | ❌ | 相关认知节点ID列表 |
| recommend_type | string | ❌ | 推荐类型：action / knowledge / decision |
| limit | integer | ❌ | 返回数量（默认5，最大20） |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "agent_id": "agent-uuid-dddd",
    "recommendations": [
      {
        "rank": 1,
        "type": "action",
        "title": "启动供应商B评估",
        "description": "供应商B当前交付准时率91%，建议优先评估作为备选",
        "confidence": 0.88,
        "related_cognitions": ["cog-uuid-1111"]
      },
      {
        "rank": 2,
        "type": "knowledge",
        "title": "补充：供应链韧性理论",
        "description": "建议学习供应链韧性相关概念以提升风险评估准确性",
        "confidence": 0.75,
        "related_cognitions": ["cog-uuid-3333"]
      }
    ],
    "generated_at": "2026-05-27T11:45:00Z"
  }
}
```

### 3.2 认知注入与检索

#### POST /api/v1/grasp/inject

**功能描述**：通用认知注入接口，将任意来源的认知数据注入知识图谱。

**请求体**：
```json
{
  "type": "concept",
  "name": "新概念名称",
  "content": "认知内容描述",
  "source": "manual|task|workflow|dispute",
  "attributes": {
    "confidence": 0.9,
    "tags": ["tag1", "tag2"]
  }
}
```

**响应 201**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "cognition_id": "cog-uuid-new",
    "status": "pending"
  }
}
```

**响应 200**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "total": 150
  }
}
```

## 附录

### A. 通用错误响应格式

```json
{
  "code": <错误码>,
  "message": "<错误信息>",
  "data": null
}
```

### B. 认证说明

所有接口均需携带有效 Bearer Token（JWT），格式如下：

```
Authorization: Bearer <access_token>
```

无效或过期令牌将返回 `401 Unauthorized`。

### C. 通用错误码

| code | HTTP 状态码 | 说明 |
|------|-------------|------|
| 0 | 200 | 成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未授权 / Token 无效 |
| 403 | 403 | 无权限访问 |
| 404 | 404 | 资源不存在 |
| 429 | 429 | 请求频率超限 |
| 500 | 500 | 服务器内部错误 |
