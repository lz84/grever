## 1. 概述

本文档定义 Nexus 智能体协同驾驭平台各模块的职责划分、边界定义、数据流向和接口依赖关系，作为概要设计层模块设计的依据。

## 3. 模块职责与接口

### 3.1 M01：ReinsCore（任务调度核心）

**职责**：NexusScheduler 的核心调度引擎，负责 Goal/Project/Task 的完整生命周期管理，是整个平台的任务调度中枢。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 目标管理 | 接收高层目标，创建 Goal，记录状态变化 |
| 项目管理 | 创建 Project，管理依赖关系，跟踪进度 |
| 任务管理 | 创建 Task，分配给 Agent，跟踪状态 |
| 调度循环 | tick 驱动，按优先级和依赖关系派发任务 |
| 争议处理 | 连续验证失败时触发人工裁决 |
| 生命周期管理 | activate / pause / resume / complete |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |
|------|------|------|------|
| 创建目标 | REST | POST /api/v1/goals | 创建 Goal |
| 获取目标 | REST | GET /api/v1/goals/{id} | 获取 Goal 详情 |
| 激活目标 | REST | POST /api/v1/goals/{id}/activate | 激活 Goal |
| 暂停目标 | REST | POST /api/v1/goals/{id}/pause | 暂停 Goal |
| 恢复目标 | REST | POST /api/v1/goals/{id}/resume | 恢复 Goal |
| 创建项目 | REST | POST /api/v1/projects | 创建 Project |
| 项目生命周期 | REST | POST /api/v1/projects/{id}/activate/pause/resume | Project 生命周期 |
| 创建任务 | REST | POST /api/v1/tasks | 创建 Task |
| 任务生命周期 | REST | POST /api/v1/tasks/{id}/pause/resume | Task 生命周期 |
| 调度 tick | 内部 | scheduler.tick() | 调度器心跳 |

**内部接口**（被其他模块调用）：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| on_task_completed(task_id) | EvoAgent | 验证通过时调用 |
| on_task_failed(task_id, reason) | EvoAgent | 验证失败时调用 |
| on_hitl_triggered(request_id) | HITL | 人类介入时调用 |
| on_hitl_responded(request_id, response) | HITL | 人类响应时调用 |

### 3.2 M02：SceneLibrary（场景库）

**职责**：存储和管理可复用的标准化解决方案模板，驱动任务的动态实例化和执行时变更。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 场景存储 | 存储场景定义（fullset 三层结构） |
| 场景匹配 | 根据目标描述推荐最合适场景 |
| 实例化裁剪 | 根据已知信息从全集中裁剪初始 Goal |
| 条件触发 | 执行时检测条件满足，创建对应 Project/Task |
| HITL 定义 | 管理场景中的 HITL 节点定义 |
| 场景版本 | 版本管理、版本对比、反馈回流 |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |
|------|------|------|------|
| 场景列表 | REST | GET /api/v1/scenarios | 获取场景列表 |
| 场景详情 | REST | GET /api/v1/scenarios/{id} | 获取场景详情 |
| 场景匹配 | REST | POST /api/v1/scenarios/match | 根据目标匹配场景 |
| 场景实例化 | REST | POST /api/v1/scenarios/{id}/instantiate | 实例化为 Goal |
| 场景创建 | REST | POST /api/v1/scenarios | 创建场景 |
| 场景更新 | REST | PATCH /api/v1/scenarios/{id} | 更新场景 |

**内部接口**：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| get_fullset(scenario_id) | ReinsCore | 获取场景全集用于实例化 |
| evaluate_conditions(goal_id) | ReinsCore | 评估条件是否满足 |
| get_projects_for_goal(goal_id) | ReinsCore | 获取 Goal 对应的 Project 列表 |

### 3.3 M03：AgentRegistry（智能体管理）

**职责**：管理所有注册到 Nexus 的 Agent，提供能力注册、健康状态跟踪、负载管理。

**核心功能**：

| 功能 | 说明 |
|------|------|
| Agent 注册 | 接收 Agent 的注册请求，写入注册表 |
| Agent 注销 | 接收注销请求，标记离线 |
| 心跳处理 | 接收心跳，更新 last_heartbeat、load、status |
| 状态管理 | online / busy / offline / stale 状态跟踪 |
| 能力同步 | 同步 Agent 能力变更到调度引擎 |
| 负载评估 | 基于心跳数据计算负载评分 |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |
|------|------|------|------|
| Agent 注册 | REST | POST /api/v1/agents/register | Agent 注册 |
| Agent 注销 | REST | POST /api/v1/agents/unregister | Agent 注销 |
| Agent 心跳 | REST | POST /api/v1/agents/heartbeat | Agent 心跳 |
| Agent 列表 | REST | GET /api/v1/agents | 获取 Agent 列表 |
| Agent 发现 | REST | GET /api/v1/agents/discover | 按能力查询 Agent |
| Agent 详情 | REST | GET /api/v1/agents/{id} | 获取 Agent 详情 |

**内部接口**：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| get_online_agents() | ReinsCore | 获取所有在线 Agent |
| get_agent_by_capability(capability_tags) | ReinsCore | 按能力查询 Agent |
| update_agent_load(agent_id, load) | ReinsCore | 更新 Agent 负载 |
| mark_agent_offline(agent_id) | 健康检查 | Agent 离线时调用 |

### 3.4 M04：GrASP（任务分解）

**职责**：将高层用户目标分解为可执行的 Task 序列，是 Reins 与 Grasp 的集成模块，处理认知相关调用。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 意图理解 | 调用 Grasp 分析用户目标，提取意图和领域上下文 |
| 任务分解 | 基于场景模板 + Grasp 认知生成任务列表 |
| 认知抽取 | 任务派发前，抽取相关认知上下文随任务下发 |
| 认知回流 | 任务完成后，将执行经验反馈给 Grasp |
| 降级策略 | Grasp 不可用时返回降级响应 |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |
|------|------|------|------|
| 意图理解 | REST | POST /api/v1/grasp/intent | 意图理解 |
| 认知抽取 | REST | POST /api/v1/grasp/dispatch-cognition | 任务派发时认知抽取 |
| 认知回流 | REST | POST /api/v1/grasp/feedback | 认知回流 |
| 智能体匹配 | REST | POST /api/v1/grasp/agent-match | 智能体能力匹配 |

**内部接口**：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| understand_intent(goal_text) | ReinsCore | 目标创建时调用 |
| decompose_to_tasks(goal_id) | ReinsCore | 目标激活时调用 |
| get_cognitions_for_task(task_id) | ReinsCore | 任务派发前调用 |
| send_feedback(task_id, result) | EvoAgent | 任务完成时调用 |

### 3.5 M05：EvoAgent（验证仲裁）

**职责**：对 Task 执行结果进行验证，对连续失败的任务触发人工裁决，并驱动 Agent 能力进化。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 结果验证 | 对 Task 执行结果进行验证（编译/API/页面检查） |
| 争议裁决 | 连续 3 次验证失败时触发人工裁决 |
| 能力进化 | 根据执行结果更新 Agent 能力标签权重 |
| 经验提取 | 从成功执行中提取可复用的经验模式 |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |
|------|------|------|------|
| 验证结果 | REST | POST /api/v1/tasks/{id}/verification | 提交验证结果 |
| 裁决提交 | REST | POST /api/v1/tasks/{id}/ruling | 提交裁决结果 |
| 裁决列表 | REST | GET /api/v1/human-review/pending | 获取待裁决列表 |
| 批量裁决 | REST | POST /api/v1/human-review/batch-ruling | 批量裁决 |

**内部接口**：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| verify_task(task_id) | ReinsCore | Task 完成后调用验证 |
| trigger_ruling(task_id) | ReinsCore | 连续失败时触发裁决 |
| update_tag_weights(agent_id, task_result) | CapabilityTag | 验证通过后更新权重 |

| 响应处理 | 接收人类响应，恢复调度流程 |
| 超时处理 | 超过设定的超时时间执行预设的默认行为 |

**对外接口**：

| 接口 | 类型 | 路径 | 说明 |

**内部接口**：

| 接口 | 调用方 | 说明 |

### 3.8 M08：A2A Hub（Agent 间通信中枢）

**职责**：作为 Agent 间消息路由的核心枢纽，负责消息路由、状态同步、事件分发。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 消息路由 | 根据消息类型路由到目标 Agent |
| 状态同步 | 实时推送执行状态给订阅方 |
| 事件分发 | 基于事件类型进行发布/订阅分发 |
| 直连建立 | Agent 发现后建立 A2A 直连 |

**部署方式**：

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| 集中式 | 中小型系统 | 单实例，架构简单 |
| 集群式 | 企业级应用 | 负载均衡 + 多实例 |
| 事件驱动 | 超大规模 | Kafka/RabbitMQ 消息队列 |

**内部接口**：

| 接口 | 调用方 | 说明 |
|------|--------|------|
| route_message(message) | Agent | 发送消息 |
| subscribe(event_type, callback) | ReinsCore | 订阅事件 |
| discover_agent(agent_id) | AgentRegistry | Agent 发现 |

## 4. 模块边界

### 4.1 内部接口 vs 外部接口

| 模块 | 内部接口 | 外部接口 |
|------|---------|---------|
| ReinsCore | scheduler.tick()、on_task_completed()、on_task_failed() | REST API、事件回调 |
| SceneLibrary | get_fullset()、evaluate_conditions() | REST API |
| AgentRegistry | get_online_agents()、get_agent_by_capability() | REST API |
| GrASP | understand_intent()、decompose_to_tasks() | REST API |
| EvoAgent | verify_task()、update_tag_weights() | REST API |
| HITL | pause_for_hitl()、resume_after_hitl() | REST API |
| CapabilityTag | match_agent()、update_weight() | REST API |
| A2A Hub | route_message()、subscribe() | Agent SDK |

**定义**：
- **内部接口**：模块间直接调用，不经过网络
- **外部接口**：通过 HTTP/REST 对外暴露

### 4.2 模块边界规则

| 规则 | 说明 |
|------|------|
| 调度归 ReinsCore | 所有调度决策由 ReinsCore 做出，其他模块不直接操作调度队列 |
| 认知归 GrASP | 所有与 Grasp 的交互通过 GrASP 模块，不允许跨模块直调 Grasp |
| 标签归 CapabilityTag | Agent 能力查询必须通过 CapabilityTag，不允许跨模块直接访问 Agent 标签 |
| 场景归 SceneLibrary | 所有场景相关操作必须通过 SceneLibrary，不允许跨模块直接访问场景数据 |
| HITL 归 HITL | 所有人机协同逻辑必须通过 HITL 模块，不允许跨模块直接操作 HumanInputRequest |

## 5. 数据流向

### 5.1 主数据流

```
用户输入目标
     │
     ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ GrASP       │────▶│ SceneLibrary │────▶│ ReinsCore   │
│ 意图理解     │     │ 场景匹配     │     │ 目标创建    │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
     ┌─────────────────────────────────────────┘
     │ ReinsCore 创建 Goal/Project/Task
     ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ CapabilityTag│────▶│ AgentRegistry│────▶│ A2A Hub     │
│ 能力匹配     │     │ Agent 发现   │     │ 任务派发    │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
     ┌─────────────────────────────────────────┘
     │ OpenClaw Agent 执行 Task
     ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ EvoAgent    │────▶│ CapabilityTag│────▶│ GrASP       │
│ 验证+裁决   │     │ 权重更新     │     │ 认知回流    │
└─────────────┘     └─────────────┘     └─────────────┘
     │
     │ 验证失败或 HITL
     ▼
┌─────────────┐
│ HITL        │
│ 人机协同    │
└─────────────┘
```

### 5.2 关键数据流

| 数据流 | 起点 | 终点 | 说明 |
|--------|------|------|------|
| 目标创建 | GrASP | ReinsCore | 意图理解后创建 Goal |
| 场景实例化 | SceneLibrary | ReinsCore | 从场景全集裁剪 Goal |
| 任务派发 | ReinsCore | AgentRegistry | 查询可用 Agent |
| Agent 匹配 | CapabilityTag | ReinsCore | 返回最佳匹配 Agent |
| 任务执行 | A2A Hub | OpenClaw Agent | 派发任务给 Agent |
| 结果验证 | EvoAgent | ReinsCore | 验证通过/失败回调 |
| 权重更新 | EvoAgent | CapabilityTag | 验证通过后更新标签权重 |
| 认知回流 | GrASP | Grasp | 任务完成后反馈经验 |
| HITL 触发 | ReinsCore | HITL | 遇到需要人类介入的节点 |
| HITL 恢复 | HITL | ReinsCore | 人类响应后恢复调度 |

### 6.2 外部系统依赖

### 6.3 循环依赖检查

| 依赖路径 | 是否循环 | 说明 |
|---------|---------|------|
| ReinsCore → CapabilityTag → ReinsCore | 是 | 必须单向：ReinsCore 依赖 CapabilityTag，CapabilityTag 不依赖 ReinsCore |
| ReinsCore → EvoAgent → ReinsCore | 是 | 必须单向：ReinsCore 依赖 EvoAgent 回调，但 EvoAgent 不依赖 ReinsCore |
| ReinsCore → SceneLibrary → ReinsCore | 否 | ReinsCore 调用 SceneLibrary，SceneLibrary 不依赖 ReinsCore |

**模块设计原则**：
- ReinsCore 处于依赖中心，其他模块被 ReinsCore 调用
- 各模块之间不允许循环依赖
- 外部服务（Grasp、vLLM）作为基础设施，不参与模块依赖图
