## 1. 概述

本文档定义 Nexus 智能体协同驾驭平台的核心概念模型和术语体系，为概要设计和详细设计提供统一的概念基础。

## 2. 核心实体关系

### 2.1 实体关系总览

Nexus 平台的核心实体关系如下，实体之间通过包含、引用、绑定等关系相互关联：

```
┌──────────────────────────────────────────────────────────────────┐
│                        核心实体关系图                              │
│                                                                  │
│  Goal（战略目标）                                                 │
│    │                                                              │
│    │ 包含（1:N）                                                  │
│    ▼                                                              │
│  Project（战略）                                                  │
│    │                                                              │
│    │ 包含（1:N）                                                  │
│    ▼                                                              │
│  Task（战术）                                                     │
│    │                                                              │
│    │ 引用（N:1）                                                  │
│    ▼                                                              │
│  Scenario（场景模板）                                             │
│    │                                                              │
│    │ 实例化（N:1）                                                 │
│    ▼                                                              │
│  ScenarioInstance（场景实例）                                      │
│                                                                  │
│  ──────────────────────────────────────────────────────────────  │
│                                                                  │
│  Agent（智能体）                                                  │
│    │                                                              │
│    │ 绑定（N:M）                                                  │
│    ▼                                                              │
│  Skill（技能）                                                    │
│    │                                                              │
│    │ 提供（N:M）                                                  │
│    ▼                                                              │
│  CapabilityTag（能力标签）                                        │
│                                                                  │
│  ──────────────────────────────────────────────────────────────  │
│                                                                  │
│  Task ──引用──▶ Scenario                                          │
│  Goal ──引用──▶ Scenario                                         │
│  Goal ──属于──▶ Project ──属于──▶ Task                          │
│  Agent ──绑定──▶ Skill ──提供──▶ CapabilityTag                   │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 目标-项目-任务层级

Goal（战略目标）包含 Project（战略），Project 包含 Task（战术），三者构成层级包含关系：

```
Goal（战略目标）
  │
  │  1:N，包含关系
  │  一个 Goal 可以包含多个 Project
  │
  ▼
Project（战略）
  │
  │  1:N，包含关系
  │  一个 Project 可以包含多个 Task
  │
  ▼
Task（战术）
  │
  │  N:1，引用关系
  │  Task 引用 Scenario 中的任务模板
  │
  ▼
Scenario（场景模板）
  │
  │  N:1，实例化关系
  │  一个 Scenario 可以实例化为多个 Goal
  │
  ▼
ScenarioInstance（场景实例）
```

### 2.3 场景与目标的关系

**场景（Scenario）是全集，目标（Goal）是子集**。场景包含所有可能要做的事，目标是根据现场信息从全集中裁剪出的实际执行路径：

| 维度 | 场景（Scenario） | 目标（Goal） |
|------|----------------|-------------|
| 类比 | PMBOK 全书 | 按 PMBOK 启动的一个实际项目 |
| 性质 | 方法论/框架 | 实际执行实例 |
| 内容 | 全流程 + 所有分支 + 所有决策点 | 裁剪后的实际执行路径 |
| 状态 | 静态的，可复用的知识资产 | 动态的，有生命周期的执行实例 |
| 关系 | 全集 | 子集（引用场景全集的一部分） |

### 2.4 Agent-Skill-CapabilityTag 关系

Agent（智能体）绑定 Skill（技能），Skill 提供 CapabilityTag（能力标签），CapabilityTag 是 Agent 能力的四维描述：

```
Agent（智能体）
  │
  │  绑定（N:M）
  │  一个 Agent 可以绑定多个 Skill
  │
  ▼
Skill（技能）
  │
  │  提供（N:M）
  │  一个 Skill 可以提供多个能力标签
  │
  ▼
CapabilityTag（能力标签）
  │
  │  四维标签体系
  │  business / professional / technical / management
  │
  ▼
用于 Task 的能力匹配
```

## 3. 核心实体定义

### 3.1 Goal（战略目标）

**定义**：用户创建的最高层任务载体，代表一个完整的业务目标。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符 |
| name | TEXT | 目标名称 |
| description | TEXT | 目标描述 |
| status | ENUM | draft / in_progress / paused / completed / failed |
| scenario_id | TEXT FK | 引用的场景模板 |
| capability_tags | JSON | 目标级能力需求（四维标签） |
| mode | ENUM | normal / exploration / optimization |
| verifier_agent | TEXT | 负责验证的 Agent |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 3.2 Project（战略）

**定义**：Goal 的子集，代表一个独立的阶段性目标。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符 |
| goal_id | TEXT FK | 所属 Goal |
| name | TEXT | 项目名称 |
| type | ENUM | mandatory（强制）/ conditional（条件触发） |
| status | ENUM | created / active / paused / completed / failed |
| condition | JSON | 条件分支定义（conditional 类型） |
| capability_tags | JSON | 项目级能力需求 |
| depends_on | JSON | 依赖的其他 Project |
| created_at | TIMESTAMP | 创建时间 |

### 3.3 Task（战术）

**定义**：Project 的子集，代表一个具体的可执行任务单元。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符 |
| project_id | TEXT FK | 所属 Project |
| goal_id | TEXT FK | 所属 Goal |
| name | TEXT | 任务名称 |
| description | TEXT | 任务描述 |
| status | ENUM | todo / in_progress / paused / done / failed / timeout |
| priority | ENUM | high / medium / low |
| type | ENUM | mandatory / conditional |
| assigned_agent | TEXT | 分配的 Agent ID |
| capability_tags | JSON | 任务级能力需求 |
| verification_rules | JSON | 验收规则 |
| verification_cycle | INT | 当前验证轮次 |
| max_verification_cycle | INT | 最大验证轮次（默认 3） |
| next_step | JSON | 下一步可选任务 |
| created_at | TIMESTAMP | 创建时间 |
| started_at | TIMESTAMP | 开始时间 |
| completed_at | TIMESTAMP | 完成时间 |

### 3.4 Scenario（场景模板）

**定义**：可复用的标准化解决方案模板，包含完整的问题应对框架。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符 |
| name | TEXT | 场景名称 |
| category | ENUM | earthquake / fire / chemical / flood / general |
| description | TEXT | 场景描述 |
| level | ENUM | goal / project |
| status | ENUM | active / archived / draft / deprecated |
| source | ENUM | manual / ai_generated / cognitive_derived / execution_derived |
| trust_level | ENUM | high / medium / low |
| fullset | JSON | 全集标签 + 所有 Project + 所有 Task（含条件） |
| template_dag | JSON | 模板 DAG |
| versions | JSON | 版本历史 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 3.5 Agent（智能体）

**定义**：注册到 Nexus 的智能体实例，运行在 OpenClaw 之上。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符（对应 OpenClaw Agent ID） |
| name | TEXT | Agent 名称（刚子/谷子/麻子/蚊子/扣子） |
| status | ENUM | online / busy / offline / stale |
| capability_tags | JSON | Agent 的四维能力标签 |
| tag_weights | JSON | 标签权重映射 |
| load | INT | 当前负载（0-100） |
| max_concurrent_tasks | INT | 最大并发任务数 |
| last_heartbeat | TIMESTAMP | 最后心跳时间 |
| registered_at | TIMESTAMP | 注册时间 |

### 3.6 Skill（技能）

**定义**：Agent 可调用的具体技能单元，是 Agent 与能力标签的中间层。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 唯一标识符 |
| name | TEXT | 技能名称 |
| description | TEXT | 技能描述 |
| agent_id | TEXT FK | 所属 Agent |
| capability_tags | JSON | 该技能提供的能力标签 |
| enabled | BOOLEAN | 是否启用 |

### 3.7 CapabilityTag（能力标签）

**定义**：描述 Agent 能力的四维标签体系，是智能体匹配的标准化语言。

**数据结构**：

```json
{
  "business": ["emergency_response", "rescue_coordination"],
  "professional": ["incident_command", "risk_assessment", "report_writing"],
  "technical": ["python", "fastapi", "postgresql"],
  "management": ["coordination", "task_dispatch", "grasp_read", "grasp_write"]
}
```

**四维说明**：

| 维度 | 含义 | 回答问题 | 衰减周期 |
|------|------|---------|---------|
| business | 业务能力 | 懂不懂这个行业 | 90 天 |
| professional | 专业能力 | 能做什么专业工作 | 60 天 |
| technical | 技术能力 | 用什么工具/语言/框架 | 30 天 |
| management | 管理能力 | 会不会用 Nexus 系统 | 持续有效 |

## 4. 状态机定义

### 4.1 Goal 状态机

```
  [draft]
     │ activate()
     ↓
  [in_progress]
     │         │
     │ pause() │ resume()
     ↓         │
  [paused] ───┘
     │
     │ complete()
     ↓
  [completed]
     │
     │ fail()
     ↓
  [failed]
```

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| draft | in_progress | activate() |
| in_progress | paused | pause() |
| paused | in_progress | resume() |
| in_progress | completed | 所有 Project 完成 |
| in_progress | failed | 不可恢复的错误 |

### 4.2 Project 状态机

```
  [created]
     │ activate()
     ↓
  [active]
     │         │
     │ pause() │ resume()
     ↓         │
  [paused] ───┘
     │
     │ complete()
     ↓
  [completed]
     │
     │ fail()
     ↓
  [failed]
```

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| created | active | activate() |
| active | paused | pause() |
| paused | active | resume() |
| active | completed | 所有 Task 完成 |
| active | failed | 不可恢复的错误 |

### 4.3 Task 状态机

```
  [todo]
     │ dispatch()（scheduler 派发）
     ↓
  [in_progress]
     │          │          │          │
     │ pause()  │ complete()│ fail()   │ timeout()
     ↓          ↓          ↓          ↓
  [paused]   [done]    [failed]   [timeout]
     │
     │ resume()
     ↓
  [todo]（重新派发）
```

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| todo | in_progress | scheduler 派发 |
| in_progress | paused | pause() |
| paused | todo | resume()（重新派发） |
| in_progress | done | 验证通过 |
| in_progress | failed | 验证失败（达到最大轮次） |
| in_progress | timeout | 执行超时 |

### 4.4 Scenario 状态机

```
  [draft]
     │ publish()
     ↓
  [active]
     │         │
     │ archive()│ deprecate()
     ↓         ↓
  [archived] [deprecated]
```

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| draft | active | 发布上线 |
| active | archived | 归档（不再使用） |
| active | deprecated | 废弃（有新版本替代） |

## 5. 生命周期

### 5.1 Goal 生命周期

```
创建 → 分解 → 激活 → 执行 → 完成/失败
  │      │       │       │
  │      │       │       └── Goal 所有 Project 完成（或不可恢复失败）
  │      │       │           状态变为 completed / failed
  │      │       │           触发认知回流到 Grasp
  │      │       │
  │      │       └── Goal 激活，开始调度 Project
  │      │           Project 按场景预设顺序执行
  │      │           执行中可动态增减 Project/Task
  │      │
  │      └── Goal 激活后自动分解为 Project/Task
  │          或由用户手动创建
  │
  └── 用户输入目标描述，选择或自动匹配场景
      创建 Goal 记录，状态为 draft
```

**关键阶段**：

| 阶段 | 说明 | 产出 |
|------|------|------|
| 创建 | 用户输入目标，选择场景模板 | Goal 记录（draft） |
| 分解 | 从场景全集中裁剪初始 Project/Task | Project/Task 列表 |
| 激活 | Goal 状态变为 in_progress | 开始调度循环 |
| 执行 | Project/Task 由 Agent 执行，验证通过 | 执行结果 |
| 完成 | 所有 Project 完成 | Goal 状态变为 completed |

### 5.2 Project 生命周期

```
创建（来自场景实例化）
  │
  │ type = mandatory → 直接激活
  │ type = conditional → 等待条件满足
  │
  ▼
激活（active）
  │
  │ 暂停（pause）←─ 来自 Goal 暂停或手动暂停
  │
  ▼
执行中（active）
  │
  │ 条件不满足时跳过（conditional）
  │ 条件满足时创建子 Project
  │
  ▼
完成（completed）—— 所有子 Task 完成
```

**关键规则**：
- Project 暂停时，级联暂停所有 in_progress 的 Task
- Project 完成时，如果所有 Task 都完成或跳过，状态变为 completed
- Project 失败时，向上触发 Goal 的失败判断

### 5.3 Task 生命周期

```
创建（来自 Project 或场景实例化）
  │
  ▼
待派发（todo）
  │
  │ scheduler tick + 能力匹配 → 派发
  │
  ▼
执行中（in_progress）
  │
  │ 暂停（pause）←─ 来自 Project 暂停或手动暂停
  │ Agent 通过心跳领取任务
  │
  ▼
验证（verifying）—— EvoAgent 执行验证
  │
  │ 验证通过 → done
  │ 验证失败且未达最大轮次 → 重试（in_progress）
  │ 验证失败且达到最大轮次 → disputed（触发人工裁决）
  │
  ▼
完成（done）或 争议（disputed）或 失败（failed）
```

**关键规则**：
- Task 是唯一的执行单元，所有工作最终落到 Task 级别
- Task 暂停时，通过心跳机制通知 Agent 停止执行
- Task 重新派发时，从头开始，不保留中间状态

### 5.4 Scenario 生命周期

```
定义 → 发布 → 活跃使用 → 归档/废弃
  │       │       │
  │       │       └── 场景在系统中持续被使用
  │       │           每次执行产生统计数据
  │       │           高频路径反馈优化场景定义
  │       │
  │       └── 场景审核测试通过后发布
  │           状态变为 active
  │
  └── 场景蓝图编写
      三种来源：纯人工定义 / 认知库推导 / 执行记录提炼
```

**关键规则**：
- 场景定义后经审核才能发布
- 场景执行后产生统计数据，用于优化场景
- 场景可以迭代更新版本（v1.0 → v1.1）
- 旧版本场景可保留，已执行的 Goal 仍引用原版本

### 5.5 Agent 生命周期

```
注册 → 心跳保活 → 执行任务 → 注销
  │        │          │
  │        │          └── Agent 执行 Task
  │        │              每次心跳上报执行结果
  │        │              触发标签权重更新
  │        │              触发认知回流
  │        │
  │        └── 定期发送心跳（30-300秒间隔）
  │            心跳超时不响应则标记 offline
  │
  └── Agent 启动时向 Nexus 注册
      提交能力列表和状态信息
```

**关键规则**：
- Agent 离线超过阈值（默认 300 秒）标记为 offline
- Agent 恢复后自动重新注册，继续领取任务
- Agent 的 capability_tags 权重通过执行行为持续更新

## 6. 核心术语表

| 术语 | 英文 | 一句话解释 |
|------|------|---------|
| Goal | 战略目标 | 用户创建的最高层任务载体，对应一个完整的业务目标 |
| Project | 战略 | Goal 的子集，代表一个独立的阶段性目标 |
| Task | 战术 | Project 的子集，代表一个具体的可执行任务单元 |
| Scenario | 场景模板 | 可复用的标准化解决方案模板，包含完整的问题应对框架 |
| ScenarioInstance | 场景实例 | Scenario 实例化后的产物，关联到具体 Goal |
| Agent | 智能体 | 运行在 OpenClaw 之上的 AI 执行实体 |
| Skill | 技能 | Agent 可调用的具体能力单元 |
| CapabilityTag | 能力标签 | 描述 Agent 能力的四维标签体系 |
| Grasp | 悟 | 认知层，提供意图理解和知识管理 |
| Reins | 御 | 驾驭层，负责任务编排和智能体协同 |
| Evo | 化 | 进化层，负责经验提取和能力进化 |
| Reach | 达 | 生态层，负责方案库和技能复用 |
| Vigil | 鉴 | 安全层，负责行为审计和权限控制 |
| HITL | 人机协同 | Human-in-the-Loop，需要人类介入的节点 |
| A2A | Agent 间通信 | Agent-to-Agent 协议，智能体间直接通信 |
| fullset | 全集 | 场景包含的所有可能任务/分支/决策点 |
| capability_tags | 能力标签 | 四维标签（business/professional/technical/management） |

*本文档定义的概念模型是 Nexus 平台所有模块设计的基础。*
