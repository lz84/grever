## 测试策略总览

Nexus 系统采用四层测试金字塔，从底层到顶层覆盖范围递增、执行成本递增：

```
        ┌──────────────┐
        │  L4 E2E 测试 │  ← 用户场景全流程（Playwright）
        ├──────────────┤
        │ L3 API 测试  │  ← REST 端点契约（pytest + TestClient）
        ├──────────────┤
        │ L2 集成测试  │  ← 域内 + 跨域交互（Mock DB/LLM）
        ├──────────────┤
        │ L1 单元测试  │  ← 函数/类级别（pytest 纯函数）
        └──────────────┘
```

每层测试的目标和职责：
- **L1 单元测试**: 验证单个函数/类的内部逻辑正确性，不依赖外部服务
- **L2 集成测试**: 验证模块间协作，允许 Mock 外部依赖（DB/LLM/第三方服务）
- **L3 API 测试**: 验证 REST 端点的输入输出契约、错误码、响应格式
- **L4 E2E 测试**: 验证真实用户场景的完整业务流程，端到端无 Mock
