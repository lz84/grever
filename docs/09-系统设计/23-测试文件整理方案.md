# 测试文件整理方案

## 原则

1. **统一目录**：所有测试放在 `packages/server/tests/`，删除 `src/tests/`
2. **按五兄弟域分类**：测试文件按归属域分到对应子目录
3. **跨域/基础设施测试**：放 `tests/shared/`

## 目录结构

```

├── reins/               驾驭域测试
```

## 执行步骤

1. 合并 src/tests/ → tests/（按域分类）
2. 创建域子目录（reins/, grasp/, evo/, reach/, vigil/, shared/）
3. 移动测试文件到对应域目录
4. 更新 import 路径（所有测试文件中的 `from src.*` → `from *`）
5. 删除 src/tests/ 目录
6. 验证：pytest 能正常发现所有测试
