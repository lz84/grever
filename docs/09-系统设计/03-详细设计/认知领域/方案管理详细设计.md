## 1. 概述

### 1.1 设计目标

Nexus 方案管理模块是 GrASP（知）系统的核心组件，负责管理目标（Goal）的解决方案生命周期。核心设计原则为 **方案迭代优化**：通过多轮方案生成、比较、收敛判断，最终找到满足约束条件的最优解。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 方案创建 | 支持目标关联的多轮方案创建，自动去重 |
| 方案比较 | 多维度加权评分计算，标记最优方案 |
| 收敛检测 | 判断方案是否收敛（标准差/差距判断） |
| 约束管理 | 约束历史记录，迭代过程约束调整 |
| 收敛趋势 | 记录收敛数据，支持可视化展示 |

## 2. 核心流程

### 2.1 方案迭代完整链路

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           方案迭代完整链路                                      │
└──────────────────────────────────────────────────────────────────────────────┘

Step 1: 方案生成
  Agent 根据目标生成多个候选方案
         │
         ▼
Step 2: 方案存储
  POST /api/v1/solutions 创建方案
  自动去重：同 goal_id + round + parameters 不重复
         │
         ▼
Step 3: 方案评分
  POST /api/v1/solutions/compare 计算多维度评分
  更新 solutions.score / status / is_optimal
         │
         ▼
Step 4: 收敛判断
  check_convergence() 检查是否收敛
  收敛条件：
  - 标准差收敛：所有方案评分标准差 < 0.05
  - 差距够大：top1/top2 评分差 > 0.5
         │
         ▼
Step 5: 连续收敛检测
  跟踪连续收敛次数
  连续 3 次收敛 → 触发自动执行
         │
         ▼
Step 6: 完成
  收敛后标记最优方案 → 进入执行阶段
```

### 2.2 方案比较时序

```
时间轴
  │
  │  POST /solutions/compare
  │  ──────────────────────────
  ▼
T=0   compare_solutions() 开始
  │
  ├─ Step 1: 查询该 goal 下所有方案
  │    SELECT * FROM solutions WHERE goal_id = :gid
  │
  ├─ Step 2: 对每个方案计算多维度评分
  │    - 质量维度
  │    - 成本维度
  │    - 时间维度
  │    - 风险维度
  │
  ├─ Step 3: 加权求和得到总分
  │    score = Σ(dimension_score * weight)
  │
  ├─ Step 4: 更新数据库
  │    UPDATE solutions SET score = :score, is_optimal = ...
  │
  ├─ Step 5: 收敛判断
  │    check_convergence(goal_id, db)
  │
  └─ Step 6: 跟踪收敛连续次数
       _track_convergence_streak(goal_id, converged, db)
```

## 3. 数据模型

### 3.1 Solution 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 方案 ID，主键 |
| goal_id | VARCHAR(50) | 关联目标 ID |
| round | INT | 迭代轮次 |
| name | VARCHAR(200) | 方案名称 |
| parameters | TEXT | 方案参数字典（JSON） |
| dimensions | TEXT | 多维度评分（JSON） |
| score | FLOAT | 综合评分 |
| project_ids | TEXT | 关联项目列表（JSON） |
| task_ids | TEXT | 关联任务列表（JSON） |
| constraints | TEXT | 约束条件（JSON） |
| status | VARCHAR(20) | 状态：draft/active/completed |
| is_optimal | BOOLEAN | 是否最优方案 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 IterationConstraint 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 约束 ID |
| goal_id | VARCHAR(50) | 关联目标 ID |
| constraint_key | VARCHAR(100) | 约束键 |
| constraint_value | TEXT | 约束值（JSON） |
| round | INT | 约束生效轮次 |
| source | VARCHAR(50) | 来源：human/ai/consensus |
| created_at | DATETIME | 创建时间 |

## 4. 状态机

### 4.1 Solution 状态机

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| `draft` | `active` | 方案被激活参与比较 |
| `active` | `completed` | 方案完成评分 |
| `active` | `draft` | 方案需要重新修改 |
| `completed` | — | 终态 |

### 4.2 收敛状态

| 状态 | 说明 |
|------|------|
| `not_converged` | 方案尚未收敛，需继续迭代 |
| `std_converged` | 标准差收敛：std < 0.05 |
| `gap_converged` | 差距收敛：top1-top2 > 0.5 |

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 400 | 参数错误 | 返回详细错误信息 |
| 404 | 方案/目标不存在 | 返回 404 错误 |
| 409 | 方案重复 | 返回已有方案 ID |
| 500 | 服务器内部错误 | 记录日志，返回 500 |

### 7.2 异常处理

```python
# 方案不存在
if not row:

# 目标不存在
goal_row = db.execute(
    text("SELECT id FROM goals WHERE id = :id"),
    {"id": goal_id}

if not goal_row:
    raise HTTPException(status_code=404, detail="Goal not found")

# 约束依赖任务不存在
```

## 8. 与其他模块的关系

### 8.1 与 Goal 模块

- 方案归属于 Goal
- 方案的 project_ids/task_ids 关联 Project/Task
- Goal 的 mode 决定迭代模式（normal/exploration/optimization）

### 8.2 与 Task 模块

- 方案通过 task_ids 关联具体 Task
- 方案的执行结果影响 Task 状态

### 8.3 与 Iteration 模块

- 方案迭代由 Iteration 模块管理
- Iteration 通过 constraint_adjustments 影响方案生成
