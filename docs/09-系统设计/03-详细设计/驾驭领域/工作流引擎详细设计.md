## 1. 概述

### 1.1 设计目标

Nexus 工作流引擎是 Reins（御）系统的核心执行引擎，负责将工作流（Workflow）分解为可执行的步骤（Step），并通过 DAG 调度实现步骤的依赖管理和并行执行。核心设计原则为 **DAG 驱动执行**：基于依赖关系构建有向无环图，支持动态编辑和状态控制。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| DAG 构建 | 从步骤依赖关系构建有向无环图 |
| 并行调度 | 支持无依赖步骤的并行执行 |
| 状态控制 | 支持 pause/resume/cancel |
| 动态编辑 | 运行时动态添加/删除/重排步骤 |
| 循环检测 | 防止循环依赖 |
| 事件追踪 | tracker.py 事件集成 |

## 2. 核心流程

### 2.1 工作流执行完整链路

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           工作流执行完整链路                                   │
└──────────────────────────────────────────────────────────────────────────────┘

Step 1: 加载工作流
  WorkflowRepository.get(workflow_id)
  加载 Workflow + 所有 Steps
         │
         ▼
Step 2: 构建 DAG
  _build_dag() 从 steps 构建依赖图
         │
         ▼
Step 3: 循环依赖检测
  _detect_cycle() 检测环形依赖
  有环 → 抛出异常
         │
         ▼
Step 4: 并行分组
  _get_parallel_groups() 分层并行组
  同一组内步骤可并行执行
         │
         ▼
Step 5: 分层执行
  逐层执行每组步骤
  检查 pause/cancel 状态
         │
         ▼
Step 6: 失败处理
  某步骤失败 → 标记依赖它的步骤为 blocked
         │
         ▼
Step 7: 完成
  所有步骤成功 → status = completed
  有失败 → status = failed
```

### 2.2 调度执行时序

```
时间轴
  │
  │  execute(workflow_id)
  │  ─────────────────────
  ▼
T=0   加载工作流和步骤
  │
  ├─ 构建 DAG
  │    step.id → set(step.dependencies)
  │
  ├─ 检测循环依赖
  │    DFS 三色标记法
  │
  ├─ 获取并行组
  │    while remaining:
  │      layer = [step for step if deps ⊆ completed]
  │
  ├─ 执行并行组 1
  │    asyncio.gather(*[execute_step(s) for s in group])
  │    │
  │    ├─ step_1.execute()
  │    ├─ step_2.execute()
  │    └─ step_3.execute()
  │
  ├─ 执行并行组 2
  │    ...
  │
  └─ 完成
       workflow.status = COMPLETED / FAILED
```

## 4. 状态机

### 4.1 Workflow 状态机

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| `idle` | `running` | execute() 被调用 |
| `running` | `paused` | pause() 被调用 |
| `running` | `completed` | 所有步骤成功完成 |
| `running` | `failed` | 关键步骤失败 |
| `running` | `cancelled` | cancel() 被调用 |
| `paused` | `running` | resume() 被调用 |
| `paused` | `cancelled` | cancel() 被调用 |

**状态流转图**：

```
[idle]
   │ execute()
   ▼
[running] ◀──────────────┐
   │ pause()             │ resume()
   ▼                      │
[paused] ─────────────────┘
   │
   │ complete()（所有步骤成功）
   ▼
[completed]

  或

[running]
   │ 关键步骤失败
   ▼
[failed]

  或

[running] / [paused]
   │ cancel()
   ▼
[cancelled]
```

### 4.2 WorkflowStep 状态机

| 当前状态 | 可转换到 | 触发条件 |
|---------|---------|---------|
| `pending` | `running` | 开始执行 |
| `running` | `done` | 执行成功 |
| `running` | `failed` | 执行失败 |
| `running` | `skipped` | 被取消 |
| `pending` | `blocked` | 依赖步骤失败 |
| `pending` | `skipped` | 被取消 |

## 7. 错误处理

### 7.1 错误类型

| 错误类型 | 说明 | 处理方式 |
|---------|------|----------|
| `CircularDependencyError` | 检测到循环依赖 | 抛出 ValueError |
| `StepNotFoundError` | 步骤不存在 | 抛出 ValueError |
| `ExecutionStateError` | 状态不允许操作 | 抛出 RuntimeError |
| `NoExecutorError` | 无步骤执行器 | 记录警告，跳过步骤 |

### 7.2 异常处理流程

```python
# 循环依赖检测
cycle = self._detect_cycle(dag)
if cycle:
    raise ValueError(f"Circular dependency detected: {' -> '.join(cycle)}")

# 步骤不存在
if insert_position.startswith("after:"):
    after_step_id = insert_position[6:]
    after_step = next((s for s in existing_steps if s.id == after_step_id), None)
    if not after_step:
        raise ValueError(f"Step not found: {after_step_id}")

# 状态不允许
if self._execution_state != ExecutionState.RUNNING:
    raise RuntimeError("Cannot pause: workflow is not running")
```

## 8. 与其他模块的关系

### 8.1 与 Tracker 模块

- 工作流执行通过 `_emit_event()` 发送事件
- 事件类型：workflow_started/completed/failed/paused/resumed/cancelled
- 步骤事件：step_started/completed/failed/cancelled/blocked

### 8.2 与 TaskAssigner 模块

- 步骤自动分配 Agent：`get_task_assigner().assign(step)`
- 根据步骤的 capabilities 选择合适 Agent

### 8.3 与 Scheduler 模块

- 工作流执行可能触发 Task 创建
- 工作流完成状态影响 Goal/Project 状态
