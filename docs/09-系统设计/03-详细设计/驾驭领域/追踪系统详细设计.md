## 1. 概述

### 1.1 设计目标

Nexus 追踪系统是 Reins（御）系统的核心监控组件，负责记录和回放任务（Task）的执行过程。核心设计原则为 **全链路追踪**：从任务开始到完成，记录每一步操作和状态变更，支持执行回放和性能分析。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 追踪启动 | 创建追踪记录，记录任务开始 |
| 事件记录 | 记录执行过程中的关键事件 |
| 状态追踪 | 追踪任务状态变更 |
| 执行报告 | 生成任务执行报告 |
| 执行日志 | 记录 Agent 与系统的通信日志 |
| 步骤回放 | 支持执行步骤的历史回放 |

## 2. 核心流程

### 2.1 追踪完整链路

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           追踪完整链路                                         │
└──────────────────────────────────────────────────────────────────────────────┘

Step 1: 追踪启动
  POST /api/v1/traces
  创建 Trace 记录
  记录 started_at
         │
         ▼
Step 2: 执行过程中
  Agent 发送消息 → 记录 agent_input
  Agent 返回结果 → 记录 agent_output
  状态变更 → 记录 state_changed
         │
         ▼
Step 3: 步骤状态查询
  GET /api/v1/traces/{task_id}/step-status
  返回当前状态 + 步骤列表
         │
         ▼
Step 4: 执行日志查询
  GET /api/v1/traces/{task_id}/execution-logs
  返回完整通信日志
         │
         ▼
Step 5: 追踪完成
  POST /api/v1/traces/{task_id}/complete
  生成最终报告
  记录 success/error/duration
         │
         ▼
Step 6: 报告查询
  GET /api/v1/reports/{task_id}
  返回执行报告
```

### 2.2 追踪时序

```
时间轴
  │
  │  POST /traces（启动追踪）
  │  ──────────────────────────
  ▼
T=0   创建 Trace
  │    workflow_id, task_id, task_title, agent_id
  │
  ├─ 执行中
  │    ├─ agent_input: 接收任务指令
  │    ├─ state_changed: pending → running
  │    ├─ agent_output: 任务处理中
  │    ├─ agent_input: 系统消息
  │    └─ agent_output: 处理结果
  │
  ├─ GET /traces/{task_id}/step-status
  │    返回当前状态 + 步骤列表
  │
  ├─ GET /traces/{task_id}/execution-logs
  │    返回通信日志
  │
  └─ POST /traces/{task_id}/complete
       生成报告：success, duration, cognitions_used
```

## 4. 状态机

### 4.1 Trace 状态

| 状态 | 说明 |
|------|------|
| `pending` | 等待开始 |
| `running` | 执行中 |
| `completed` | 已完成 |
| `failed` | 失败 |
| `cancelled` | 已取消 |

### 4.2 事件类型

| 事件类型 | 说明 |
|----------|------|

| `agent_output` | Agent 输出 |
| `state_changed` | 状态变更 |
| `task_completed` | 任务完成 |
| `task_failed` | 任务失败 |

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 404 | Trace 不存在 | 返回 404 错误 |
| 500 | 服务器内部错误 | 记录日志，返回空结果 |

### 7.2 异常处理

```python
@router.get("/api/v1/traces/{task_id}")
def get_trace(task_id: str):
    trace = _get_tracker_sync().get_trace(task_id)
    report = _get_tracker_sync().get_report(task_id)
    
    if trace:
        return trace.get_summary()
    elif report:
        return report.to_dict()
    else:
        db_report = _get_tracker_sync().get_trace_report(task_id)
        if db_report:
            return db_report.to_dict()
        raise HTTPException(status_code=404, detail="Trace not found")
```

## 8. 与其他模块的关系

### 8.1 与 Workflow 模块

- 追踪与 Workflow 执行关联
- workflow_id 关联到工作流实例
- 步骤状态与 Workflow Step 状态联动

### 8.2 与 Task 模块

- 追踪记录 Task 的执行过程
- task_id 关联到具体 Task
- 执行日志记录 Agent 与 Task 的交互

### 8.3 与 Agent 模块

- agent_id 记录执行 Agent
- 心跳日志关联追踪记录
