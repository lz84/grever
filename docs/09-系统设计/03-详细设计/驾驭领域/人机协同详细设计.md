## 一、概述

### 1.1 文档目的

本文档是人机协同（HITL, Human-in-the-Loop）系统的详细设计文档，涵盖三种 HITL 模式（待裁决/disputed、待审批/approval、待输入/assist）的完整设计、触发条件、API 路由、状态转换、权限检查、超时处理以及前端交互流程。是架构设计文档 [17-manual-ruling-center-design.md](../../02-架构设计/17-manual-ruling-center-design.md) 的实现层补充。

### 1.2 命名规范

| 场景 | 前端文案 | 路由/路径 | 代码层（内部变量） |
|------|---------|----------|------------------|
| 连续验证失败的争议任务 | **人工裁决** | `/rulings` | `disputed` |
| 任务完成需要人工确认 | **人工审批** | `/rulings?tab=waiting` | `waiting_human` |
| 任务执行需要人类提供信息 | **人工协助** | `/human-assist` | `human_input` |

**原则**：用户可见文案全部用"人工"，路由用英文语义化，代码层变量名不动（成本太高，内部变量不影响用户体验）。

### 1.3 三种 HITL 模式总览

| 模式 | 触发条件 | 操作选项 | 状态转换 | 超时处理 |
|------|---------|---------|---------|---------|
| **待裁决** (disputed) | Task 验证连续 3 次失败 | approve / reject / request_changes | `disputed` → `done` / `in_progress` / `verifying` | 无超时（需人工主动处理） |
| **待审批** (approval) | Task 完成但需人工确认 | approve / reject | `waiting_human` → `done` / `in_progress` | 可配置 timeout_minutes |
| **待输入** (assist) | 场景条件分支需要人类输入 | submit / reject | 场景级暂停 → 恢复执行 | 可配置 timeout_minutes + timeout_action |

## 二、待裁决（disputed）模式

### 2.1 触发条件

Task 进入 `disputed` 状态的唯一条件：

```
Task 验证连续失败 3 次 → 自动进入 disputed 状态
```

具体流程：

```
Task 执行完成
     │
     ▼
验证规则执行（编译检查 / API检查 / 页面检查）
     │
     ├─ 全部通过 → Task.status = "done" ✅
     │
     └─ 有检查项失败
           │
           ▼
       重试计数 +1
           │
           ├─ 重试次数 < 3 → Task.status = "in_progress"（重新执行）
           │
           └─ 重试次数 = 3 → Task.status = "disputed" 🔴
                                → 创建 HumanReviewEntry
                                → 通知相关用户
```

### 2.2 裁决操作

进入 `disputed` 状态的 Task 有三种裁决操作：

| 操作 | action 值 | 状态转换 | 说明 |
|------|----------|---------|------|
| **通过** | `done` | `disputed` → `done` | 人工确认任务通过，标记完成 |
| **打回** | `in_progress` | `disputed` → `in_progress` | 打回给 Agent 重新执行 |
| **调整验收标准** | `verifying` | `disputed` → `verifying` | 修改验收规则后重新验证 |

### 2.3 API 路由

```python
# POST /api/v1/tasks/{task_id}/ruling
class TaskRulingRequest(BaseModel):
    ruling: str | None = None          # 裁决意见（可选）
    action: Literal["done", "in_progress", "verifying"]  # 裁决动作
    verification_rules: dict | None = None  # 调整后的验收规则（action=verifying 时必填）

async def task_ruling(
    task_id: str,
    request: TaskRulingRequest,
    current_user: User,
    db: Session
):
    task = await db.query(Task).filter(Task.id == task_id).first()
    if not task:
        raise HTTPException(404, "Task not found")

    # 权限检查
    if task.status != "disputed":
        raise HTTPException(400, f"Task status is {task.status}, not disputed")

    # 权限检查：required_role / assigned_to
    if not _has_ruling_permission(current_user, task):
        raise HTTPException(403, "No permission to rule this task")

    # 执行裁决
    task.status = request.action
    task.ruling_comment = request.ruling
    task.ruled_by = current_user.id
    task.ruled_at = now()

    if request.action == "verifying" and request.verification_rules:
        task.verification_rules = request.verification_rules
        task.retry_count = 0  # 重置重试计数

    # 记录裁决日志
    await _create_ruling_log(task_id, current_user.id, request, db)

    # 如果是 approve，同时关闭对应的 HumanReviewEntry
    if request.action == "done":
        entry = await db.query(HumanReviewEntry).filter(
            HumanReviewEntry.task_id == task_id,
            HumanReviewEntry.status == "pending"
        ).first()
        if entry:
            entry.status = "resolved"
            entry.resolved_by = current_user.id
            entry.resolved_at = now()

    await db.commit()
```

### 2.4 权限检查

```python
def _has_ruling_permission(user: User, task: Task) -> bool:
    """
    裁决权限检查：
    1. 管理员（admin role）→ 有权限
    2. Task 的 verifier_agent 对应的用户 → 有权限
    3. Task 的 assigned_to 指定的用户 → 有权限
    4. required_role 检查
    """
    # 管理员直接通过
    if user.role == "admin":
        return True

    # 检查 assigned_to
    if task.assigned_to and task.assigned_to == user.id:
        return True

    # 检查 required_role（如果 Task 定义了角色要求）
    if task.required_role:
        return user.role in task.required_role

    # 检查 verifier_agent
    if task.verifier_agent:
        verifier_user = _get_user_by_agent_id(task.verifier_agent)
        if verifier_user and verifier_user.id == user.id:
            return True

    return False
```

## 三、待审批（approval）模式

### 3.1 触发条件

Task 进入 `waiting_human`（待审批）状态的条件：

```
方式 1：场景模板中定义了 approval 类型的 condition
方式 2：Task 完成时 verification_rules 中包含 "requires_approval": true
方式 3：手动通过 API 将 Task 转为 waiting_human 状态
```

### 3.2 审批操作

| 操作 | action 值 | 状态转换 | 说明 |
|------|----------|---------|------|
| **通过** | `done` | `waiting_human` → `done` | 人工确认任务通过 |
| **拒绝** | `in_progress` | `waiting_human` → `in_progress` | 拒绝，打回重新执行 |

### 3.3 API 路由

```python
# POST /api/v1/tasks/{task_id}/approval
class ApprovalRequest(BaseModel):
    action: Literal["done", "in_progress"]  # approve / reject
    comment: str | None = None              # 审批意见
    reject_reason: str | None = None        # 拒绝原因（action=reject 时必填）

async def task_approval(
    task_id: str,
    request: ApprovalRequest,
    current_user: User,
    db: Session
):
    task = await db.query(Task).filter(Task.id == task_id).first()
    if not task:
        raise HTTPException(404, "Task not found")

    if task.status != "waiting_human":
        raise HTTPException(400, f"Task status is {task.status}, not waiting_human")

    # 拒绝时必填原因
    if request.action == "in_progress" and not request.reject_reason:
        raise HTTPException(400, "reject_reason is required when rejecting")

    # 权限检查
    if not _has_approval_permission(current_user, task):
        raise HTTPException(403, "No permission to approve this task")

    # 执行审批
    task.status = request.action
    task.approval_comment = request.comment
    task.reject_reason = request.reject_reason
    task.approved_by = current_user.id
    task.approved_at = now()

    await db.commit()
```

### 3.4 超时处理

待审批支持超时机制：

```python
# 定时任务：检查超时的审批
async def check_approval_timeouts(db: Session):
    """
    检查所有 waiting_human 状态的 Task
    如果超过 timeout_minutes 未处理，执行超时动作
    """
    TIMEOUT_CONFIG = {
        "high": 30,     # 高优先级 30 分钟
        "medium": 120,  # 中优先级 2 小时
        "low": 480,     # 低优先级 8 小时
    }

    pending = await db.query(Task).filter(
        Task.status == "waiting_human"
    ).all()

    for task in pending:
        priority = task.priority or "medium"
        timeout_min = TIMEOUT_CONFIG.get(priority, 120)

        elapsed = (now() - task.updated_at).seconds / 60
        if elapsed > timeout_min:
            # 超时处理： escalate（升级通知）
            await _escalate_approval(task, db)
```

超时动作（timeout_action）：

| timeout_action | 说明 |
|---------------|------|
| `escalate` | 升级通知（通知更高级别的审批人） |
| `auto_approve` | 自动通过（默认行为，不推荐） |
| `auto_reject` | 自动拒绝 |
| `none` | 无超时处理（默认值，需人工处理） |

## 四、待输入（assist）模式

### 4.1 触发条件

待输入（assist）由场景条件分支触发，当场景模板中的 `condition.type` 为 `human_input` 时：

```json
{
  "condition": {
    "type": "human_input",
    "input_type": "text",
    "prompt": "请输入预估泄漏量（立方米）：",
    "default": 0,
    "timeout_action": "use_default",
    "timeout_minutes": 60,
    "var_name": "estimated_leak_volume"
  }
}
```

### 4.2 提交操作

| 操作 | action 值 | 说明 |
|------|----------|------|
| **提交** | `submit` | 人类提供输入，注入到场景上下文 |
| **拒绝** | `reject` | 拒绝提供输入，按 timeout_action 处理 |

### 4.3 API 路由

```python
# POST /api/v1/human-input-requests/{request_id}/submit
class HumanInputSubmitRequest(BaseModel):
    action: Literal["submit", "reject"]
    value: str | int | float | bool | None = None  # 输入值（action=submit 时必填）
    comment: str | None = None

async def human_input_submit(
    request_id: str,
    request: HumanInputSubmitRequest,
    current_user: User,
    db: Session
):
    hir = await db.query(HumanInputRequest).filter(
        HumanInputRequest.id == request_id
    ).first()
    if not hir:
        raise HTTPException(404, "HumanInputRequest not found")

    if hir.status != "pending":
        raise HTTPException(400, f"Request status is {hir.status}, not pending")

    # 提交时必填 value
    if request.action == "submit" and request.value is None:
        raise HTTPException(400, "value is required when submitting")

    # 权限检查
    if not _has_assist_permission(current_user, hir):
        raise HTTPException(403, "No permission to submit this request")

    # 处理提交
    hir.status = "answered"
    hir.answered_by = current_user.id
    hir.answered_at = now()

    if request.action == "submit":
        hir.submitted_value = request.value
        hir.comment = request.comment

        # 注入到场景上下文
        await _inject_to_scenario_context(
            hir.goal_id, hir.var_name, request.value, db
        )

        # 触发场景条件引擎继续执行
        await _trigger_scenario_conditions(hir.goal_id, db)

    elif request.action == "reject":
        # 拒绝：执行 timeout_action
        timeout_action = hir.timeout_action or "use_default"
        if timeout_action == "use_default":
            hir.submitted_value = hir.default_value
            await _inject_to_scenario_context(
                hir.goal_id, hir.var_name, hir.default_value, db
            )
            await _trigger_scenario_conditions(hir.goal_id, db)
        elif timeout_action == "skip_project":
            await _skip_project(hir.project_id, db)
        elif timeout_action == "skip_task":
            await _skip_task(hir.task_id, db)

    await db.commit()
```

### 4.4 超时处理

待输入支持超时机制，超时后按 `timeout_action` 执行：

| timeout_action | 说明 |
|---------------|------|
| `use_default` | 使用默认值继续执行（默认） |
| `skip_project` | 跳过整个 Project |
| `skip_task` | 跳过当前 Task |
| `escalate` | 升级通知 |

```python
# 定时任务：检查超时的待输入请求
async def check_human_input_timeouts(db: Session):
    pending = await db.query(HumanInputRequest).filter(
        HumanInputRequest.status == "pending"
    ).all()

    for hir in pending:
        timeout_min = hir.timeout_minutes or 60
        elapsed = (now() - hir.created_at).seconds / 60

        if elapsed > timeout_min:
            # 执行超时动作
            timeout_action = hir.timeout_action or "use_default"
            if timeout_action == "use_default":
                hir.submitted_value = hir.default_value
                hir.status = "answered"
                hir.answered_by = "system_timeout"
                await _inject_to_scenario_context(
                    hir.goal_id, hir.var_name, hir.default_value, db
                )
            elif timeout_action == "skip_project":
                await _skip_project(hir.project_id, db)
                hir.status = "answered"
            elif timeout_action == "escalate":
                await _escalate_human_input(hir, db)

    await db.commit()
```

## 五、权限检查

### 5.1 权限模型

```
权限检查层级：

1. required_role（角色要求）
   - 如果 Task/HumanInputRequest 定义了 required_role 列表
   - 用户的 role 必须在列表中

2. assigned_to（指定处理人）
   - 如果 Task/HumanInputRequest 指定了 assigned_to
   - 只有该用户有权限处理

3. 管理员（admin）
   - 管理员拥有所有权限

4. 默认权限
   - 如果以上都没有定义，所有登录用户都有权限
```

### 5.2 权限检查函数

```python
def _check_permission(user: User, target: HumanReviewEntry | HumanInputRequest | Task) -> bool:
    """
    统一权限检查函数
    """
    # 管理员直接通过
    if user.role == "admin":
        return True

    # 检查 required_role
    required_role = getattr(target, "required_role", None)
    if required_role:
        if user.role not in required_role:
            return False

    # 检查 assigned_to
    assigned_to = getattr(target, "assigned_to", None)
    if assigned_to:
        if user.id != assigned_to:
            return False

    return True
```

### 5.3 权限配置示例

```python
# 创建 Task 时指定权限
task = Task(
    id="task-123",
    title="实现用户管理模块",
    required_role=["admin", "verifier"],  # 只有 admin 和 verifier 角色可裁决
    assigned_to="user-xxx",               # 指定 user-xxx 处理
    ...
)

# 创建 HumanInputRequest 时指定权限
hir = HumanInputRequest(
    id="req-001",
    goal_id="goal-xxx",
    required_role=["admin"],              # 只有 admin 可提交
    assigned_to="user-yyy",               # 指定 user-yyy 处理
    ...
)
```

## 六、超时处理机制

### 6.1 超时配置汇总

| 模式 | 超时配置 | 默认值 | 说明 |
|------|---------|--------|------|
| **待裁决** (disputed) | 无超时 | — | 需人工主动处理，不自动超时 |
| **待审批** (approval) | timeout_minutes | 按优先级：30/120/480 | 超时后执行 timeout_action |
| **待输入** (assist) | timeout_minutes | 60 | 超时后执行 timeout_action |

### 6.2 超时动作汇总

| timeout_action | 适用模式 | 说明 |
|---------------|---------|------|
| `escalate` | approval / assist | 升级通知（通知更高级别的审批人/处理人） |
| `use_default` | assist | 使用默认值继续执行 |
| `auto_approve` | approval | 自动通过 |
| `auto_reject` | approval | 自动拒绝 |
| `skip_project` | assist | 跳过整个 Project |
| `skip_task` | assist | 跳过当前 Task |
| `none` | 所有 | 无超时处理（默认值） |

### 6.3 超时检查定时任务

```python
# 每 60 秒执行一次
async def check_all_timeouts(db: Session):
    """
    统一超时检查任务
    检查所有 waiting_human 和 pending human_input 的超时
    """
    # 1. 检查待审批超时
    await check_approval_timeouts(db)

    # 2. 检查待输入超时
    await check_human_input_timeouts(db)

    # 3. 发送超时通知
    await _send_timeout_notifications(db)
```

## 八、add-hitl / terminate / takeover 端点设计

### 8.1 add-hitl 端点

用于手动为 Task 添加 HITL 节点（人工介入点）：

```python
# POST /api/v1/tasks/{task_id}/add-hitl
class AddHitlRequest(BaseModel):
    hitl_type: Literal["disputed", "waiting_human", "human_input"]
    required_role: list[str] | None = None    # 指定处理角色
    assigned_to: str | None = None            # 指定处理人
    timeout_minutes: int | None = None        # 超时时间
    timeout_action: str | None = None         # 超时动作
    prompt: str | None = None                 # 人工输入时的提示语
    input_type: str | None = None             # 输入类型（text/number/boolean）
    default_value: str | None = None          # 默认值

async def add_hitl(
    task_id: str,
    request: AddHitlRequest,
    current_user: User,
    db: Session
):
    task = await db.query(Task).filter(Task.id == task_id).first()
    if not task:
        raise HTTPException(404, "Task not found")

    # 权限检查：只有 admin 或 Task 的 verifier 可以添加 HITL
    if current_user.role != "admin" and task.verifier_agent != current_user.id:
        raise HTTPException(403, "No permission to add HITL")

    if request.hitl_type == "waiting_human":
        # 转为待审批
        task.status = "waiting_human"
        task.required_role = request.required_role
        task.assigned_to = request.assigned_to
        task.timeout_minutes = request.timeout_minutes or 120
        task.timeout_action = request.timeout_action or "escalate"

    elif request.hitl_type == "human_input":
        # 创建 HumanInputRequest
        hir = HumanInputRequest(
            goal_id=task.goal_id,
            project_id=task.project_id,
            task_id=task.id,
            prompt=request.prompt or "请提供必要信息",
            input_type=request.input_type or "text",
            default_value=request.default_value,
            timeout_minutes=request.timeout_minutes or 60,
            timeout_action=request.timeout_action or "use_default",
            required_role=request.required_role,
            assigned_to=request.assigned_to,
            status="pending"
        )
        db.add(hir)

    elif request.hitl_type == "disputed":
        # 直接转为争议状态
        task.status = "disputed"
        task.required_role = request.required_role
        task.assigned_to = request.assigned_to

    await db.commit()
```

### 8.2 terminate 端点

用于终止 HITL 请求（取消人工介入）：

```python
# POST /api/v1/hitl/{hitl_id}/terminate
class TerminateHitlRequest(BaseModel):
    reason: str  # 终止原因（必填）
    action: Literal["skip", "retry", "cancel"] = "skip"

async def terminate_hitl(
    hitl_id: str,
    request: TerminateHitlRequest,
    current_user: User,
    db: Session
):
    # 查找对应的 HITL 条目（可能是 Task 或 HumanInputRequest）
    entry = await _find_hitl_entry(hitl_id, db)
    if not entry:
        raise HTTPException(404, "HITL entry not found")

    # 权限检查
    if not _has_terminate_permission(current_user, entry):
        raise HTTPException(403, "No permission to terminate")

    if request.action == "skip":
        # 跳过：标记 HITL 为已终止，继续执行
        entry.status = "terminated"
        entry.terminate_reason = request.reason
        entry.terminated_by = current_user.id

        # 如果是 HumanInputRequest，注入默认值
        if isinstance(entry, HumanInputRequest):
            await _inject_to_scenario_context(
                entry.goal_id, entry.var_name, entry.default_value, db
            )
            await _trigger_scenario_conditions(entry.goal_id, db)

    elif request.action == "retry":
        # 重试：重新创建 HITL 请求
        entry.status = "terminated"
        entry.terminate_reason = request.reason
        await _recreate_hitl(entry, db)

    elif request.action == "cancel":
        # 取消：终止整个 Goal
        entry.status = "terminated"
        entry.terminate_reason = request.reason
        goal = await db.query(Goal).filter(Goal.id == entry.goal_id).first()
        if goal:
            goal.status = "terminated"

    await db.commit()
```

### 8.3 takeover 端点

用于接管 HITL 处理权（将处理权转移给其他用户）：

```python
# POST /api/v1/hitl/{hitl_id}/takeover
class TakeoverHitlRequest(BaseModel):
    target_user_id: str  # 接管目标用户 ID
    comment: str | None = None

async def takeover_hitl(
    hitl_id: str,
    request: TakeoverHitlRequest,
    current_user: User,
    db: Session
):
    entry = await _find_hitl_entry(hitl_id, db)
    if not entry:
        raise HTTPException(404, "HITL entry not found")

    # 权限检查：只有 admin 或当前 assigned_to 可以转移
    if current_user.role != "admin" and entry.assigned_to != current_user.id:
        raise HTTPException(403, "No permission to takeover")

    # 执行接管
    entry.assigned_to = request.target_user_id
    entry.takeover_comment = request.comment
    entry.takeover_by = current_user.id
    entry.takeover_at = now()

    # 通知新处理人
    await _notify_user(request.target_user_id, f"新的 HITL 请求已分配给你: {hitl_id}", db)

    await db.commit()
```

## 九、状态转换汇总

### 9.1 Task 状态转换（含 HITL）

```
   [todo]
      │ dispatch (scheduler)
      ↓
   [in_progress]
      │ complete (Agent 提交结果)
      ↓
   [verifying]
      │ verify_pass
      ↓
   [done] ✅
      │
      │ verify_fail × 3
      ↓
   [disputed] 🔴
      │ approve → [done]
      │ reject  → [in_progress]（重新执行）
      │ adjust  → [verifying]（调整验收标准后重新验证）
      │
      │ requires_approval = true
      ↓
   [waiting_human] 🟡
      │ approve → [done]
      │ reject  → [in_progress]（重新执行）
```

### 9.2 HumanInputRequest 状态转换

```
   [pending] 🟢
      │ submit (人类提供输入)
      ↓
   [answered]
      │ 注入上下文 → 触发场景条件引擎
      │
      │ reject (人类拒绝)
      ↓
   [answered]（按 timeout_action 处理）
      │
      │ timeout (超时)
      ↓
   [answered]（按 timeout_action 处理）
```
