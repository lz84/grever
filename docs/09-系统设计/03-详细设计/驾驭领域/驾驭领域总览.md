## 一、驾驭领域概述

驾驭领域是 Nexus 系统的核心执行引擎，负责任务从创建到完成的完整生命周期。

### 1.1 子领域清单

| # | 子领域 | 职责 | 对应文档 |
|---|--------|------|---------|
| 1 | 任务调度 | Goal/Project/Task 生命周期、DAG 调度、派发 | [任务调度详细设计](./任务调度详细设计.md) |
| 2 | 智能体注册 | Agent 注册、心跳、在线状态、负载管理 | [智能体注册详细设计](./智能体注册详细设计.md) |
| 3 | 人机协同 | HITL 三种交互模式、审批、通知、权限 | [人机协同详细设计](./人机协同详细设计.md) |
| 4 | 验证与仲裁 | 结果验证、争议裁决、方案对比 | [验证与仲裁详细设计](./验证与仲裁详细设计.md) |
| 5 | 能力标签 | 四维标签体系、匹配引擎、权重管理 | [能力标签详细设计](./能力标签详细设计.md) |

## 二、子领域交互关系

```
                    ┌──────────────┐
                    │   能力标签    │
                    │  (匹配引擎)   │
                    └──────┬───────┘
                           │ 提供匹配结果
                           ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  智能体注册   │◄───│   任务调度    │───►│  人机协同     │
│ (Agent信息)  │    │  (调度核心)    │    │  (拦截/审批)  │
└──────────────┘    └──────┬───────┘    └──────────────┘
       负载/在线            │ 执行结果
       状态                 ▼
                    ┌──────────────┐
                    │ 验证与仲裁    │
                    │  (验证/裁决)  │
                    └──────────────┘
```

### 2.1 核心流程

1. **任务派发**：调度核心查询智能体注册获取在线/负载信息 → 调用能力标签匹配引擎 → 派发任务
2. **任务拦截**：调度核心检查场景条件 → 触发人机协同 → 等待人类响应 → 继续/终止
3. **结果验证**：Agent 执行完成 → 验证与仲裁接收结果 → 判断通过/失败/争议
4. **权重调整**：验证结果影响能力标签权重 → 影响下次匹配结果

## 三、数据契约

### 3.1 调度核心 → 智能体注册

| 数据 | 格式 | 说明 |
|------|------|------|
| agent_id | string | 智能体唯一标识 |
| trigger_mode | string | 触发模式 (push/pull) |
| heartbeat_interval | int | 心跳间隔（秒） |

### 3.2 调度核心 → 能力标签

| 数据 | 格式 | 说明 |
|------|------|------|
| task_id | string | 任务唯一标识 |
| required_capabilities | JSON object | 四维能力需求 |
| context | JSON object | 上下文信息 |

### 3.3 验证与仲裁 → 能力标签

| 数据 | 格式 | 说明 |
|------|------|------|
| task_id | string | 任务唯一标识 |
| agent_id | string | 执行智能体 |
| verdict | string | 裁决结果 (pass/fail/disputed) |
| score_delta | float | 权重变化量 |
