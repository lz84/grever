## 1. 概述

### 1.1 设计目标

Nexus 任务调度引擎是 Reins（御）系统的核心执行引擎，负责将用户目标（Goal）分解为可执行任务（Task），并通过调度循环主动管理任务的生命周期。核心设计原则为 **Human on the Loop**：引擎主动执行，人类仅在关键节点介入。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 心跳驱动调度 | Agent 通过心跳触发任务分配 |
| 主动轮询 | 调度器定时扫描超时/离线任务 |
| 孤儿任务恢复 | 检测并回收离线 Agent 的任务 |
| DAG 依赖解锁 | 任务完成时自动解锁下游任务 |
| 结果验证 | 验证 Agent 上报的任务结果 |
| 状态持久化 | 所有状态变更写入 DB，保证可恢复性 |

## 2. 核心流程

### 2.1 完整执行链路

任务从创建到完成的完整流程：

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           任务完整执行链路                                    │
└──────────────────────────────────────────────────────────────────────────────┘

Step 1: Goal 创建
  用户输入高层目标
         │
         ▼
Step 2: Project 创建
  Goal 被分解为多个 Project（阶段）
         │
         ▼
Step 3: Task 创建
  每个 Project 包含多个 Task
  Task 创建时通过 depends_on 设置依赖关系
         │
         ▼
Step 4: 派发（调度器 + 心跳）
  ┌─────────────────────────────────────────────────┐
  │  调度器 tick（每 30 秒）                         │
  │  ─ 查询 todo/pending 且无 assigned_agent 的任务   │
  │  ─ 按能力匹配 + 负载均衡选择 Agent               │
  │  ─ 分配 assigned_agent → status = in_progress   │
  └─────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────┐
  │  Agent 心跳（POST /agents/{id}/heartbeat）      │
  │  ─ 调度器响应待领取任务列表                       │
  │  ─ Agent 领取后更新 assigned_agent              │
  └─────────────────────────────────────────────────┘
         │
         ▼
Step 5: 执行
  Agent 接收任务，执行具体操作
  执行结果通过 PATCH /tasks/{id} 上报
         │
         ▼
Step 6: 验证
  ┌─────────────────────────────────────────────────┐
  │  TaskValidator 验证结果                           │
  │  ─ 通过 → status = done                         │
  │  ─ 失败 → status = review_needed                │
  │  ─ 需重试 → status = todo（重新派发）           │
  └─────────────────────────────────────────────────┘
         │
         ▼
Step 7: 依赖解锁
  当任务完成时：
  ─ DependencyResolver 检查 task_dependencies 表
  ─ 解锁所有依赖该任务的下游任务（waiting → todo）
  ─ 检查 Project 是否所有任务都完成 → Project.status = done
         │
         ▼
Step 8: 完成
  所有 Task done → Goal.progress = 1.0 → Goal.status = completed
```

### 2.2 调度循环时序

```
时间轴
  │
  │  Scheduler tick（每 30 秒）
  │  ──────────────────────────────
  ▼
T=0   tick() 开始
  │
  ├─ Step 1: _scan_agent_health()
  │    查询 agents 表，检查 last_heartbeat
  │    更新 health_status：online → stale → offline
  │
  ├─ Step 2: _recover_timeout_tasks()
  │    查询 status = in_progress AND started_at < (now - 30min)
  │    更新为 timeout，并减少 Agent 的 current_tasks
  │
  ├─ Step 3: _recover_offline_tasks()
  │    查询 health_status = offline 的 Agent
  │    回收其所有 in_progress 任务 → status = todo
  │
  ├─ Step 4: _redistribute_recovered_tasks()
  │    查询 recovery_count > 0 且 retry_count < max_retries 的任务
  │    排除上次分配的 Agent，重新分配
  │
  ├─ Step 5: _assign_pending_tasks()
  │    查询 todo/pending 且无 assigned_agent 的任务
  │    按负载均衡分配给在线 Agent
  │
  ├─ Step 6: _unlock_dependencies()
  │    查询所有 waiting 状态任务
  │    检查其 depends_on 是否全部 done → 是则解锁为 todo
  │
  └─ Step 7: 更新统计
       SchedulerStats 记录本 tick 结果
```

## 3. 调度器设计

### 3.1 调度器架构

调度器位于 `reins/scheduler/core.py`，由以下组件构成：

```
NexusScheduler
  ├── stats: SchedulerStats              — 调度统计
  ├── health_manager: AgentHealthManager — Agent 健康度管理
  ├── task_recoverer: TaskRecoverer      — 任务回收
  ├── task_assigner: TaskAssigner        — 任务分配
  ├── dependency_resolver: DependencyResolver — 依赖解锁
  ├── result_verifier: ResultVerifier     — 结果验证
  └── project_executors: Dict[str, ProjectExecutor] — 项目执行器池
```

### 3.2 心跳驱动机制

Agent 通过定期心跳（默认每 30 秒）保持与调度器的连接：

```
Agent                                Nexus Server
  │                                      │
  │──── POST /agents/{id}/heartbeat ────▶
  │     { status, load, current_tasks }  │
  │                                      │
  │◀─── 200: { pending_tasks[...] } ─────│
  │     （调度器分配的待领取任务）         │
  │                                      │
  │──── GET /tasks/{id} ─────────────────▶
  │     （领取任务详情）                   │
  │                                      │
  │◀─── 200: { task details } ───────────│
```

**心跳处理流程**（`reins/api/_agents_heartbeat_routes.py`）：
1. 更新 `agents.last_heartbeat` 为当前时间
2. 如果 Agent 状态为 stale/offline → 恢复为 online
3. 重置 `consecutive_offline_count`
4. 返回待领取任务列表（`pending_tasks`）

### 3.3 轮询机制

调度器以固定间隔（`TICK_INTERVAL`，默认 30 秒）执行一次完整的调度循环：

```python
# reins/scheduler/core.py
class NexusScheduler:
    TICK_INTERVAL = 30  # 秒

    async def _tick_loop(self):
        while self._running:
            try:
                await self._tick()
            except Exception as e:
                logger.error(f"[Scheduler] Tick error: {e}")
            await asyncio.sleep(self.TICK_INTERVAL)
```

### 3.4 孤儿任务恢复

孤儿任务指：任务状态为 `in_progress` 但其 assigned Agent 已离线的情况。

**触发时机**：
1. Server 重启时（`_cleanup_stale_tasks`）
2. 调度器 tick 中（`_recover_offline_tasks`）

**处理逻辑**：

```python
# 1. Server 重启清理（_cleanup_stale_tasks）
UPDATE tasks
SET status = 'paused',
    paused_reason = 'orphan_on_restart'
WHERE status = 'in_progress'

# 2. Agent 离线回收（_recover_offline_tasks）
# 找到 health_status = 'offline' 的 Agent
SELECT id FROM agents WHERE health_status = 'offline'
# 回收其所有 in_progress 任务
UPDATE tasks
SET status = 'todo',
    assigned_agent = NULL,
    recovery_count = recovery_count + 1
WHERE assigned_agent = :agent_id
  AND status IN ('todo', 'pending', 'in_progress')

# 3. 超时回收（_recover_timeout_tasks）
UPDATE tasks
SET status = 'timeout',
    timeout_reason = '执行超时'
WHERE status = 'in_progress'
  AND started_at < (now - TASK_TIMEOUT_MINUTES)
```

**paused_reason 语义**：

| 值 | 含义 | 触发时机 |
|----|------|---------|
| `orphan_on_restart` | Server 重启导致孤儿 | Server 重启时 |
| `orphan_on_offline` | Agent 离线导致孤儿 | Agent 心跳超时 |
| `orphan_on_timeout` | 任务超时 | 执行超 30 分钟 |
| `human` | 人类主动暂停 | 人类调用 pause API |

## 5. DAG 执行

### 5.1 依赖关系存储

Task 的 DAG 依赖关系通过以下三处同步存储（`reins/api/tasks_crud.py`）：

| 存储位置 | 说明 | 用途 |
|---------|------|------|
| `tasks.depends_on` (JSON 列) | 当前任务的依赖列表 | 快速查询当前任务依赖 |
| `tasks.next_step` (JSON 列) | 依赖当前任务的下游任务 | 正向链接，便于画图 |
| `task_dependencies` (关系表) | 关系型依赖记录 | 执行链路读取 |

**同步函数**：`_sync_depends_on_all(task_id, new_deps, old_deps, db, is_create)`

```python
def _sync_depends_on_all(task_id: str, new_deps: list, old_deps: list, db: Session, is_create: bool = False):
    """
    同步 depends_on 变更到三个地方：
    1. tasks.depends_on JSON 列
    2. tasks.next_step JSON 列（父任务的正向链接）
    3. task_dependencies 关系表
    """
    # 1. 同步 task_dependencies 关系表
    removed_deps = set(old_deps) - set(new_deps)
    added_deps = set(new_deps) - set(old_deps)

    for dep_id in removed_deps:
        db.query(TaskDependency).filter_by(task_id=task_id, dependency_id=dep_id).delete()

    for dep_id in added_deps:
        parent = db.query(Task).filter(Task.id == dep_id).first()
        if not parent:
            raise HTTPException(400, f"Dependency task '{dep_id}' not found")
        db.add(TaskDependency(task_id=task_id, dependency_id=dep_id))

    # 2. 同步 next_step（父任务的正向链接）
    for dep_id in removed_deps:
        parent = db.query(Task).filter(Task.id == dep_id).first()
        if parent:
            parent_next = _parse_json_list(parent.next_step)
            parent_next = [x for x in parent_next if x != task_id]
            parent.next_step = json.dumps(parent_next) if parent_next else '[]'

    for dep_id in added_deps:
        parent = db.query(Task).filter(Task.id == dep_id).first()
        if parent:
            parent_next = _parse_json_list(parent.next_step)
            if task_id not in parent_next:
                parent_next.append(task_id)
                parent.next_step = json.dumps(parent_next)
```

### 5.2 依赖解锁流程

当任务完成时，`_unblock_downstream_tasks` 自动解锁所有依赖它的下游任务：

```python
def _unblock_downstream_tasks(completed_task_id: str, db: Session):
    """
    当任务完成时，自动解锁依赖它的所有 waiting 下游任务。

    触发时机：任务状态变为 done/completed 时
    """
    # 1. 查询 task_dependencies 表，找到所有依赖 completed_task_id 的任务
    downstream = db.execute(
        text("""
            SELECT task_id FROM task_dependencies
            WHERE dependency_id = :dep_id
        """),
        {"dep_id": completed_task_id}
    ).fetchall()

    # 2. 对每个下游任务检查是否所有依赖都已满足
    for row in downstream:
        child_id = row[0]
        child = db.query(Task).filter(Task.id == child_id).first()
        if not child or child.status not in ('waiting', 'paused', 'todo'):
            continue

        child_deps = _parse_json_list(child.depends_on)
        all_deps_done = all(
            db.query(Task).filter(Task.id == dep_id).first().status in ('done', 'completed')
            for dep_id in child_deps
        )

        if all_deps_done:
            child.status = 'todo'
            child.paused_reason = None
            child.blocked_reason = None
```
