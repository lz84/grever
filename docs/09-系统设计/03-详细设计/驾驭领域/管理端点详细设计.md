## 1. 概述

### 1.1 设计目标

Nexus 管理端点是 Reins（御）系统的管理面板，提供 Agent 重注册、任务重置、僵尸任务清理等运维能力。核心设计原则为 **运维友好**：提供直观的操作接口，快速定位和解决问题。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| Agent 管理 | Agent 列表查询、状态设置、重注册 |
| 任务管理 | 任务列表查询、状态重置 |
| 僵尸清理 | 批量清理孤儿任务和超时任务 |
| 心跳日志 | 记录 Agent 心跳历史 |

## 2. 核心流程

### 2.1 Agent 管理流程

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           Agent 管理流程                                      │
└──────────────────────────────────────────────────────────────────────────────┘

Case 1: Agent 重注册
  Agent 离线后恢复
  → POST /admin/agents/{id}/reregister
  → 状态恢复为 online
  → 心跳重置
         │
Case 2: Agent 强制下线
  Agent 异常需手动下线
  → POST /admin/agents/{id}/force-offline
  → 回收其任务
  → 状态设为 offline
         │
Case 3: Agent 状态设置
  手动设置状态
  → POST /admin/agents/{id}/set-status
  → 验证状态值
  → 更新状态
         │
Case 4: Agent 重启
  完全重启 Agent
  → POST /admin/agents/{id}/restart
  → 回收任务 + 重注册
```

### 2.2 任务重置流程

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           任务重置流程                                        │
└──────────────────────────────────────────────────────────────────────────────┘

Task 异常需重置
  → POST /admin/tasks/{id}/reset
  → status → todo
  → assigned_agent → NULL
  → 错误信息 → 清空
  → 重新进入调度队列
```

### 2.3 僵尸任务清理流程

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           僵尸任务清理                                        │
└──────────────────────────────────────────────────────────────────────────────┘

POST /admin/cleanup/zombie-tasks

Step 1: 孤立任务清理
  status = in_progress AND assigned_agent NOT IN agents
         │
Step 2: verifying 超时清理
  status = verifying AND updated_at < (now - 1h)
         │
Step 3: blocked/timeout 超时清理
  status IN (blocked, timeout) AND updated_at < (now - 24h)
         │
Step 4: offline agent 任务清理
  status = in_progress AND assigned_agent IN offline agents
         │
Step 5: 返回清理报告
  {success, cleaned_count, details}
```

## 6. 错误处理

### 6.1 错误码定义

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 404 | Agent/Task 不存在 | 返回 404 错误 |
| 400 | 无效的状态值 | 返回 400 错误，列出有效值 |

### 6.2 验证规则

```python
# Agent 存在验证
row = db.execute(
    text("SELECT id, name, status FROM agents WHERE id = :aid"),
    {"aid": agent_id}
).fetchone()
if not row:
    raise HTTPException(status_code=404, 
                        detail=f"Agent {agent_id} not found")

# Task 存在验证
row = db.execute(
    text("SELECT id, title, status FROM tasks WHERE id = :tid"),
    {"tid": task_id}
).fetchone()
if not row:
    raise HTTPException(status_code=404,
                        detail=f"Task {task_id} not found")

# 状态值验证
valid_statuses = ("online", "busy", "offline", "idle")
if request.status not in valid_statuses:
    raise HTTPException(status_code=400,
                        detail=f"Invalid status: {request.status}")
```

## 7. 与其他模块的关系

### 7.1 与 Scheduler 模块

- 重注册/重启 Agent 影响调度器任务分配
- 任务重置后重新进入调度队列
- 僵尸清理与调度器的孤儿任务恢复类似

### 7.2 与 Task 模块

- 任务重置修改 Task 状态
- Agent 任务回收调用 Task 模块

### 7.3 与 Agent 模块

- Agent 状态设置直接操作 agents 表
- 心跳日志关联 agents 表
