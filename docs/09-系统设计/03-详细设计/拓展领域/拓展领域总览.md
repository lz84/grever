## 一、拓展领域概述

拓展领域为 Nexus 系统提供可扩展的能力来源，包括场景模板、行业包和系统扩展功能。

### 1.1 子领域清单

| # | 子领域 | 职责 | 对应文档 |
|---|--------|------|---------|
| 1 | 场景库 | 场景模板存储、匹配、实例化 | [场景库详细设计](./场景库详细设计.md) |
| 2 | 行业包 | 行业标准包、定制包、生命周期 | [行业包详细设计](./行业包详细设计.md) |
| 3 | 系统扩展 | MCP 服务器、Skills、附件、产出物 | 待补充 |

## 二、子领域交互关系

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   场景库      │    │   行业包      │    │  系统扩展     │
│ (模板/实例化) │◄───│ (标准/定制)  │    │ (MCP/Skills)  │
└──────────────┘    └──────────────┘    └──────────────┘
       │                    │                    │
       └────────────────────┼────────────────────┘
                            ▼
                    ┌──────────────┐
                    │  驾驭领域     │
                    │ (调度核心)    │
                    └──────────────┘
```

### 2.1 核心流程

1. **场景来源**：用户自定义定义 或 GrASP 认知总结 → 存储到场景库
2. **场景实例化**：Goal 激活时 → 场景库匹配最适场景 → 从模板裁剪出初始 Project/Task
3. **行业包定制**：行业包提供标准模板 + 定制规则 → 影响场景实例化结果
