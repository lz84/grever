## 一、概述

### 1.1 模块定位

系统扩展模块是拓展领域的核心能力层，提供 MCP Server 管理、技能管理、统一附件管理和 AI 产出物管理四大子模块。

### 1.2 子模块清单

| 模块 | 职责 | 端点数 | 对应接口文档 |
|------|------|--------|-------------|
| **MCP** | MCP Server 实例管理、工具发现与调用 | 7 | [系统扩展接口.md](../../04-接口契约/系统扩展接口.md) |
| **Skills** | Agent 技能包的元信息、文件结构和安装引导 | 5 | 同上 |
| **Attachments** | 统一附件管理，支持所有实体的文件附件 | 7 | 同上 |
| **Artifacts** | AI 生成的完整文件产出物的存储和检索 | 6 | 同上 |

## 二、MCP 模块详细设计

### 2.1 MCP Server 数据模型

```sql
CREATE TABLE mcp_servers (
    server_id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    url TEXT NOT NULL,
    description TEXT,
    status TEXT NOT NULL DEFAULT 'inactive',  -- active / inactive / error
    tools_count INTEGER DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 2.2 MCP 工具发现流程

```
GET /api/v1/mcp-servers/{server_id}/tools
    ↓
向 MCP Server 发送 tools/list 请求（JSON-RPC）
    ↓
解析返回的工具列表
    ↓
写入本地缓存（tools_count 更新）
    ↓
返回工具列表
```

## 四、Attachments 模块详细设计

### 4.1 统一附件数据模型

```sql
-- 附件元信息
CREATE TABLE attachments (
    id TEXT PRIMARY KEY,
    filename TEXT NOT NULL,
    original_filename TEXT NOT NULL,
    file_path TEXT NOT NULL,          -- 物理存储路径
    mime_type TEXT,
    file_size INTEGER NOT NULL,
    created_by TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    -- 安全字段
    is_public BOOLEAN DEFAULT FALSE,
    expires_at DATETIME               -- NULL = 永不过期
);

-- 附件-实体关联表
CREATE TABLE attachment_references (
    id TEXT PRIMARY KEY,
    attachment_id TEXT NOT NULL REFERENCES attachments(id) ON DELETE CASCADE,
    entity_type TEXT NOT NULL,        -- goal / project / task / scenario / agent
    entity_id TEXT NOT NULL,
    display_name TEXT,                -- 可选覆盖文件名
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (attachment_id, entity_type, entity_id)
);

CREATE INDEX idx_ref_entity ON attachment_references(entity_type, entity_id);
CREATE INDEX idx_ref_attachment ON attachment_references(attachment_id);
```

### 4.2 上传流程

```
POST /api/v1/attachments/upload
    ↓
验证文件大小（默认上限 100MB）
    ↓
验证文件类型（禁止 exe/bat/sh/cmd/ps1/vbs）
    ↓
生成 UUID 作为文件 ID
    ↓
存储到物理路径：attachments/{year}/{month}/{uuid}_{filename}
    ↓
写入 attachments 表
    ↓
如果指定了 entity_type/entity_id，写入 attachment_references 表
    ↓
返回附件元信息
```

### 4.3 下载权限控制

- 非公开附件：通过 JWT 验证用户身份，只有创建者或关联实体 owner 可下载
- 公开附件：任何已认证用户可下载
- 过期附件：所有用户不可下载

## 五、Artifacts 模块详细设计

### 5.1 产出物数据模型

```sql
CREATE TABLE artifacts (
    artifact_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    type TEXT NOT NULL,                 -- code / report / image / other
    content TEXT NOT NULL,              -- 产出物内容（文本）
    file_path TEXT,                     -- 如果是文件类型（图片等）
    file_size INTEGER,
    metadata TEXT,                      -- JSON：language, model, version 等
    version INTEGER DEFAULT 1,
    created_by TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 产出物版本管理

每次 `PATCH` 更新时：
1. `version` 字段 +1
2. 旧内容保留在 `artifacts` 表，不删除
3. `updated_at` 更新为当前时间
