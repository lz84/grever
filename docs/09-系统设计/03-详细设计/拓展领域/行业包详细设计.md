## 一、概述

行业包（Industry Pack）是拓展域的重要组成部分，为 Nexus 系统提供可复用的行业解决方案模板。每个行业包封装特定行业的标准能力标签、场景模板和知识条目，支持标准包和定制包两种形态。

## 二、行业包设计

### 2.1 标准包 vs 定制包

| 类型 | pack_type | 说明 | base_pack_id |
|------|-----------|------|--------------|
| 标准包 | `standard` | 预置的行业能力包，客户可直接使用 | 无 |
| 定制包 | `custom` | 客户在标准包基础上定制扩展 | 必须指定 |

### 2.2 行业包数据结构

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | TEXT (PK) | 包 ID | ✅ |
| name | TEXT | 包名称 | ✅ |
| industry | TEXT | 行业标识 | ✅ |
| version | TEXT | 版本号 | ✅ |
| pack_type | TEXT | 类型：standard / custom | ✅ |
| base_pack_id | TEXT | 基础标准包 ID（仅定制包） | 定制包必填 |
| description | TEXT | 包描述 | 否 |
| status | TEXT | 状态：draft / published / archived | ✅ |
| created_at | INTEGER | 创建时间 | ✅ |
| updated_at | INTEGER | 更新时间 | 否 |

### 2.3 定制包约束

```sql
CREATE TRIGGER validate_custom_pack_insert
BEFORE INSERT ON industry_packs
WHEN NEW.pack_type = 'custom' AND NEW.base_pack_id IS NULL
BEGIN
    SELECT RAISE(ABORT, 'custom_pack_requires_base: 定制包必须指定 base_pack_id');
END;
```

## 三、包实例化流程

### 3.1 实例化路径

```
行业包（IndustryPack）
    │
    ├── 标准包 ──→ 实例化 ──→ Goal + Project + Task（capability_tags 填充）
    │
    └── 定制包 ──→ 加载定制内容 ──→ 合并标签/场景/知识
                     │
                     └── 实例化 ──→ Goal + Project + Task（capability_tags 含定制标签）
```

### 3.2 定制内容合并规则

| 操作 | 说明 |
|------|------|
| add | 新增标签/场景/知识到定制包 |
| modify | 修改标准包中已有标签/场景/知识 |
| remove | 在定制包中标记不引用某标准项 |

### 3.3 定制标签命名空间规则

- 标准包标签格式：`行业前缀:能力标识符`（如 `chem:hazmat-identification`）
- 定制包标签格式：`行业前缀:客户缩写:能力标识符`（如 `chem:xxpark:specific-hazmat`）
- 正则校验：
  - 标准标签：`re.match(r'^[a-z]+:[a-z][a-z0-9-]*$', tag_id)`
  - 定制标签：`re.match(r'^[a-z]+:[a-z]+:[a-z][a-z0-9-]*$', tag_id)`

### 3.4 实例化 API

```
POST /api/v1/industry-packs/{pack_id}/instantiate
Body: {
  "goal_name": "XX园区危化品泄漏处置"
}
→ 创建 Goal，Task 的 capability_tags 从定制包的标签填充
```
