## 1. 概述

**Agent SDK** 是 Nexus 平台的**Agent 能力增强组件**，提供本地和远程协同的能力层。

**核心价值**:
- **本地能力独立存在**:即使 Nexus 悟不认可，Agent 也可用本地认知、本地驾驭、本地进化
- **本地 + 远程协同**:本地处理基础任务，远程协同增强能力
- **渐进式依赖**:本地方案可用，远程协同增强

### 1.1 定位对比

| 错误理解 | 正确理解 |
|---------|---------|
| Agent 开发工具 | Agent 能力增强组件 |
| 讲怎么开发 Agent | 讲怎么增强 Agent 能力 |
| 连接 Nexus 为主 | 本地 + 远程协同 |
| 依赖远程服务 | 本地优先，远程增强 |

### 1.2 架构原则（关键修正）

**核心原则**:SDK 和技能的出入口**必须是 Agent**,不接受外部直接 API 调用。

| 错误理解 | 正确理解 |
|---------|---------|
| SDK 有 API 接口 | SDK 没有 API，是 Agent 内部组件 |
| 技能有 API 接口 | 技能没有 API，是 Agent 内部能力 |
| 外部可以直接调用 SDK/技能 | 外部必须通过 Agent 调用 SDK/技能 |

**正确理解**:

1. **SDK = Agent 端组件**:运行在 Agent 进程内，没有网络 API
2. **技能 = Agent 端组件**:运行在 Agent 进程内，没有网络 API
3. **SDK/技能的出入口 = Agent**:外部只能通过 Agent 调用 SDK/技能
4. **不接受直接 API 调用**:SDK/技能不是独立服务，不能作为 API 暴露

**架构示意**:

```
┌─────────────────────────────────────────────────────────┐
│                    外部调用者                            │
│              (用户 / 其他 Agent / API)                    │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│                      Agent                              │
│  ┌───────────────────────────────────────────────────┐  │
│  │  Agent SDK (能力增强组件)                          │  │
│  │  ┌─────────────┐  ┌─────────────┐  ┌──────────┐  │  │
│  │  │  Grasp      │  │  Reins      │  │  Reach   │  │  │
│  │  │  (认知)     │  │  (驾驭)     │  │  (方案)  │  │  │
│  │  └─────────────┘  └─────────────┘  └──────────┘  │  │
│  │                                                   │  │
│  │  ┌─────────────┐  ┌─────────────┐                │  │
│  │  │  Evo        │  │  Vigil      │                │  │
│  │  │  (进化)     │  │  (安全)     │                │  │
│  │  └─────────────┘  └─────────────┘                │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │  本地技能注册表                                    │  │
│  │  - 技能 1 (本地工具调用)                           │  │
│  │  - 技能 2 (本地数据处理)                           │  │
│  │  - 技能 N (扩展能力)                               │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                     │
                     ▼
            内部能力，无外部 API
```

**关键区别**:

1. **SDK ≠ API 服务**:SDK 运行在 Agent 进程内，不对外暴露 API
2. **技能 ≠ API 服务**:技能是 Agent 的本地能力，不对外暴露 API
3. **调用路径**:外部 → Agent → SDK/技能 (必须通过 Agent)
4. **不是独立服务**:SDK/技能不能作为独立服务运行

### 1.3 设计理念

**核心原则**:

1. **本地能力优先**:Agent 需要本地能力，不完全依赖远程
2. **本地认知系统**:即使 Nexus 悟不认可，本地也可用
3. **本地方案可用**:即使未上传到 Nexus，本地也可用
4. **远程协同增强**:与 Nexus 远程协同，获取更强的能力

### 1.4 参考设计

参考 **ontology skill** 的本地知识库设计模式:
- 本地存储:`memory/ontology/graph.jsonl`
- 本地 schema:`memory/ontology/schema.yaml`
- 本地查询：支持离线使用
- 远程同步：可选上传到 Nexus

**注意**:这些能力层都是 Agent 的内部组件，**没有 API 接口**。外部调用必须通过 Agent 方法。

### 2.2 核心组件

| 能力层 | 组件 | 职责 | 参考设计 |
|--------|------|------|---------|
| 认知 | GraspMemory | 本地记忆存储 | ontology graph.jsonl |

### 2.3 本地优先原则

**流程说明**:

场景：用户发起需要认知支持的任务

1. **先查本地记忆**:检查本地知识是否可用、本地缓存是否更新
2. **【本地可用】**:使用本地能力，无需联网 → 体验流畅，无网络依赖
3. **【本地不可用或需增强】**:请求远程协同，包括请求 Nexus 悟认知、请求 Nexus 化经验、请求 Nexus 鉴安全

## 3. 能力层详解

### 3.1 认知系统 (Grasp)

**定位**:即使 Nexus 悟不认可，本地也可用的认知系统

**设计参考**:ontology skill 的本地知识库设计

**调用方式**:通过 Agent 方法调用，不对外暴露 API

**核心组件**:

**本地存储初始化**:

- **存储路径**:本地内存目录下的 `graph.jsonl`和`schema.yaml` 文件
- **初始化流程**:创建本地目录、创建默认 schema、加载本地图数据

**默认 Schema 定义**:

定义本地可用的实体类型，包括:
- **Person**:需要 name 字段
- **Task**:需要 title 和 status 字段，status 为枚举类型 (open/in_progress/done)
- **Project**:需要 name 字段
- **Document**:需要 title 和 path 字段

**关系定义**:

- **has_owner**:项目/任务与人之间的拥有关系，多对一
- **blocks**:任务之间的阻塞关系，无环约束

**本地实体操作**:

- **创建实体**:生成实体 ID、类型、属性、创建时间、更新时间，追加到本地图文件
- **查询实体**:遍历本地图文件，按类型和过滤条件匹配，返回符合条件的实体列表
- **过滤器匹配**:检查实体属性是否满足过滤条件

**同步到远程**:

- **读取本地图**:从本地文件读取实体数据
- **调用 Nexus 悟 API**:将本地数据上传到 Nexus 悟
- **处理同步结果**:返回同步响应

**使用示例**:

1. 初始化本地认知系统
2. 创建本地实体 (即使离线也可用)
3. 查询本地实体 (无需联网)
4. 可选：同步到 Nexus 悟 (需要联网)

**核心优势**:

- ✅ **离线可用**:无需网络连接即可使用
- ✅ **数据私有**:本地存储，数据不出本地
- ✅ **快速响应**:本地读取，无网络延迟
- ✅ **渐进增强**:可同步到 Nexus 悟，但不是必须

### 3.2 驾驭增强 (Reins)

**定位**:任务编排本地化，不依赖 Reins 远程服务

**调用方式**:通过 Agent 方法调用，不对外暴露 API

**核心功能**:

**本地工作流引擎**:

- **任务队列管理**:维护本地任务队列
- **工作流定义**:定义包含步骤、状态、时间的本地工作流
- **工作流注册**:注册工作流到引擎
- **任务执行**:获取任务上下文、执行任务逻辑、更新状态

**与 Reins 的关系**:

- **本地优先**:本地工作流可独立使用
- **远程增强**:可上传到 Reins 获取更强的编排能力
- **双向同步**:本地和远程可同步工作流定义

### 3.3 进化缓存 (Evo)

**定位**:经验本地存储，不依赖 Evo 远程服务

**调用方式**:通过 Agent 方法调用，不对外暴露 API

**核心功能**:

**本地经验缓存**:

- **缓存目录**:本地目录下的经验文件
- **经验记录**:包含任务 ID、经验内容、时间戳、标签
- **缓存操作**:保存经验到本地文件、从本地读取经验、按标签搜索经验
- **搜索功能**:支持按标签过滤，遍历缓存文件进行匹配

**与 Evo 的关系**:

- **本地优先**:经验可本地缓存，无需上传
- **远程增强**:可上传到 Evo 获取全局经验共享
- **增量同步**:本地缓存 + 远程同步，双重保障

### 3.4 方案库 (Reach)

**定位**:工具本地注册，不依赖 Reach 远程服务

**调用方式**:通过 Agent 方法调用，不对外暴露 API

**注意**:技能没有 API 接口，必须通过 Agent 调用。

**核心功能**:

**本地技能注册表**:

- **技能注册**:记录技能名称和实现
- **技能获取**:根据名称查找技能
- **技能调用**:获取技能、获取凭证、执行调用、记录性能指标

**凭证管理**:

- **本地凭证**:本地存储管理凭证
- **凭证获取**:根据技能需求获取对应凭证

**性能监控**:

- **性能记录**:记录技能调用耗时
- **性能查询**:查询技能性能数据

**与 Reach 的关系**:

- **本地优先**:工具可本地注册和调用
- **远程增强**:可从 Nexus 获取 Skill 市场
- **双向同步**:本地和远程 Skill 可同步

### 3.5 安全防护 (Vigil)

**定位**:策略本地执行，不依赖 Vigil 远程服务

**调用方式**:通过 Agent 方法调用，不对外暴露 API

**核心功能**:

**本地策略加载**:

- **策略目录**:本地目录下的策略文件
- **基础策略**:加载基础安全策略
- **用户策略**:加载用户自定义策略
- **策略合并**:合并基础策略和用户策略

**策略执行**:

- **策略检查**:遍历规则列表，检查动作是否匹配
- **策略评估**:评估上下文是否满足策略要求 (如权限检查)
- **违规记录**:记录策略违规事件

**与 Vigil 的关系**:

- **本地优先**:策略可本地执行，无需网络
- **远程增强**:可同步到 Nexus 鉴获取全局策略
- **本地兜底**:即使网络断，本地策略依然有效

## 4. 与 Nexus 远程协同

本地能力层可独立使用，也可与 Nexus 远程协同增强。

### 4.1 认知同步：本地→Nexus 悟

**同步流程**:

1. **读取本地实体**:按类型筛选或获取所有实体
2. **上传到 Nexus 悟**:调用同步 API，携带实体数据、来源、时间戳
3. **处理同步结果**:返回同步响应

**同步时机**:

- 定期同步：每天一次
- 手动同步：用户主动触发
- 增量同步：检测到本地变化时

### 4.2 经验上报：本地→Nexus 化

**上报流程**:

1. **获取本地经验**:按任务 ID 获取或搜索所有经验
2. **上报到 Nexus 化**:调用上报 API，携带经验数据和来源
3. **获取全局经验**:返回全局经验列表

**上报策略**:

- 本地优先：本地缓存优先，上传可选
- 批量上报：积累一定数量后批量上传
- 隐私保护：敏感信息本地脱敏

### 4.3 安全审计：本地→Nexus 鉴

**审计流程**:

1. **获取本地审计日志**:从本地获取审计日志
2. **上传到 Nexus 鉴**:调用审计 API，携带日志数据、时间戳
3. **获取远程策略更新**:返回更新后的策略列表
4. **更新本地策略**:同步远程策略到本地

**审计策略**:

- 本地审计：本地记录审计日志
- 远程审计：可选上传到 Nexus 鉴进行全局分析
- 策略同步：远程策略可下发到本地

## 7. 总结

### 7.1 核心价值

**Agent SDK** 作为 Nexus 平台的**能力增强组件**,提供:

1. **本地能力独立存在**:五个本地能力层，不依赖 Nexus 即可使用
2. **本地 + 远程协同**:本地优先，远程增强
3. **渐进式依赖**:即使 Nexus 悟不认可，Agent 也可用本地方案

### 7.2 使用场景

**场景一：离线使用**

- ❌ **错误**:外部直接调用 SDK
- ✅ **正确**:通过 Agent 方法调用

**场景二：本地 + 远程协同**

1. 先使用本地能力
2. 需要增强时同步到远程

**场景三：网络断开时的兜底**

1. 尝试远程同步
2. 网络断开时，使用本地能力兜底
3. 本地能力依然可用

### 7.3 设计原则

1. **本地优先**:所有能力首先本地可用
2. **远程增强**:远程协同是增强，不是必须
3. **渐进依赖**:从本地到远程，逐步扩展
4. **离线兜底**:网络断开时，本地能力依然可用

## 9. 关键修正总结

### 9.1 架构原则修正

| 项目 | 旧理解 | 新理解 |
|------|--------|--------|
| SDK 定位 | 有 API 的工具 | Agent 端组件，无 API |
| 技能定位 | 有 API 的能力 | Agent 端能力，无 API |
| 调用方式 | 外部直接调用 | 必须通过 Agent 调用 |
| 架构关系 | SDK 是独立服务 | SDK 是 Agent 内部组件 |

### 9.2 调用路径

```
错误路径：外部 → SDK/技能（❌ 不允许直接 API 调用）
正确路径：外部 → Agent → SDK/技能（✅ 必须通过 Agent）
```

### 9.3 设计要点

1. **SDK 没有 API**:SDK 运行在 Agent 进程内，不对外暴露网络接口
2. **技能没有 API**:技能是 Agent 的本地能力，不对外暴露网络接口
3. **Agent 是出入口**:所有对 SDK/技能的调用必须通过 Agent 的方法
4. **不是独立服务**:SDK/技能不能作为独立服务运行，必须依附于 Agent
