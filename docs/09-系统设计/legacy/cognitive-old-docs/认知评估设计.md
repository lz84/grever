# 认知评估设计（Cognitive Assessment）

**整理人**: 刚子  
**讨论时间**: 2026-04-17  
**状态**: 初稿

---

## 一、定位

认知评估是 **Grasp 子系统** 的一部分，不是 Nexus 核心业务模型的内容。

**核心目的**：评估知识库的健康度，帮助用户了解认知数据的质量。

**不是**：
- ❌ 不是评估 Agent 的能力
- ❌ 不是评估任务的知识覆盖率

**而是**：
- ✅ 评估知识库中认知文档的质量

---

## 二、评估维度

当前实现 4 个维度：

| 维度 | 说明 | 计算方式 |
|------|------|---------|
| **检索质量** | 检索出的认知质量和置信度 | 基于 cognitions.jsonl 中的置信度字段 |
| **上下文利用率** | 标签覆盖率和元数据完整度 | 认知文档的标签数量、元数据完整度 |
| **注入准确率** | 注入后直接发布的比例 | 注入后未经修改直接发布的认知占比 |
| **知识新鲜度** | 认知的时效性 | 基于创建时间和最后使用时间 |

---

## 三、数据来源

| 数据源 | 路径 | 说明 |
|--------|------|------|
| 认知文档 | `data/cognitions.jsonl` 或 `skills/grasp/memory/grasp/cognitions.jsonl` | Grasp 知识库 |
| 追踪报告 | `data/memory/reins/` 或 `data/memory/` | 执行追踪报告 |

---

## 四、API

```
GET /api/v1/grasp/cognition-assessment/{agent_id}
```

返回指定 Agent 的认知评估结果，包含：
- `overall_score` — 总体评分
- `dimensions` — 各维度评分
- `stats` — 统计数据
- `recommendation` — 改进建议
