# Sprint 78: 迭代闭环 — 讨论驱动执行

## 背景
Sprint 77 实现了讨论区，但讨论结束后不会自动更新约束和触发下一轮迭代。

## 改动范围

### 后端
1. **讨论结束检测** — AI 分析对话，判断是否达成共识
   - 关键词：可以/执行/同意/好/开始
   - 或 AI 主动判断对话已充分
   
2. **共识触发约束更新** — 达成共识后：
   - 解析讨论内容，提取约束调整（工期/成本/安全系数）
   - 更新 `iteration_constraints` 表（写入新轮次约束）
   - 更新 `goal_iterations` 状态为 `executing`
   
3. **自动触发下一轮迭代** — 约束更新后：
   - 调用 `auto_capture_solution` 或创建新 tasks
   - 触发 ProjectExecutor 启动下一轮

### 前端
1. **讨论区状态指示** — 达成共识后显示：
   - ✅ "达成共识，已更新约束，开始执行 Iteration N+1"
   - 禁用输入框（当前轮已结束）
   
2. **迭代计划预览** — 显示下一轮计划：
   - 预期任务列表
   - 约束变化对比（vs 上一轮）

## Done Criteria
- [ ] 讨论达成共识后自动更新 constraints
- [ ] 自动创建下一轮迭代记录
- [ ] 前端显示共识状态
- [ ] Python 编译 + TS 编译 0 errors
