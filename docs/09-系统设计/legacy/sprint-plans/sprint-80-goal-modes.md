# Sprint 80：目标详细页三种模式对比与实现

> **日期**：2026-05-14
> **触发**：目标详细页三种模式共用一个页面，但不同模式下展示内容完全不同
> **根因**：之前的实现没有按模式区分展示逻辑，所有模式看到的页面基本一样

---

## 一、三种模式的本质区别

| 维度 | 常规模式 (Normal) | 迭代模式 (Optimization) | 探索模式 (Exploration) |
|------|-------------------|------------------------|----------------------|
| **一句话** | "按流程执行，做完就完" | "每轮是一个工程，根据结果决定下一轮" | "找到预设目标的最优解" |
| **目标** | 按既定流程一次性完成 | 多轮迭代推进 → 目标收敛 | 多轮迭代优化方案 → 对比选最优 |
| **迭代单元** | 无迭代 | 每轮 = 一个完整工程 | 每轮 = 一个方案 |
| **推进方式** | 线性执行 | 串行工程推进，工程A结果 → 决定工程B安排 | 并行方案竞争，方案A/B/C → 对比选优 |
| **AI 角色** | 不参与决策 | 项目经理：每轮结束分析结果，建议下一轮 | 方案设计师：每轮优化方案，对比优劣 |
| **人机交互** | 人启动 → 执行 → 人验收 | AI分析 → 人确认/调整 → AI执行下一轮 | AI生成方案 → 人确认/调整 → AI优化下一轮 |
| **结束条件** | 所有任务完成 | 收敛（评分不再显著提升）| 找到满意方案 / 达到最大轮次 |
| **类比** | 标准项目管理 | Sprint 迭代（每轮是一个 Sprint）| 科研实验（探索解空间） |

## 二、页面展示差异

### 共同部分（三种模式都有）
- 目标标题 + 状态 + 描述
- 执行进度（已完成任务 / 总任务）
- 工作目录信息
- 验证智能体设置
- 运行模式卡片（显示当前模式 + 参数 + 修改按钮）
- 目标流程（Workflow 可视化）
- 关联工程列表
- 任务列表

### 常规模式专属
- ❌ 无迭代控制面板
- ❌ 无方案对比
- ❌ 无收敛进度
- ❌ 无迭代历史
- 最简洁，只有执行信息

### 迭代模式专属
- ✅ **迭代控制面板**：当前轮次、下一轮/暂停/宣布收敛按钮
- ✅ **迭代历史**（工程视角）：
  - 每个 Iteration 是一个"工程"
  - 显示：工程做了什么、结果如何、AI 建议下一轮做什么
  - 人机交互：AI 建议卡片 → [同意] [调整] [跳过本轮] → 确认
  - 类似项目复盘的时间线
- ✅ **收敛进度条**：X/N 轮，已收敛/未收敛状态
- ❌ 无方案对比卡片

### 探索模式专属
- ✅ **迭代控制面板**：当前轮次、下一轮/暂停/宣布收敛按钮
- ✅ **方案对比卡片**：
  - 横向展示所有生成的方案
  - 每个方案显示：名称、评分、状态（最优/合规/不合规/已拒绝）
  - 维度评分：工期、成本、安全等
  - 绿色高亮最优方案
- ✅ **迭代历史**（方案视角）：
  - 每个 Iteration 对应一个方案的生成和优化
  - 显示：方案名称、评分、优化轨迹
  - 人机交互：AI 分析方案 → 建议调整方向 → 人确认 → 生成下一方案
- ❌ 无收敛进度条（用方案数量替代）

## 六、数据模型

### goal_iterations 表（两种模式通用）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | 迭代ID |
| goal_id | TEXT FK | 目标ID |
| iteration_number | INT | 迭代序号 |
| solution_id | TEXT FK | 关联方案（探索模式用） |
| score | REAL | 评分 |
| status | TEXT | completed / planned / active |
| ai_analysis | TEXT | AI 分析报告 |
| ai_suggestion | TEXT (JSON) | 结构化建议：{analysis, suggestion, comparison} |
| human_response | TEXT | 人的回复 |
| constraint_adjustments | TEXT (JSON) | 人调整后的约束值 |
| ai_discussion | TEXT (JSON) | 对话历史 [{role, content, timestamp}] |
| started_at | TEXT | 开始时间 |
| completed_at | TEXT | 完成时间 |

### goals 表 mode 字段
| mode | 说明 |
|------|------|
| normal | 常规模式 |
| exploration | 探索模式 |
| optimization | 迭代模式 |

## 七、前端展示逻辑

```tsx
// GoalDetail.tsx 伪代码

const mode = goal.mode; // 'normal' | 'exploration' | 'optimization'

// 迭代控制面板：探索模式 + 迭代模式 都显示
{(mode === 'exploration' || mode === 'optimization') && (
  <IterationControlPanel goalId={id} />
)}

// 方案对比卡片：仅探索模式显示
{mode === 'exploration' && (
  <SolutionComparison goalId={id} />
)}

// 收敛进度条：仅迭代模式显示
{mode === 'optimization' && (
  <ConvergenceProgress goalId={id} maxRounds={goal.max_rounds} />
)}

// 迭代历史：探索模式 + 迭代模式 都显示，但内容视角不同
{(mode === 'exploration' || mode === 'optimization') && (
  <IterationHistory goalId={id} mode={mode} />
)}
```

## 八、后端 API（两种模式通用）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/goals/{goal_id}/iterations` | GET | 获取迭代列表 |
| `/goals/{goal_id}/iterations/{iter_id}/analysis` | POST | 生成 AI 分析 |
| `/goals/{goal_id}/iterations/{iter_id}/confirm` | POST | 确认 AI 建议 |
| `/goals/{goal_id}/iterations/{iter_id}/adjust` | POST | 调整约束 + 创建下一轮 |
| `/goals/{goal_id}/iterations/{iter_id}/start-next` | POST | 启动下一轮迭代 |
| `/goals/{goal_id}/iteration-status` | GET | 获取迭代运行状态 |
| `/goals/{goal_id}/iterations/{iter_id}/discuss` | GET/POST | 获取/发送讨论消息 |

后端 API 对两种模式是通用的，区别在于前端如何展示。
