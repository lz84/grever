## 一、业务流程梳理

### 流程 A：完整流程（Goal → Scenario → Workflow → Project → Task）

```
用户输入 Goal（某地7.2级地震救援）
  ↓
系统匹配 Scenario（显示推荐预案列表）
  ↓ 用户选择"地震救援预案"
  ↓
实例化 Goal 级 Workflow（6 阶段）
  ↓
用户通过对话修正 Workflow（"把阶段1改名为快速响应"）
  ↓
点击"确认并拆分"
  ↓
自动创建 6 个 Project（每个阶段一个）+ 6 个 Task（execution 节点）
  ↓
Project 按 phase_order 排序
  ↓
Task 分配给 Agent 并执行
  ↓
Agent 执行 → Task 状态变更 → Goal 完成
```

### 流程 B：精简流程（Project → Scenario → Workflow → Task）

```
直接创建 Project → 匹配 Project 级 Scenario
→ 实例化 Project 级 Workflow
→ 节点创建 Task → 分配 Agent → 执行
```

### 流程 C：极简流程（Task 直接执行）

```
直接创建 Task → 分配 Agent → 执行 → 反馈
```

### 其他业务流程

| 流程 | 描述 |
|------|------|
| 对话式编辑 | Workflow DAG 对话编辑（重命名/插入/删除/合并/移动） |
| Task 执行 | 领取 → 执行 → 反馈 |
| 争议管理 | 创建争议 → 讨论 → 仲裁 → 关闭 |
| 成果物 | 上传 → 查看 → 下载 → 删除 |
| 场景库 | 浏览 → 详情 → 匹配 |
| 认知库 | 查询 → 注入 → 评估 |
| Agent 管理 | 注册 → 状态 → 能力 |
| 安全 | 监控 → 告警 → 审计 |
