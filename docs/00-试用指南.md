# Nexus 试用指南

**版本**: v0.1
**日期**: 2026-05-27
**适用对象**: 同事/朋友试用
**预计时间**: 15 分钟

---

## 一句话介绍

> **Nexus 是多 AI Agent 协同管理平台。**
> 你有一个目标，系统自动分解成任务，分配给最合适的 AI 去执行，人在关键节点把关。

**核心卖点**：从"有个目标"到"活干上了"，中间那段时间省掉。

---

## 体验路线

### 第 1 步：打开系统（1 分钟）

浏览器访问 `http://localhost:5173/`

你应该看到：
- 左侧菜单
- 中间仪表盘

### 第 2 步：看看有什么 Agent（1 分钟）

左侧菜单 → **智能体管理**

你应该看到 5 个 Agent：
- **刚子**（CEO/指挥调度）
- **谷子**（CFO/股票交易）
- **麻子**（CTO/代码开发）
- **蚊子**（CIO/内容创作）
- **扣子**（开发专员）

绿色的 = 在线，灰色的 = 离线。

> Agent 在线需要 OpenClaw Gateway 在运行。如果全灰，不影响试用其他功能。

### 第 3 步：创建一个目标（3 分钟）

左侧菜单 → **目标管理** → **创建目标**

填写：
- **目标名称**: 比如 "完成代码审查"
- **描述**: 比如 "对 utils/api.ts 文件进行代码审查，检查 API 调用是否规范"
- **选择场景**: 如果有预设场景，选一个；没有就跳过

点击 **创建**。

### 第 4 步：看系统如何分解（2 分钟）

目标创建后，系统会自动分解成：
- **Projects**（项目）
- **Tasks**（任务）

进入目标详情 → **项目列表** → 点击一个项目 → **任务列表**

你应该看到：
- 任务之间有依赖关系（DAG）
- 每个任务有优先级、能力标签

### 第 5 步：看看场景库（2 分钟）

左侧菜单 → **场景库**

这里展示可用的场景模板：
- **场景模板** = 预设的业务流程
- 选择一个场景 → 查看详情 → 可以看到里面定义了哪些项目和任务

### 第 6 步：看看能力标签（2 分钟）

左侧菜单 → **能力标签库**

这里展示系统的"能力词典"：
- 每个 Agent 有什么能力
- 每个任务需要什么能力
- 匹配引擎根据这个自动分配任务

---

## 重点体验

| 功能 | 看什么 | 为什么重要 |
|------|--------|-----------|
| **目标 → 项目 → 任务分解** | 三层结构是否清晰 | 这是核心概念 |
| **DAG 依赖关系** | 任务之间的先后顺序 | 这是调度的基础 |
| **能力标签匹配** | Agent 和任务如何通过标签关联 | 这是自动分配的核心 |
| **场景模板** | 预设流程是否直观 | 这是行业包的载体 |
| **HITL（人在回路）** | 任务可以配置人工审批节点 | 这是差异化能力 |

## 如果你只想看 1 个功能

**看这个**：目标管理 → 选择一个目标 → 看它分解成什么

**为什么**：这一个页面就能说清楚 Nexus 是干什么的。

1. **你第一眼看到系统时，觉得它是干什么的？**
