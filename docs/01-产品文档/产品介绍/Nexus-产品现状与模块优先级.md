## 二、模块优先级排序

### 2.1 排序原则

1. **销售演示优先**：能直接给客户看的模块优先开发
2. **依赖关系**：基础模块先做，上层模块后做
3. **已有 API 优先**：后端已有的模块先做前端，快速出活
4. **核心差异化优先**：进化层（Evo）是核心差异化，不能太晚做

### 2.2 优先级排序

| 优先级 | 模块 | 对应层 | 理由 | 预计工期 |
|--------|------|--------|------|----------|
| **P0** | 01-工作台 | 入口 | 给客户第一印象，所有模块的入口，API 已有 | 1 周 |
| **P0** | 02-协同中心 | 御 | 核心功能，目标→项目→任务→执行链路，API 已有 | 2 周 |
| **P0** | 07-系统管理 | 基础设施 | 智能体管理是基础，API 已有 | 1 周 |
| **P1** | 05-认知中心 | 悟 | 差异化能力（认知底座），部分 API 已有 | 2 周 |
| **P1** | 08-可视化 | 可视化 | 给客户看的看板/Trace，现有数据可拼 | 2 周 |
| **P2** | 03-场景库 | 达 | 核心差异化（场景复用），需新增 API | 3 周 |
| **P2** | 06-安全中心 | 鉴 | 安全织网，需新增 API | 2 周 |
| **P3** | 04-技能库 | 化 | 核心差异化（进化能力），需新增 API | 3 周 |

### 2.3 分阶段开发计划

#### 第一阶段：MVP 演示版（3 周）
**目标**：能跑通完整演示流程，给客户看

| 模块 | 工期 | 交付物 |
|------|------|--------|
| 工作台 | 1 周 | 全局概览、统计卡片、快捷入口 |
| 协同中心 | 1 周 | 目标→项目→任务→执行 全链路 |
| 系统管理 | 1 周 | 智能体列表/详情 |

**演示流程**：
1. 打开工作台 → 看到全局数据
2. 进入协同中心 → 创建目标 → 自动分解 → 分配智能体 → 执行监控
3. 进入系统管理 → 查看智能体状态
4. 进入认知中心 → 查看知识库

#### 第二阶段：差异化能力版（+4 周）
**目标**：展示核心竞争力

| 模块 | 工期 | 交付物 |
|------|------|--------|
| 认知中心 | 2 周 | 知识库 + 评估 + 注入管理 |
| 可视化 | 2 周 | 看板 + Trace 查看 + 报表 |

**演示新增**：
1. 认知中心 → 查看认知资产 + 评估报告
2. 可视化 → 看板展示 + 执行 Trace 回放

#### 第三阶段：完整能力版（+5 周）
**目标**：五层架构完整展示

| 模块 | 工期 | 交付物 |
|------|------|--------|
| 场景库 | 3 周 | 场景列表 + 详情 + 收藏 + 反馈回收 |
| 安全中心 | 2 周 | 监控 + 审计 + 告警 |

**演示新增**：
1. 场景库 → 查看成熟场景 + 实例化
2. 安全中心 → 审计日志 + 告警

#### 第四阶段：技能库版（+3 周）
**目标**：核心差异化（自举性进化）

| 模块 | 工期 | 交付物 |
|------|------|--------|
| 技能库 | 3 周 | 技能列表 + 详情 + Capsule 市场 + 进化轨迹 |

**演示新增**：
1. 技能库 → 查看技能列表 + Capsule 市场 + 进化轨迹

### 2.4 总工期估算

| 阶段 | 时间 | 累计 | 状态 |
|------|------|------|------|
| MVP 演示版 | 3 周 | 3 周 | 可给客户看 |
| 差异化能力版 | 4 周 | 7 周 | 可销售 |
| 完整能力版 | 5 周 | 12 周 | 可交付 |

## 三、关键依赖与风险

### 3.1 后端 API 依赖

| 模块 | 缺失 API | 优先级 | 阻塞前端 |
|------|----------|--------|----------|
| 认知中心 | `/api/v1/grasp/inject/*` | P1 | ⚠️ 前端可 mock |
| 场景库 | 所有 API | P2 | ❌ 必须后端先做 |
| 安全中心 | `/api/v1/audit/logs`、`/api/v1/alerts` | P2 | ❌ 必须后端先做 |
| 技能库 | 所有 API | P3 | ❌ 必须后端先做 |

### 3.2 风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 后端 API 开发滞后 | 前端阻塞 | 第一阶段用 mock 数据，第二阶段补真实 API |
| 场景库 API 设计复杂 | 工期延长 | 先做列表/详情，反馈回收后续迭代 |
| 进化层（Evo）是核心差异化但排最后 | 销售演示缺乏亮点 | 第一阶段至少做 UI mock，展示概念 |
