# 天将：智能体协同驾驭平台

让 AI 从"能回答问题"走向"能自主协同、持续进化、全程可信"

---

## 你面临的真实问题

联勤保障/大型复杂指挥体系正在经历什么？

- 有多个专业 AI 智能体，但各自为战，无法协同
- 有成熟的预案和 SOP，但新任务来临时，无法快速组合、调整、执行
- 有经验丰富的指挥员，但经验是隐性的、不可传承的，人员变动直接导致能力断层
- 有自动化工具，但只能执行固定流程，遇到突发情况就"卡死"，需要人手动干预
- 有 AI 平台，但无法审计"AI 做了什么决策、依据是什么、谁批准的"

**核心痛点：智能体是孤立的、经验是隐性的、协同是不可控的、决策是不可溯的**

---

## 解决方案

天将：不是又一个 AI 平台，而是一套**智能体协同驾驭体系**。

把分散的智能体、预案、认知、经验，组织成可协同、可进化、可审计的统一驾驭平台。

一句话理解：

- 你有 5 个专业智能体 → 天将 让它们像"五兄弟"一样分工协作
- 你有一套预案 SOP → 天将 把它变成可实例化、可修改、可反馈的活体场景
- 你有一个指挥员经验 → 天将 把经验沉淀为场景库，下次直接用

## 已验证的场景（联勤保障适用）

### 场景 1：应急保障任务链协同

- **输入**：突发事件情报、可用资源清单、保障预案库
- **处理**：自动匹配预案 → 分解任务 → 分配情报分析/物资调度/医疗研判智能体 → 执行 → 监控
- **输出**：完整保障方案 + 各智能体执行报告 + 资源调度建议
- **价值**：从"人找预案、人分任务"到**"智能体自动协同、指挥员做决策"**

### 场景 2：物资调度智能研判

- **输入**：物资需求、库存数据、运输能力、历史案例
- **处理**：物资研判智能体分析需求 → 运输智能体规划路线 → 调度智能体分配资源 → 异常时自动调整
- **输出**：最优调度方案 + 风险预警 + 备选方案
- **价值**：从"经验驱动调度"到**"数据 + 预案 + 智能体协同"**综合决策

### 场景 3：保障预案沉淀与进化

- **输入**：历次保障任务的执行数据（成功/失败/耗时/冲突）
- **处理**：自动提取成功模式 → 沉淀为场景库 → 标注成功率/耗时/适用条件 → 下次任务自动推荐
- **输出**：带数据背书的成熟场景库，越用越准
- **价值**：从"每次任务从头摸索"到**"站在历史经验上决策"**

### 场景 4：多智能体安全管控

- **输入**：所有智能体的执行行为日志
- **处理**：实时审计 + 异常检测 + 权限拦截 + 信任评分
- **输出**：安全态势看板 + 告警 + 行为回溯报告
- **价值**：从"出了事再查"到**"全程可控、实时拦截、事后可溯"**

## 实现路径：五层体系

天将 以五层架构构建完整的智能体驾驭体系，每层各司其职：

### 第一层：认知底座

把文档、条令、预案、专家经验组织成可推理的知识体系。
不是"存一堆文档"，而是理解对象、关系、规则，形成**可查询的认知资产**。

> 对标：Palantir Ontology（本体建模），但我们更开放、可定制、Agent 原生集成。

### 第二层：协同驾驭

接收目标 → 自动分解任务 → 匹配最合适的智能体 → 协同执行 → 异常自愈。

> 对标：AutoGen / LangGraph（协同框架），但我们是配置即可用的企业平台，不是开发框架。

### 第三层：技能库

从执行中提取成功经验 → 封装为可复用能力 → 在智能体之间传播。

> **全球唯一**：竞品普遍无进化能力。AutoGen Memory 只是"记住"，化是"进化 + 传承"。

### 第四层：场景库

把验证过的成熟方案沉淀为场景，标注成功率、耗时、适用条件，供下次任务直接实例化。

> 对标：OpenAI Agent Store（Agent 分发），但我们是能力+经验复用市场，粒度更细，带数据背书。

### 第五层：安全织网

贯穿所有层：权限执行 + 行为审计 + 异常检测 + 信任评分。智能体用可靠行为"挣"信任，用违规行为"丢"权限。

> 对标：Okta for AI / Agat Guardian，但我们是 Agent 原生安全，不是身份安全或模型安全的后加模块。

## 快速响应话术

### 客户问：这和 Palantir AIP 有什么区别？

Palantir 是"数据融合 + 任务映射"的封闭平台，天将 是"智能体协同 + 自举进化"的开放平台。
Palantir 没有进化能力（执行完就结束了），天将 每次执行都会沉淀经验、进化能力。
Palantir 学习曲线陡峭、价格昂贵（百万美元/年），天将 配置即可用、支持私有化部署。

### 客户问：这和 AutoGen / CrewAI 有什么区别？

AutoGen 和 CrewAI 是开发框架，需要程序员写代码才能用，面向研究场景。
天将 是企业级平台，配置即可用，内置可视化界面，面向生产环境。
更关键的是，它们都没有进化能力、没有原生安全、没有方案复用市场。

### 客户问：落地周期多久？

建议从单一场景切入（如应急保障任务链协同、物资调度智能研判）。
先跑通"目标分解 → 智能体协同 → 执行反馈 → 场景沉淀"的完整闭环，
验证效果后再逐步扩展到其他场景。
通常 **3 个月** 可看到首个 MVP 价值。

### 客户问：需要投入什么？

- 核心团队：1 名 AI 工程专家 + 1 名领域专家（联勤/指挥/保障）
- 数据准备：现有预案、SOP、历史任务数据、专家经验（无需特殊格式）
- 我们的角色：平台 + 方法论 + 场景落地 + 持续运营

### 客户问：智能体自己能进化？安全怎么保证？

是的，但**安全从第一天就内建于架构**：
- 所有智能体行为全程审计，可追溯、可回放
- 权限基于信任评分动态调整——越可靠权限越大
- 实时异常检测：越权尝试、频率异常自动拦截
- 人类保留最终否决权，可随时冻结任何智能体
- 信任不是一次给的，是智能体用可靠行为"挣"来的
