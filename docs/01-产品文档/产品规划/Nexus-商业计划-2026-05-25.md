## 一、产品定位

### Nexus 是什么

> **多 AI Agent 协同管理平台。**

让 AI 像团队一样工作，人在关键节点把关。

### Nexus 不是什么

| 不是 | 说明 |
|------|------|
| AI 开发框架 | 不是 LangChain、CrewAI、AutoGen |
| 聊天机器人 | 不是 ChatGPT、Claude |
| Agent 运行时 | 不是 OpenClaw、vLLM |

### 核心价值主张

> **从"有个目标"到"活干上了"，中间那段时间值多少钱。**

| 场景 | 以前 | 用 Nexus |
|------|------|---------|
| 化工应急 | 出事 → 打电话 → 找预案 → 分工 → 1-2 小时 | 选场景 → 点一下 → 15 分钟出预案 |
| 代码审查 | 人工逐行看 → 反馈 → 修改 → 1-2 天 | 自动分配 → AI 审查 → 汇总报告 → 2 小时 |
| Bug 修复 | 拉群 → 定位 → 分配 → 修复 → 验证 → 4-8 小时 | 选场景 → 自动走流程 → 1-2 小时 |

## 二、商业模式

### 核心公式

```
商业收入 = 行业包销售 + 企业功能授权 + 定制咨询 + 技术支持
```

### 四层产品结构

```
┌─────────────────────────────────────────────────┐
│              免费（开源社区版）                    │
│  核心引擎 + 场景库功能 + 示例场景 + 基础前端       │
└──────────────────────┬──────────────────────────┘
                       │ 安装行业包 ↓
┌──────────────────────┴──────────────────────────┐
│              收费（标准行业包）                    │
│  行业场景包 + 行业技能 + 行业知识 + Agent 方案     │
│  ¥5-10 万/年                                    │
└──────────────────────┬──────────────────────────┘
                       │ 定制工具生成 ↓
┌──────────────────────┴──────────────────────────┐
│              收费（定制行业包）                    │
│  标准包 + 客户定制 + 系统集成 + HITL 高级功能      │
│  ¥10-50 万（一次性 + 年维护）                     │
└──────────────────────┬──────────────────────────┘
                       │ 授权合作伙伴使用 ↓
┌──────────────────────┴──────────────────────────┐
│              收费（定制工具授权）                   │
│  行业包定制工具 → 咨询公司/集成商用它做定制        │
│  按年授权 / 作为企业版的一部分                    │
└─────────────────────────────────────────────────┘
```

> 行业包定制工具是**将定制咨询从手艺活变成生产线**的关键基础设施。详见《行业包定制工具-产品设计-2026-05-25.md》。

## 三、行业包：核心商业产品

### 什么是行业包

行业包不是软件功能，是**解决方案的交付形态**。

```
行业包 = 能力标签库 + 场景 + 技能 + 知识 + Agent方案
```

| 组件 | 是什么 | 示例（化工应急） |
|------|--------|-----------------|
| **能力标签库** | 行业能力词典，定义标签体系 | `professional: hazmat-identification`（危化品识别） |
| **场景** | 行业流程模板 | 危化品泄漏处置流程（12 个 Task） |
| **技能** | Agent 能力配置（工具 + 验证脚本 + capability_tags） | 扩散计算工具、MSDS 解析器 |
| **知识** | 行业规则 + 合规依据 | 《危险化学品事故应急救援预案编制导则》 |
| **Agent方案** | 团队角色配置（需要哪些 AI、各有什么能力） | 现场评估 Agent、决策支持 Agent、预案生成 Agent |

### 为什么这样设计

| 组件 | 作用 | 商业逻辑 |
|------|------|---------|
| 能力标签库 | 定标准 | 锁定生态，一旦用了就难切换 |
| 场景 | 展示价值 | 让客户看懂"这系统能干嘛" |
| 技能 | 让 Agent 能干活 | 开箱即用的能力组合 |
| 知识 | 提供决策底气 | 合规依据，让客户敢用 |
| Agent方案 | 告诉客户需要哪些 AI | 降低客户决策成本 |

### 行业包的分层

| 层级 | 内容 | 价格 |
|------|------|------|
| **开源示例包** | 2 个软件开发场景 + 1 个基础化工场景（通用流程） | ¥0 |
| **标准行业包** | 行业完整场景包（全场景 + 技能 + 知识库 + Agent 方案） | ¥5-10 万/年（待市场验证） |
| **定制行业包** | 标准包 + 客户定制 + 系统集成 + 培训 | 一事一议（根据定制范围和行业复杂度定价） |

> **定制定价逻辑**：软件开发行业的定制成本极低（有免费包在先，定制主要是参数配置）；复杂行业（化工、军工）定制成本较高，定价相应更高。

| 矛盾 | 解法 |
|------|------|
| 纯标准化卖不出去（每个园区情况不同） | 标准化骨架 + 定制化皮肤 |
| 纯定制化的做不大（一个人干不过来） | 标准化骨架复用，定制部分收费 |

**标准化骨架**：通用流程、通用决策规则、通用角色定义（开源版能看）

**定制化皮肤**：园区具体设施信息、人员组织架构、特殊预案、系统集成（收费）

### 行业包定制工具

**定位**：将定制咨询从手艺活变成生产线的 B2B 内部工具。

**用户不是终端客户，是咨询团队**（初期是你自己，未来是合作伙伴）。

| 模块 | 功能 | 输入 | 输出 |
|------|------|------|------|
| 预案解析器 | Word/PDF → 结构化场景 | 客户预案文档 | YAML 场景草案 |
| 场景编辑器 | 标准骨架上可视化定制 | 标准场景 + 客户修改 | 定制场景 |
| 组织架构映射器 | 组织架构 → Agent 方案 | 客户组织架构表 | Agent-角色绑定 |
| 设施参数配置器 | 设施信息 → 场景参数 | 设施表格/系统 | 场景参数配置 |
| 验证与发布 | 验证 + 打包 | 定制后的行业包 | .nexus 格式文件 |

**商业模式**：

| 模式 | 说明 |
|------|------|
| 自用 | 提高定制效率，一个人干原来三个人的活 |
| 授权合作伙伴 | 咨询公司/集成商用它做定制，按年授权 |
| 企业版内置 | 大型企业买企业版后，可以自己定制行业包 |

**开发路线**：先手动定制 1-2 个客户摸流程 → 半自动（场景编辑器 + 参数配置器）→ 智能解析（LLM 解析文档）→ 全自动定制流水线。详见《行业包定制工具-产品设计-2026-05-25.md》。

## 四、开源策略

### 开源范围

| 开源 | 不开源 |
|------|--------|
| 核心调度引擎（DAG + 能力匹配 + 派发） | 行业包（场景 + 技能 + 知识 + Agent 方案） |
| 场景库功能（管理、浏览、安装、运行） | 企业功能（HITL 高级版/多租户/SSO/审计/高可用） |
| ORM 模型 + DB schema + 迁移脚本 | 客户定制代码 |
| 基础前端 | **行业包定制工具**（核心商业资产） |
| 3 个示例场景（软件开发 2 个 + 化工基础 1 个） | |
| API 文档 + 架构文档 | |

### 许可证

**Apache 2.0**（商业友好，降低企业使用门槛）

### HITL 的分法

| 开源版 | 收费版 |
|--------|--------|
| 基础 HITL（单个审批节点） | 多级审批流 + 超时升级 + 多通道通知（飞书/钉钉/企微） |

### 场景库策略

- **场景库功能开源**：让人能跑起来、能试用、能看到"原来我可以装一个场景就能用"
- **场景内容收费**：客户买的不是场景文件，是别人的领域知识 + 最佳实践

### 社区运营计划

| 阶段 | 时间 | 目标 | 关键动作 |
|------|------|------|---------|
| 第 1 步：开源引擎 | 1-2 个月 | GitHub 上线 | 代码清理 + 写文档 + 快速开始 + 技术文章导流 |
| 第 2 步：建社区 | 3-6 个月 | 100-500 star | 接受 PR + 场景编写指南 + 技术文章每月 1-2 篇 |
| 第 3 步：卖行业包 | 6-12 个月 | 第一个付费客户 | 行业包商业化 + 企业功能 + 案例 |

## 五、目标客户

### 客户画像

| 客户 | 痛点 | 我们怎么解决 | 定价 |
|------|------|-------------|------|
| 化工园区/应急管理局 | 出事到出预案要 1-2 小时 | 选场景 → 15 分钟出预案，每一步有人负责 | 方案费 10-50 万 |
| 技术团队/IT 部门 | AI 各干各的，没人协调 | "帮你管好它们" | 按年订阅，按 Agent 数 |
| 第三方安全服务机构 | 服务客户时缺工具 | 标准化应急方案 + 工具 | 按客户数授权 |

### 冷启动策略

| 阶段 | 找谁 | 为什么 |
|------|------|--------|
| 第 1 步 | 朋友的公司/熟人团队 | 容忍度高，能帮你发现真实问题 |
| 第 2 步 | 中小化工企业 | 决策快，痛点真实 |
| 第 3 步 | 大园区/应急管理局 | 需要案例背书才能敲开门 |

**不要一开始找大客户。** 大客户要求高、周期长，东西还没打磨好就进去了，做坏了口碑就没了。

## 六、竞争差异化

| 维度 | CrewAI/AutoGen | LangChain | Dify/Coze | Nexus |
|------|---------------|-----------|-----------|-------|
| 定位 | Agent 框架 | Agent 框架 | AI 应用编排 | **协同管理平台** |
| 用户 | 开发者 | 开发者 | 开发者+业务 | **业务决策者 + 开发者** |
| HITL | 没有 | 手动加 | 有限 | **内置，每个任务可配置** |
| 场景 | 没有 | 没有 | 有工作流模板 | **行业包 = 场景 + 技能 + 知识 + Agent 方案** |
| 能力匹配 | 手动指定 | 无 | 无 | **四维标签自动匹配** |
| 多 Agent 协调 | 有限 | 无 | 有限 | **原生 DAG + 派发 + 验收** |
| 国内访问 | 受限 | 受限 | ✅ 可用 | ✅ 国内优先适配 |

### 一句话差异化

> **别人要么全自动（走偏了不管），要么全手动（效率低）。我们是"AI 干活，人把关"。**

### 切换壁垒

客户从现有方案切换到 Nexus 的成本：
- **从 CrewAI/AutoGen**：迁移成本低（都是代码），但需要重新理解"场景"概念
- **从 Dify/Coze**：迁移成本中等，Nexus 的 HITL 和行业包是核心差异
- **从零开始**：Nexus 就是起点，没有切换成本

> 国内市场 Nexus 优先适配百炼/vLLM，海外市场优先 OpenAI/Claude。

## 八、收入预测

### 竞品定价参考（2026-05-25 调研）

| 竞品 | 免费版 | 付费版（Cloud） | 企业版 |
|------|--------|---------------|--------|
| **Dify** | Sandbox（免费） | Professional $59/mo（~¥400/月）、Team $159/mo（~¥1100/月） | 联系销售（估计 ¥20-50 万/年） |
| **Coze（扣子）** | 个人免费 | 按用户 ¥39-99/月 | 联系销售 |
| **CrewAI/AutoGen** | 开源免费 | 无 | 无 |
| **LangChain** | 开源免费 | LangSmith 有付费计划 | 联系销售 |

**定位差异**：竞品是"工具型产品"（按用户/按用量计费），Nexus 是"解决方案交付"（行业包 + 定制）。**我们不卖工具，卖的是"帮你把 AI 用起来"的方案**。

### 第一个付费客户

> **来源：通过关系网络接触的企业或政府用户。**

| 类型 | 特点 | 预计签约周期 |
|------|------|-------------|
| 企业客户 | 决策快、痛点真实、但预算有限 | 1-3 个月 |
| 政府客户 | 预算充足、决策慢、需要合规文件 | 3-12 个月 |

**冷启动策略不变**：先找朋友的公司/熟人团队做免费试用，发现真实问题，打磨后再签单。

### 收入阶段（初步规划）

| 阶段 | 时间 | 重点 | 预期 |
|------|------|------|------|
| 冷启动 | 0-6 个月 | 开源上线 + 试用 + 打磨产品 | ¥0 |
| 第一单 | 6-12 个月 | 第一个付费客户（通过关系网络） | 收入从 0 到 1 |
| 验证模式 | 12-18 个月 | 用前两个客户案例说服第三个 | 收入增长，关键验证期 |
| 规模化 | 18-24 个月 | 复制成功模式，扩大客户群 | 待定（需第一单数据修正） |

> ⚠️ **以上是规划假设，不是承诺。** 第一个客户签单后，用实际数据修正。

### 定价说明

- **标准行业包**：¥5-10 万/年（待市场验证）。参考 Dify 企业版 ¥20-50 万/年，Nexus 的行业包包含更多解决方案成分（场景 + 技能 + 知识 + Agent 方案），定价可在此区间内。软件开发行业的免费行业包在定制时成本极低（接近 0）；复杂行业（化工、军工）定制成本较高，定价相应更高。
- **定制行业包**：根据定制范围和行业复杂度一事一议。
- **定制工具授权**：¥2-5 万/年（工具授权费，服务费另算）。

### 知识来源

行业包中的"知识"组件（包括行业规范、合规文档、最佳实践等）由业务专家提供。初期由团队内部积累（如软件开发业务专家、化工应急业务专家），随客户增加逐步扩展知识库。

## 九、开源社区到付费客户的转化路径

### 设计思路

GitHub star 不等于客户。转化路径的核心是：**让开源用户先跑起来，然后在关键节点发现"这个我自己做太麻烦了，不如买现成的"。**

### 转化漏斗

```
┌─────────────────────────────────────────────────┐
│  阶段 1：知道 Nexus                               │
│  渠道：GitHub、技术文章、社区分享、口碑传播         │
│  目标：star 数增长                                │
│  关键动作：写技术文章、提交好用的代码、回复 Issue   │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  阶段 2：本地部署 + 试用                          │
│  触发：clone 仓库 → 跑起来 → 看到示例场景         │
│  关键指标：部署成功率、首次运行时间                │
│  关键动作：一键部署脚本 + 3 个示例场景（开箱即用）  │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  阶段 3：发现"缺场景"                            │
│  触发：用户跑通了，但发现自己行业没有场景包       │
│  关键动作：社区里分享"如何编写场景"教程            │
│  阻力点：写场景需要懂 YAML + 业务，门槛较高        │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  阶段 4：考虑买行业包                            │
│  触发：发现"自己写场景太费劲，买个现成的省事"     │
│  转化点：标准行业包 ¥5-10 万/年                  │
│  关键动作：行业包 demo 环境（让客户先试后买）      │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  阶段 5：定制需求                                │
│  触发：标准包不够用，需要结合自身情况定制         │
│  转化点：定制行业包 ¥10-50 万（一事一议）         │
│  关键动作：提供定制咨询服务                       │
└─────────────────────────────────────────────────┘
```

### 关键转化节点

| 从阶段 | 到阶段 | 转化条件 | 预计转化率 |
|--------|--------|---------|-----------|
| 知道 → 试用 | star → clone | 有清晰文档 + 示例场景 | ~10%（技术社区正常水平） |
| 试用 → 发现缺场景 | 跑起来 → 想找自己的行业场景 | 场景库里有多个行业包 | ~30%（跑起来后自然需求） |
| 发现缺 → 买行业包 | 自己写太费劲 → 买现成的 | 有 demo 环境 + 试用版 | ~10%（决策周期 3-6 个月） |
| 行业包 → 定制 | 标准包不够用 | 业务复杂度高 | ~50%（企业客户大概率需要定制） |

> ⚠️ **转化率数据为行业参考值，非实际数据。** 第一个客户签单后修正。

### 转化路径的核心逻辑

1. **开源版是"体验版"**：让客户先用起来，而不是先卖后试
2. **行业包是"捷径"**：客户自己写场景要花 1-2 个月，买行业包 1 小时装好
3. **定制是"最后一公里"**：行业包不是万能，每个客户都需要结合自身情况
