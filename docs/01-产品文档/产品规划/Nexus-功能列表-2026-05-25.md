## 三、多 Agent 平台对接 ★

### 3.1 平台适配器框架

| ID | 功能 | 优先级 | 版本 | 说明 |
|----|------|--------|------|------|
| PA-01 | **Agent 抽象层定义** | 🔴 P0 | (OSS) | 统一 Agent 接口（dispatch/execute/verify/heartbeat） |
| PA-02 | **适配器注册机制** | 🔴 P0 | (OSS) | 动态注册/发现平台适配器 |
| PA-03 | **OpenClaw 适配器** | 🔴 P0 | (OSS) | 已有，需重构为适配器模式 |

### 3.2 Agent 抽象层设计

```
┌─────────────────────────────────────────────┐
│              Nexus 调度引擎                    │
│   (DAG 调度 + 能力匹配 + HITL + 验收)          │
└──────────────────────┬──────────────────────┘
                       │ 统一接口
┌──────────────────────┴──────────────────────┐
│            Agent 抽象层（Agent Abstraction）   │
│                                             │
│   interface AgentPlatform:                  │
│     - dispatch(task) → execution_id         │
│     - get_status(execution_id) → status     │
│     - get_result(execution_id) → result     │
│     - cancel(execution_id)                  │
│     - heartbeat() → platform_status         │
│     - get_capabilities() → [tags]           │
└──┬──────┬──────┬──────┬──────┬──────┬──────┘
   │      │      │      │      │      │
   ▼      ▼      ▼      ▼      ▼      ▼
 OpenClaw OpenAI Claude 百炼  vLLM  Dify  ...
```

### 3.3 适配器优先级决策

| 适配器 | 优先级 | 理由 |
|--------|--------|------|
| OpenClaw | 🔴 已有 | 当前唯一适配器 |
| OpenAI API | 🟡 第一 | 最通用，客户最可能有 |
| 阿里百炼 | 🟡 第二 | 国内客户最常用 |
| vLLM 本地 | 🟡 第三 | 私有化部署刚需 |
| Claude | 🟡 第四 | 部分客户偏好 |
| Dify/Coze | 🟢 后续 | 工作流平台，集成复杂度高 |
