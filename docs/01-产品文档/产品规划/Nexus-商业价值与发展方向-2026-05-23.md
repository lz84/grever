## 一、我们手上到底有什么

### 已跑通的核心链路

```
选场景模板 → 自动分解为 Projects/Tasks → 匹配 Agent → 自动派发执行 → 验收回收
```

**这不是技术描述，这是产品价值。**

它解决了一个很具体、很贵的问题：

> **从"有个目标"到"活干上了"，中间那段时间值多少钱。**

### 真实资产盘点

| 资产 | 说明 | 状态 |
|------|------|------|
| **一套跑得通的引擎** | 目标分解 + 能力匹配 + DAG调度 + 自动派发 + 验收 | ✅ 已跑通 |
| **化工园区应急领域知识** | 危化品泄漏、火灾、气体扩散等场景流程 | ⚠️ 有知识，需结构化 |
| **软件开发实践经验** | 代码审查、Bug修复、项目初始化等日常流程 | ⚠️ 在脑子里，需沉淀 |
| **前端界面** | 能看、能操作、能演示 | ✅ 可用 |
| **五层架构设计** | Grasp/Reins/Evo/Reach/Vigil | 📄 设计文档，代码实现率 30-75% |

### 不是什么

| 不是 | 说明 |
|------|------|
| AI 开发框架 | 不是 LangChain、不是 CrewAI |
| 聊天机器人 | 不是 ChatGPT、不是 Claude |
| Agent 运行时 | 不是 OpenClaw、不是 vLLM |

**我们做的是：让 AI 像团队一样协同工作的平台。**

## 二、商业价值：两条路并行

### 路径一：卖场景模板（短期变现）

**定位**：行业 AI 协同方案

**商业模式**：不是卖软件授权，是卖"方案包"

```
方案包 = 场景模板 × N + 平台部署 + 培训
```

#### 目标客户

| 客户 | 买什么 | 痛点 | 定价逻辑 |
|------|--------|------|---------|
| 化工园区/应急管理局 | 危化品泄漏+火灾+气体扩散 场景 | 出事到出预案要 1-2 小时 | 方案费，10-50 万 |
| 技术团队/IT 部门 | 代码审查+Bug修复+项目初始化 场景 | AI 各干各的，没人协调 | 按年订阅，按 Agent 数 |
| 第三方安全服务机构 | 标准化应急方案 | 服务客户时缺工具 | 按客户数授权 |

#### 卖的不是工具，是"把脑子里的流程变成 AI 能执行的方案"

客户不会说"我需要一个多 Agent 协同平台"，他们会说：

- "出了事，我希望 15 分钟内知道该干什么、谁来干"
- "新项目启动，我不想每次都从头搭"
- "线上出了 Bug，我不想花 4 小时拉群定位"

**我们卖的就是"把这几句话变成自动化流程"。**

#### 关键场景（化工应急）

| 场景 | 优先级 | 以前 | 现在 |
|------|--------|------|------|
| 危化品泄漏 | 🔴 先做 | 打电话 → 找预案 → 分工 → 1-2h | 选场景 → 点一下 → 15min 出预案 |
| 储罐火灾 | 🟡 第二个 | 同上 | 同上 |
| 有毒气体扩散 | 🟡 第三个 | 同上 | 同上 |
| 公用工程中断 | 🟢 后面 | 同上 | 同上 |

#### 关键场景（软件开发）

| 场景 | 优先级 | 以前 | 现在 |
|------|--------|------|------|
| 代码审查 | 🔴 先做 | 人工逐行看 → 反馈 → 修改 | 自动分配 → AI 审查 → 汇总报告 |
| Bug 修复 | 🟡 第二个 | 拉群 → 定位 → 分配 → 修复 → 验证 → 4-8h | 选场景 → 自动走流程 → 1-2h |
| 项目初始化 | 🟡 第三个 | 开会讨论 → 分工 → 搭环境 → 2-3天 | 选场景 → 自动分解 → 半天 |
| 事故复盘 | 🟢 后面 | 凭记忆写报告 | 自动提取时间线 → 根因分析 → 改进项 |
| 数据迁移 | 🟢 后面 | 手动方案 → 验证 → 执行 → 回滚预案 | 标准化流程 → 自动验证 |

### 路径二：卖工具能力（中长期规模化）

**定位**：多 AI Agent 协同管理平台

**商业模式**：平台 + 场景模板市场 + 场景编辑器

```
平台 = 场景管理 + Agent协同 + 执行监控 + 验收 + 场景编辑器
```

#### 目标客户

| 客户 | 痛点 | 我们怎么解决 |
|------|------|-------------|
| 有 3 个以上 AI 但管不好的团队 | "AI 各干各的，没人协调" | "我们帮你管好它们" |
| 想用 AI 但不知道从哪开始的 IT 部门 | "单个 AI 好用，多个就乱" | "选个场景模板就开始" |
| 需要标准化流程的企业 | "每次都要从头想流程" | "场景模板拿来就用" |

#### 定价

| 模式 | 定价逻辑 | 适用客户 |
|------|---------|---------|
| SaaS | 按 Agent 数/月 | 中小团队 |
| 私有化 | 按年授权 + 实施费 | 大型企业/政府 |
| 开源引擎 + 企业版 | 核心免费，高级功能收费 | 开发者社区 → 企业客户 |

## 三、两条路的关系

**核心是同一套引擎，只是场景模板不同。** 就像同一辆卡车，换不同的货。

```
                    ┌─────────────────────┐
                    │   Nexus 协同引擎     │
                    │  (目标分解+调度+验收) │
                    └──────────┬──────────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
              ┌─────┴─────┐        ┌─────┴─────┐
              │ 化工应急   │        │ 软件开发   │
              │ 场景模板包 │        │ 场景模板包 │
              └───────────┘        └───────────┘
                    │                     │
              化工园区/应急局        技术团队/IT部门
              方案费 10-50万         订阅/授权
```

**路径一是现金流，路径二是规模。** 先用路径一验证价值、拿到客户，再用路径二规模化。

## 四、当前最大的障碍

### 不是技术问题，是"信任"问题

化工园区领导问你："你这套系统，我凭什么敢用？"

他听不懂"五层架构"，不关心"86 个 Sprint"。

他能听懂的只有一句话：

> **"你帮我把出事到出预案的时间从 2 小时缩到 15 分钟，而且每一步都有人负责、有记录可查。"**

### 信任来自三个东西

| 信任来源 | 怎么做 |
|---------|--------|
| **能看** | 3 分钟演示视频，让人一眼看懂 |
| **能试** | 免费试用，拿真实场景跑一遍 |
| **有人用** | 第一个付费客户就是最好的广告 |

## 六、HITL（人在回路）：必须做

### 为什么 HITL 是产品分水岭

**真实世界里没有全自动的高价值工作。**

| 场景 | 没有 HITL | 有 HITL |
|------|----------|---------|
| 软件开发 | AI 直接用了 Redis，但客户已经有 Memcached 运维工具 → 全白干 | AI 提方案，人拍板用 Memcached，AI 干活 |
| 化工应急 | AI 建议疏散半径 500 米，但下风向有学校 → 出人命 | AI 计算，人改成 1000 米，AI 重新执行 |
| 代码审查 | AI 判断"这段代码风格不对"但标准可能是错的 | 人定标准，AI 逐行检查，人最终确认 |

**HITL 不是拖慢效率，是让 AI 不会走偏。**

而且这是我们的**差异化优势**：

| 竞品 | HITL 能力 |
|------|----------|
| AutoGen/CrewAI | 没有 |
| LangChain | 手动加 Human-in-the-Loop 节点，很麻烦 |
| Palantir | 有，但只在顶层，粒度粗 |
| **Nexus** | **每个 Task 都可以设 condition_type，细粒度控制** |

### HITL 现有基础（已有 70%，缺关键桥梁）

**已经有的**：

| 组件 | 状态 | 说明 |
|------|------|------|
| `human_input_requests` 表 | ✅ 有 | 17 列，含 timeout/branches 等完整字段 |
| CRUD API | ✅ 有 | submit/reject/pending/stats 全都有 |
| 前端页面 | ✅ 有 | HumanInputPage / Dashboard / Analytics |
| `waiting_human` 状态 | ✅ 有 | Task 状态机已支持 |
| `DependencyResolver.unlock_on_human_input` | ✅ 有 | 人类提交后自动解锁下游任务 |
| 飞书通知 | ✅ 有 | 任务变 waiting_human 时推送 |
| 超时处理 | ✅ 有 | waiting_human 超时自动处理 |
| `condition_type` 字段 | ✅ 有 | scenario_projects + scenario_tasks 都有 |
| 四种 condition_type | ✅ 有 | none / auto_eval / human_decision / human_input |

**缺的关键桥梁**：

| 缺失 | 说明 | 影响 |
|------|------|------|
| **① 实例化不创建 HITL 请求** | `scenario_instantiate_v2.py` 的 `_evaluate_condition` 对 human_* 直接跳过，不创建 Task | human_decision 的 Task 根本不会出现在执行队列里 |
| **② 执行前不检查 condition_type** | Agent 领到任务直接干，不判断是否需要先等人类 | 应该"先审批再执行"的任务变成了"直接执行" |
| **③ 前端没有场景 HITL 编排视图** | HumanInputPage 是通用页面，没有和场景实例化关联 | 用户看不到"哪些任务在等人" |

### HITL 实现方案

**核心流程**：

```
场景蓝图定义
  → condition_type = human_decision（需人审批）/ human_input（需人提供数据）
  → 实例化为 Task
  → 执行链路检查 condition_type
  → 需要 HITL → 创建 human_input_request → Task 状态 = waiting_human
  → 飞书通知用户
  → 人在前端审批/输入
  → 提交 → Task 状态变 todo（或 done）
  → DependencyResolver 解锁下游依赖任务
  → 执行继续
```

**分三步走**：

| 步骤 | 做什么 | 文件 | 工作量 |
|------|--------|------|--------|
| **第一步：实例化创建** | 修改 `_evaluate_condition`，human_* 不跳过，改为创建 Task + human_input_request，状态设为 waiting_human | `scenario_instantiate_v2.py` | 半天 |
| **第二步：执行前拦截** | Agent 领任务时，检查 task 是否 waiting_human，是则跳过执行，等人类 | `task_runner.py` / `project_executor.py` | 半天 |
| **第三步：前端打通** | ScenarioDetail 页加"HITL 审批"Tab，展示当前场景下所有等待人类的节点 | `ScenarioDetail.tsx` | 1 天 |

**两种 HITL 类型的区别**：

| 类型 | 触发时机 | 人做什么 | 示例 |
|------|---------|---------|------|
| `human_decision` | 执行前 | 审批/拍板（同意/拒绝/修改方案） | "是否启动 I 级应急响应？" |
| `human_input` | 执行中 | 提供数据/信息 | "当前下风向风力几级？学校有多少人？" |

### HITL 的商业价值

**对化工应急客户**：

> "你的预案里每一步谁拍板、需要什么信息，都可以预设好。系统自动执行，到关键节点自动等你审批。不是替代人，是让 AI 帮你把预案跑起来。"

**对软件开发客户**：

> "AI 帮你写代码、做审查，但架构选型、代码风格标准，你来定。AI 干活，你把关。"

**差异化一句话**：

> **别人要么全自动（走偏了不管），要么全手动（效率低）。我们是"AI 干活，人把关"。**

## 七、战略定力

### 不做什么

| 不做什么 | 为什么 |
|---------|--------|
| 继续做大而全的五层架构 | 一个人做不过来，客户也不关心 |
| 做完整的验证流程组织 | 重要但优先级低于 HITL 和场景打磨 |
| 做场景市场生态 | 等有了 10 个以上真实场景再说 |

### 坚持做什么

| 坚持做什么 | 为什么 |
|-----------|--------|
| 把真实流程变成场景模板 | 这是核心竞争力 |
| **HITL 打通** | 没有 HITL，产品不可用，不是不好用的问题 |
| 让引擎跑得越来越快 | 技术债越拖越贵 |
| 找真实客户要反馈 | 闭门造车是死路 |

---

## 八、一句话总结

> **Nexus 不造 AI。Nexus 让 AI 像团队一样工作，人在关键节点把关。**

**卖给谁**：有 AI 但管不好的团队、需要标准化流程的行业。

**卖什么**：场景模板 + 协同引擎 + HITL 审批。

**怎么卖**：先让人看懂（演示），再让人试用（免费），最后让人买单（方案）。

**核心差异化**：别人要么全自动（走偏了不管），要么全手动（效率低）。我们是"AI 干活，人把关"。
