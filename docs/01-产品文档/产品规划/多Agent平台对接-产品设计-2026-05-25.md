**参与**: 用户 + 刚子

## 一、为什么需要对接多平台

### 现状

Nexus 当前只对接 OpenClaw 一个平台。所有 Task 派发都通过 OpenClaw CLI 完成。

### 问题

| 问题 | 影响 |
|------|------|
| 客户可能没有 OpenClaw | 没法用 Nexus |
| 客户可能已经买了其他 AI 平台 | 不愿意换 |
| 客户可能有多个 AI 平台混用 | 只能管一个 |
| 客户可能有本地部署的 vLLM | 无法利用 |
| 单个平台宕机 | 整个系统瘫痪 |

### 核心价值

> **Nexus 不应该绑定任何特定的 AI 平台。客户用什么 AI，Nexus 就管什么 AI。**

这就像 Kubernetes 不管你的容器是 Docker 还是 containerd 跑的——它只管调度。

## 三、架构设计

### 三层架构

```
┌─────────────────────────────────────────────────┐
│              Nexus 调度引擎                        │
│  (DAG 调度 + 能力匹配 + HITL + 验收)               │
│                                                   │
│  不关心底层用什么 AI，只管：                        │
│  - 派发给谁                                        │
│  - 什么时候派发                                    │
│  - 怎么验收结果                                    │
└──────────────────────┬──────────────────────────┘
                       │ 统一接口（AgentPlatform）
┌──────────────────────┴──────────────────────────┐
│           Agent 抽象层（Agent Abstraction Layer）  │
│                                                   │
│  定义所有平台必须实现的统一接口：                     │
│  - dispatch(task) → execution_id                 │
│  - get_status(execution_id) → status             │
│  - get_result(execution_id) → result             │
│  - cancel(execution_id)                          │
│  - heartbeat() → platform_status                 │
│  - get_capabilities() → [capability_tags]        │
└──┬────┬────┬────┬────┬────┬────┬────┬──────────┘
   │    │    │    │    │    │    │    │
   ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼
 Open  Open  Claude 百炼  火山  vLLM Dify 自定义
 Claw AI
```

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **调度引擎不知道平台** | 引擎只调用 AgentPlatform 接口，不关心是哪个平台 |
| **适配器注册制** | 新平台写一个适配器，注册到系统即可，不需要改核心代码 |
| **Agent 绑定平台** | 每个 Agent 注册时声明自己的平台类型和配置 |
| **模型与平台解耦** | 同一个平台可以有多个模型（如百炼有 qwen3.6-plus/qwen3-coder） |

## 四、Agent 抽象层

### 统一接口定义

```python
from abc import ABC, abstractmethod
from typing import Any, Optional
from dataclasses import dataclass
from enum import Enum

class ExecutionStatus(Enum):
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"
    TIMEOUT = "timeout"
    CANCELLED = "cancelled"

@dataclass
class TaskSpec:
    """派发给 Agent 的任务规格"""
    task_id: str
    description: str          # Task 描述
    context: dict             # 上下文（context_md 等）
    acceptance_criteria: list # 验收标准
    capability_tags: list     # 需要的能力标签
    timeout_seconds: int      # 超时时间
    metadata: dict            # 额外元数据

@dataclass
class ExecutionResult:
    """执行结果"""
    execution_id: str
    status: ExecutionStatus
    output: Any               # 输出内容
    error: Optional[str]      # 错误信息
    started_at: int           # 开始时间（Unix timestamp）
    completed_at: Optional[int]  # 完成时间
    cost: Optional[float]     # 调用费用（美元）
    tokens_used: Optional[dict] # token 使用情况

@dataclass
class PlatformStatus:
    """平台状态"""
    platform_type: str
    healthy: bool
    model_name: str
    latency_ms: int
    rate_limit_remaining: Optional[int]
    error: Optional[str]

@dataclass
class PlatformCapability:
    """平台能力"""
    tags: list[str]           # capability_tags
    models: list[str]         # 支持的模型列表
    max_concurrent: int       # 最大并发数
    supports_streaming: bool  # 是否支持流式输出

class AgentPlatform(ABC):
    """Agent 平台抽象基类"""
    
    @abstractmethod
    async def initialize(self, config: dict) -> None:
        """初始化平台连接"""
        pass
    
    @abstractmethod
    async def dispatch(self, task: TaskSpec) -> str:
        """派发任务，返回 execution_id"""
        pass
    
    @abstractmethod
    async def get_status(self, execution_id: str) -> ExecutionStatus:
        """查询执行状态"""
        pass
    
    @abstractmethod
    async def get_result(self, execution_id: str) -> ExecutionResult:
        """获取执行结果"""
        pass
    
    @abstractmethod
    async def cancel(self, execution_id: str) -> bool:
        """取消执行"""
        pass
    
    @abstractmethod
    async def heartbeat(self) -> PlatformStatus:
        """健康检查"""
        pass
    
    @abstractmethod
    async def get_capabilities(self) -> PlatformCapability:
        """获取平台能力"""
        pass
    
    @property
    @abstractmethod
    def platform_type(self) -> str:
        """平台类型标识"""
        pass
```

## 五、适配器注册机制

### 注册表

```python
# platforms/registry.py

_PLATFORM_REGISTRY: dict[str, type[AgentPlatform]] = {}

def register_platform(platform_type: str):
    """装饰器：注册平台适配器"""
    def decorator(cls: type[AgentPlatform]):
        _PLATFORM_REGISTRY[platform_type] = cls
        return cls
    return decorator

def get_platform(platform_type: str) -> AgentPlatform:
    """获取平台适配器实例"""
    if platform_type not in _PLATFORM_REGISTRY:
        available = list(_PLATFORM_REGISTRY.keys())
        raise ValueError(f"Unknown platform '{platform_type}'. Available: {available}")
    return _PLATFORM_REGISTRY[platform_type]()

def list_platforms() -> list[str]:
    """列出所有已注册的平台"""
    return list(_PLATFORM_REGISTRY.keys())
```

### 适配器示例

```python
# platforms/openai_adapter.py

from .registry import register_platform
from .base import AgentPlatform, TaskSpec, ExecutionResult, PlatformStatus, PlatformCapability

@register_platform("openai")
class OpenAIAdapter(AgentPlatform):
    
    def __init__(self):
        self.client = None
        self.config = None
    
    async def initialize(self, config: dict) -> None:
        from openai import AsyncOpenAI
        self.client = AsyncOpenAI(api_key=config["api_key"])
        self.config = config
    
    async def dispatch(self, task: TaskSpec) -> str:
        # 将 Task 转换为 OpenAI API 调用
        response = await self.client.chat.completions.create(
            model=self.config.get("model", "gpt-4"),
            messages=[
                {"role": "system", "content": "你是一个任务执行 Agent。"},
                {"role": "user", "content": task.description},
            ],
            max_tokens=task.timeout_seconds // 10,  # 粗略估算
        )
        # 生成 execution_id 并存储结果
        execution_id = f"openai-{uuid4().hex[:12]}"
        # 存储 execution 状态...
        return execution_id
    
    async def get_status(self, execution_id: str) -> ExecutionStatus:
        # 查询本地存储的执行状态
        pass
    
    async def get_result(self, execution_id: str) -> ExecutionResult:
        # 返回执行结果
        pass
    
    async def cancel(self, execution_id: str) -> bool:
        # OpenAI API 不支持取消已发出的请求
        return False
    
    async def heartbeat(self) -> PlatformStatus:
        try:
            await self.client.models.list()
            return PlatformStatus(
                platform_type="openai",
                healthy=True,
                model_name=self.config.get("model", "gpt-4"),
                latency_ms=0,
            )
        except Exception as e:
            return PlatformStatus(
                platform_type="openai",
                healthy=False,
                model_name="",
                latency_ms=0,
                error=str(e),
            )
    
    async def get_capabilities(self) -> PlatformCapability:
        return PlatformCapability(
            tags=["technical:llm", "professional:text-generation"],
            models=["gpt-4", "gpt-3.5-turbo"],
            max_concurrent=100,
            supports_streaming=True,
        )
    
    @property
    def platform_type(self) -> str:
        return "openai"
```

## 八、任务派发流程（多平台）

```
Task 需要派发
  ↓
1. 能力匹配引擎找到最佳 Agent
  ↓
2. 查看 Agent 的 platform_type
  ↓
3. 获取对应平台适配器
   platform = get_platform(agent.platform_type)
  ↓
4. 初始化平台连接（如果未初始化）
   await platform.initialize(agent.api_config)
  ↓
5. 派发任务
   execution_id = await platform.dispatch(task_spec)
  ↓
6. 记录到 executions 表
  ↓
7. 执行完成后，通过平台适配器获取结果
   result = await platform.get_result(execution_id)
  ↓
8. 更新 Task 状态，触发验收
```

## 九、模型路由

### 问题

同一个平台可以有多个模型，不同模型适合不同任务。

### 路由规则

```
Task 需求 → 模型选择引擎 → 最优模型

选择维度：
  - 能力匹配（模型的能力标签 vs Task 需要的标签）
  - 成本（每次调用费用）
  - 延迟（平均响应时间）
  - 可用性（当前是否健康）
  - 用户偏好（管理员设置的优先级）

示例：
  Task: "审查 Python 代码质量"
  → 需要: technical:python, technical:code-review
  → 候选模型:
     - gpt-4 (匹配度 0.9, 费用 $0.03/次, 延迟 800ms)
     - qwen3-coder (匹配度 0.8, 费用 $0.005/次, 延迟 500ms)
     - claude-sonnet (匹配度 0.85, 费用 $0.015/次, 延迟 600ms)
  → 按成本优先: 选 qwen3-coder
  → 按质量优先: 选 gpt-4
```

### 路由配置

```yaml
routing:
  strategy: "cost_first"  # cost_first / quality_first / balanced / custom
  rules:
    - condition: "task.capability_tags contains 'code-review'"
      prefer: ["gpt-4", "claude-sonnet"]
      avoid: ["gpt-3.5-turbo"]
    - condition: "task.capability_tags contains 'msds-parsing'"
      prefer: ["qwen3-coder"]
    - condition: "task.timeout_seconds < 30"
      prefer: ["qwen3.6-plus"]  # 快速响应
```

## 十、多平台容错（企业版）

### 场景

主平台（如 OpenAI）宕机时，自动切换到备选平台。

```
健康检查发现 OpenAI 不健康
  ↓
查找备选平台（相同能力标签）
  ↓
将 pending 任务重新派发到备选平台
  ↓
通知管理员平台切换
```

### 配置

```yaml
failover:
  openai:
    backup: ["claude", "bailian"]  # 备选平台顺序
    trigger: "unhealthy_for_5m"     # 触发条件
    notify: true                    # 通知管理员
  bailian:
    backup: ["vllm"]
    trigger: "unhealthy_for_2m"
```

## 十三、风险与对策

| 风险 | 可能性 | 影响 | 对策 |
|------|--------|------|------|
| 不同平台 API 差异大 | 高 | 高 | 抽象层只定义最小接口集，差异化在适配器内处理 |
| API Key 泄露 | 中 | 极高 | 加密存储（AES-256），不在日志中输出 |
| 平台速率限制 | 高 | 中 | 适配器内置速率控制，排队等待 |
| 平台返回格式不一致 | 高 | 中 | 适配器负责格式转换，统一输出 Nexus 标准格式 |
| 执行超时难以统一 | 中 | 高 | 各平台有自己的超时策略，调度层设置最大值 |
| 流式输出适配复杂 | 中 | 低 | 先不支持流式，后续迭代 |
